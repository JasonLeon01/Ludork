# 01. 概览

本文件夹记录从明文 Python 脚本中收集到的蓝图可见 API。由蓝图数据派生出来的生成类不包含在内。

## 阅读规则

- 原始名称使用 Python 模块路径和函数名。
- 当 API 声明了 `LOC("...")` 元数据时，显示名和描述从 `Sample/Data/Locale/Locale.xlsx` 解析。
- 如果没有本地化元数据，蓝图节点选择器会回退到函数名，本手册也采用同样规则。
- 类章节先列出明文类体中直接声明的公共类变量、适用的蓝图可编辑组件字段，再列出该类暴露给蓝图的 API 方法。
- `Source.NodeFunctions` 单独成篇，因为这些函数在节点选择器中作为独立节点组出现。

## 编辑器元数据

Python 函数和类可以用 `@Meta(...)` 附加编辑器侧提示。目前编辑器识别这些 key：

- `DisplayName` 和 `DisplayDesc`：节点显示名和描述，通常写成 `LOC("KEY")`。
- `DropBox`：节点参数下拉选项，例如 `{"attr": ["pitch", "volume"]}`。
- `Rely`：字段或参数之间的编辑依赖，例如 `{"target": ["source", True]}`。
- `PathVars`：以 `Assets` 下目录为根的路径字段或节点参数，例如 `[("texturePath", "Characters")]`。
- `ColourVars`：颜色字段或参数，例如 `["lightColour"]`。值仍保存 RGBA 元组，但在编辑器中显示颜色方块。
- `Vector2Vars`、`Vector2fVars`、`Vector2iVars`、`Vector2uVars`、`PairVars`、`PairFloatVars`、`PairIntVars`：二维字段或节点参数，在编辑器中显示为 x/y 数值编辑器。
- `Vector3Vars`、`Vector3fVars`、`Vector3iVars`、`Vector3uVars`：三维字段或节点参数，在编辑器中显示为 x/y/z 数值编辑器。
- `MoveRouteVars`：移动路线节点参数，在编辑器中显示为地图参考路线编辑器。值保存为相对 `(dx, dy)` 步进。
- `Transfer`：地图坐标节点参数，在编辑器中显示为地图格子选择对话框。格式：`[("posVarName", "mapVarName")]`。每个元组将坐标参数与另一个参数关联，后者的当前值用于在选择器中预选地图。确认选择后，若关联的地图参数为空或指向不存在的地图，会自动填入当前选中的地图 key。
- `InvalidVars`：在蓝图属性编辑器中隐藏的类变量，例如 `["defaultRect"]`。
- `RectRangeVars`：使用图片范围选择器编辑的矩形字段，例如 `{"defaultRect": "texturePath"}`。

带 Python 签名默认值的节点参数仍会显示在蓝图编辑器中。默认值应使用 Python 基础值（`None`、`bool`、`int`、`float`、`str`，或由这些值组成的容器）。如果默认值要指向引擎或编辑器对象，应写成字符串，并由函数内部显式处理。
