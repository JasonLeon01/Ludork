# 15. 数据、本地化与存档

本页说明 Sample 游戏运行时如何连接数据、本地化和存档。

## 数据引用

地图、蓝图、通用数据、本地化和素材会通过名称、标签、key 和路径互相引用。改名不只是外观变化：所有引用它的地图、事件、窗口和存档路径都必须仍能解析。

## 本地化文本

运行时本地化模块从 `Data/Locale` 加载导出的语言文件。这些文件的编辑与导出由官方本地化工具插件（`Plugins/OfficialLocaleTools`）处理，而不是引擎内置 UI。编辑器工作流见 `06.Data Localisation and Testing`，插件结构见 `23.Plugin System`。

```python
from Engine import Locale

title = Locale.GetContent("GAME_TITLE")
languages = Locale.GetLocaleKeys()
```

形如 `{SLIME}` 的文本会在数据加载或显示时由运行时本地化辅助逻辑解析。

## 占位符规则

所有语言必须保留同名占位符。如果 `en_GB` 包含 `Found {count} keys`，则 `zh_CN` 也必须包含 `{count}`。

## 存档内容

Sample 存档系统会记录：

- 玩家位置、数值、库存、装备和状态；
- 当前地图和场景相关状态；
- 已获得过的物品/装备 ID，用于让拾取详情消息只在首次获得时弹出；
- 新增角色、移动过的角色和已移除角色，例如打开的门和已拾取物；
- 改变过的地形；
- 已记录的楼层传送点；
- 截图和存档槽详情数据。

## 存档函数

自定义流程需要直接控制存档时，可在可视化事件中调用存档辅助函数：

```python
from Source.NodeFunctions.Save import GetSavePath, SaveGame, LoadGame

path = GetSavePath(1, "dat")
SaveGame(path)
result = LoadGame(path)
```

## 兼容建议

新增存档字段时，保持旧存档可读。缺失字段应有默认值，已移除角色或改变地形应继续通过稳定地图名和标签识别。
