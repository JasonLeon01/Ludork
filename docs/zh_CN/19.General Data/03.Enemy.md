# 03. 敌人

**Enemy** 标签页对应 `/Data/General/Enemy.dat`。关联 `EnemyInfo`，支持 `onDefeat` 蓝图事件。

## 字段

**name**、**desc**

怪物手册与战斗界面的本地化 key。

**MAXHP**、**ATK**、**DEF**

`Battler` 伤害计算使用的核心战斗属性。

**EXP**、**GOLD**

玩家战胜该敌人后获得的奖励。

**drops**

击败后掉落的物品 ID 列表。Sample 中均为空；条目为与 `/Data/General/Item.dat` 成员 key 一致的字符串。

**special**

将 **Special** 记录 ID 映射到可选数值字符串的字典。存在该键即启用对应特殊能力；需要数值参数时由战斗逻辑读取 value。

Sample 示例：

| 敌人 | special 条目 | 效果 |
|------|--------------|------|
| `Skeleton` | `Poisoning: "1"` | 启用与 `Poisoning` 相关的中毒行为。 |
| `Slime` | `Weaken: "2"` | 施加层数为 `2` 的虚弱。 |
| `Rock` | `Hard: ""` | 战斗中提高有效防御。 |
| `Mage` | `Magic: ""` | 攻击无视对方防御。 |
| `Knight` | `Compete: ""`、`MultiHit: "2"` | 攻击与对方 ATK 竞争；每回合攻击两次。 |

`MultiHit` 的 value 按整数解析为命中次数（空值时默认为 `1`）。

## 蓝图事件

**onDefeat**

敌人被击败后触发。可用于机关门、事件总线等机制，参见 `20.Usage Tips/03.Enemies` 与 `ConditionEnemies` 蓝图范例。

## 与地图蓝图关联

`/Data/Blueprints/Enemies/` 下的敌人蓝图将 `attrs.ID` 设为成员 key（如 `Slime`）。数值与文本来自通用数据；`texturePath`、`defaultRect` 留在蓝图中配置。

## 默认 Sample 成员

`Skeleton`、`Slime`、`Rock`、`Mage`、`Bat`、`Knight`。

新增敌人时请复制相近记录，设置唯一 ID 并调整数值与 `special`，再创建或复制使用相同 `ID` 的地图蓝图。
