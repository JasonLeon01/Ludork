# 10. 地图与角色

本章说明运行时地图和 actor 模型，不涉及地图数据的编辑方式。

## GameMap

`Global.GameMap` 表示已加载的地图。它拥有瓦片层、actor、灯光、粒子、摄像机、通行缓存和寻路工具。

常用方法：

- `getPlayer()`
- `setPlayer(player)`
- `getAllActors()`
- `getActorByTag(tag)`
- `getActorsByPosition(position)`
- `getActorsByRange(position, radius)`
- `spawnActor(actor, layer)`
- `destroyActor(actor)`
- `isPassable(actor, targetPosition)`
- `findPath(start, goal)`
- `getCamera()`
- `getTilemap()`
- `getSize()`
- `getTerrainTile(layerName, position)`
- `getTerrainTilePositions(layerName, tileID)`
- `setTerrainTile(layerName, position, tileID)`
- `setTerrainTiles(layerName, positions, tileID)`
- `destroyTerrain(layerName, position, tileID)`
- `destroyTerrainList(layerName, positions, tileID)`
- `addLight(light)`
- `setAmbientLight(color)`
- `addCommonTip(text)`

从模板地图场景获取当前地图：

```python
from Global import System
from Source.Scenes.SceneMap import Scene as SceneMap

scene = Cast(SceneMap, System.getScene())
gameMap = scene.getGameMap()
```

## 加载地图

模板地图场景从 `Data/Maps` 加载数据：

```python
scene.gotoMapAndPos("Map_01.dat", (3, 5), blockTransition=False)
```

`gotoMapAndPos()` 会在地图文件变化时重新加载地图，把地图状态写入 `GameInstance`，并在第三个参数不是 `True` 时播放地图 transition。

## Actor

`Engine.Gameplay.Actors.Actor` 是地图角色基类。

actor 子类常设置的类变量：

- `collisionEnabled`：是否阻挡移动。
- `tickable`：是否派发 tick 事件。
- `speed`：每秒移动像素。
- `texturePath`：角色贴图路径。
- `defaultRect`：默认贴图矩形。
- `defaultTranslation`、`defaultRotation`、`defaultScale`、`defaultOrigin`：默认变换。

基本移动方法：

- `MapMove(offset)`：按 `(1, 0)` 等偏移移动一格。
- `setRoute(routine)`：设置一串格子偏移作为自动路线。
- `getRoute()`
- `isMoving()`
- `isInRoute()`
- `stop()`
- `setMoveEnabled(enabled)`
- `getMoveEnabled()`

示例：

```python
from Engine import Vector2i

player.MapMove(Vector2i(1, 0))
player.setRoute([Vector2i(0, 1), Vector2i(1, 0)])
```

## Actor 事件

Actor 定义了蓝图式事件：

- `onCreate()`
- `onTick(deltaTime)`
- `onLateTick(deltaTime)`
- `onFixedTick(fixedDelta)`
- `onDestroy()`
- `onCollision(other)`
- `onOverlap(other)`

需要逐帧事件时，启用 `tickable`。

```python
class Chest(Actor):
    collisionEnabled = True
    tickable = False

    def onOverlap(self, other: list[Actor]) -> None:
        self.destroy()
```

## 碰撞与重叠

启用碰撞的 actor 尝试移动时：

1. 检查目标格是否越界。
2. 检查瓦片通行和四方向通行。
3. 检查目标格中启用碰撞的 actor。
4. 被 actor 阻挡时派发 `onCollision`。
5. 成功进入共享格子后派发 `onOverlap`。

`GameMap.getCollision(actor, targetPosition)` 和 `GameMap.getOverlaps(actor)` 可直接执行相关查询。

## 地形读写

当前 `GameMap` 支持读取和替换瓦片层地形：

```python
tile = gameMap.getTerrainTile("Ground", (6, 12))
gameMap.setTerrainTile("Ground", (6, 12), None)
```

`tileID` 可以是普通瓦片 ID、自动图块 key，或 `None`。`setTerrainTile()` 只改当前地图内存；`destroyTerrain()` 和 `destroyTerrainList()` 会把改动写入 `GameInstance`，之后存档会保留这些地形变化。

## 标签

地图数据可给 actor 分配 tag。tag 适合用于对话锚点和事件查找：

```python
npc = gameMap.getActorByTag("old_man")
if npc:
    npc.setVisible(False)
```

模板地图场景的 `showMessage(refActorTag, name, message)` 使用 tag 将消息窗口定位到 actor 附近。

## 灯光

`Global.Light` 保存：

- `position`
- `color`
- `radius`
- `intensity`

示例：

```python
from Global import Light
from Engine import Vector2f, Color

light = Light(Vector2f(160, 96), Color(255, 220, 160, 255), 180.0, 1.0)
gameMap.addLight(light)
```

使用 `setAmbientLight(color)` 设置地图基础光照。

## 寻路

`findPath(start, goal)` 返回地图位置列表。

```python
path = gameMap.findPath(player.getMapPosition(), targetPos)
```

如果要用于 actor 移动，需要把位置列表转换为步进偏移再传给路线。

## 摄像机

每个地图都有一个 `Camera`。默认情况下，调用 `setPlayer(player)` 时会把玩家设为摄像机父对象。

脚本需要当前视图位置或地图到屏幕关系时，可使用 `getCamera()`。

## 战斗与状态

模板项目的玩家和敌人使用 `Source.Battler` 混入战斗属性。核心属性放在 `infoComp` 组件中，例如 `HP`、`MAXHP`、`ATK`、`DEF`、`EXP`、`GOLD`，玩家还包含 `LEVEL` 和 `CLASS`。

状态由 `Source.Infos.StateInfo` 表示，并挂在 battler 上：

- `addState(stateID)`
- `removeState(stateID)`
- `hasState(stateID)`
- `getStateIDs()`
- `triggerStateWalk()`
- `triggerStateHook(stateID)`

`StateInfo` 的节点事件包含 `onWalk` 和 `onHookTriggered`。当前模板中，带有 `Poisoning` 特性的敌人在战斗后会给玩家添加 `Poisoned` 状态；玩家移动结束时会触发中毒状态的 `onWalk`。
