# 02. 门

门是魔塔地图中的通行障碍与机关载体。默认范例位于 `/Data/Blueprints/Doors/`；机关门范例位于 `/Data/Blueprints/Doors/ConditionDoors/`。

## 一触即开的门

`BP_Door_General` 为碰撞后即打开的普通门。若需要相同逻辑但不同外观，请从该蓝图派生新蓝图，并修改 `texturePath`（必要时一并调整 `defaultRect`）即可。

## 彩色钥匙门

黄、蓝、红钥匙门已分别提供 `BP_Door_Y`、`BP_Door_B`、`BP_Door_R` 等范例。它们派生自 `BP_Door_General`，通过 `needKeyID` 指定所需钥匙在 `/Data/General/Item.dat` 中的 ID。

若要新增例如绿门，可从 `BP_Door_Y` 派生，将 `needKeyID` 改为对应钥匙 ID，并更新纹理区域以匹配素材。

## 机关门

`/Data/Blueprints/Doors/ConditionDoors/` 中的门不依赖钥匙，而是通过事件总线协作开启。根据是否需要变量条件，从对应范例派生即可。

### 条件机关门

需要游戏变量达到目标值后才开启时，从 `BP_Door_ConditionExample` 派生，并修改以下属性：

- `eventBusVariableName`：关联的游戏变量名。
- `targetConditionValue`：开门所需的目标数值。

条件机关门统一使用 `ConditionalDoorAutoOpen` 作为“条件变量可能已变化”的通知事件总线 key。门在 `onCreate` 中注册该 key；事件总线触发后读取 `eventBusVariableName` 指定的变量，并与 `targetConditionValue` 比较，满足条件后执行开门逻辑。多个条件机关门可以共用同一个通知 key，只要各自的变量名不同即可。

为敌人或其他触发源连线时，先递增或写入同一个游戏变量，再调用 `TriggerEventBus("ConditionalDoorAutoOpen")`。具体节点连线可参考范例蓝图。

### 触发式机关门

只要事件总线被触发即开启、无需检查变量条件时，从 `BP_Door_TriggerExample` 派生，并修改以下属性：

- `eventBusName`：监听的事件总线名称。

门在 `onCreate` 中注册 `AutoOpen` 蓝图事件；任意逻辑调用 `TriggerEventBus` 投递同名 key 后，门会直接执行 `openDoor()`。具体节点连线可参考范例蓝图。
