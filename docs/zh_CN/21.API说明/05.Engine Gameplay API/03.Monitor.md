# 03. Monitor

源文件：`Sample/Engine/Utils/Monitor.py`

## 模块 `Engine.Utils.Monitor`

变量监控机制，拦截任意对象上的 `setattr` 操作，类似 React 的 `useEffect` 监听器。通过一次性修改类级别的 `__setattr__` 来检测属性写入；未注册监控的实例仅产生一次字典查找开销。

**函数**

#### `monitor(obj: object, name: str, callback: Callable[..., Any], params: List[Any] = []) -> None`

原始路径：`Engine.Utils.Monitor.monitor`

在任意对象上注册变量监控器。当通过属性赋值设置 `obj.name` 时，已注册的 `callback` 将以 `callback(oldValue, newValue, *params)` 形式调用，其中 `oldValue` 是赋值前属性的值，`newValue` 是被设置的新值。如果属性之前不存在，`oldValue` 将为哨兵值 `_MISSING`。

多个对象可独立监控，单个对象也可在不同属性名上注册监控器。每个 `(obj, name)` 对只允许注册一个回调；对同一 `(obj, name)` 再次调用 `monitor` 会覆盖之前的注册。

**参数**
- `obj` – 要监控属性的目标对象
- `name` – 要监听的属性名
- `callback` – 可调用对象，当属性被设置时以 `callback(oldValue, newValue, *params)` 形式调用
- `params` – 在两个必需值之后传入的位置参数列表（默认为空列表）

**警告：** 如果回调中再次设置同一对象上的被监控变量（如在 `health` 的监控器中执行 `obj.health = 0`），重入保护会阻止嵌套回调的执行，但值仍会被正常设置。请刻意避免此模式。

**执行输出：** `default`

**示例**
```python
from Engine.Utils.Monitor import monitor, _MISSING

def on_level_change(old, new):
    delta = new - (old if old is not _MISSING else 0)
    player.infoComp.HP += delta * 400
    player.infoComp.ATK += delta * 2
    player.infoComp.DEF += delta * 2

monitor(player.infoComp, "LEVEL", on_level_change, [])
player.infoComp.LEVEL = 2  # HP +400, ATK +2, DEF +2
```

---

#### `unmonitor(obj: object, name: str) -> None`

原始路径：`Engine.Utils.Monitor.unmonitor`

移除之前注册的变量监控器。如果给定的 `(obj, name)` 未注册监控器，则无任何效果。

**参数**
- `obj` – 目标对象
- `name` – 要停止监听的属性名

**执行输出：** `default`
