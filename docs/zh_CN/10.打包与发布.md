 # Ludork：2D RPG 工具包 - 帮助手册
 ## 打包与发布

 本章概述如何将 Ludork 项目打包成可供玩家直接运行的版本。底层构建过程由随 Ludork 提供的工具处理，作为创作者，你主要通过编辑器提供的打包界面来完成操作。

 ### 打包目标

 当你为项目打包时，一般希望：
 - 生成一个独立的应用或游戏目录，其中包含运行时引擎以及你的项目数据。
 - 包含所有必要的资源（Data、Assets、配置与本地化文件等）。
 - 在可能的情况下，不包含仅用于开发的文件。

 Ludork 的打包工具会根据项目内容，将这些组件打包在一起。

 ### 从编辑器发起打包

 编辑器通常会在菜单或工具栏中提供类似 **打包（Pack）** 或 **构建（Build）** 的入口。大致流程为：
 - 确保项目处于一致状态（地图、配置与公共函数等均已保存）。
 - 在编辑器中打开打包对话框或相关工具。
 - 在支持的平台上选择目标平台或打包模式。
 - 启动构建过程。

 在后台，打包过程会：
 - 收集项目的数据和资源。
 - 与运行时引擎一起打包。
 - 在指定的构建或发布目录中生成打包结果。

 具体可选项会随平台和版本有所差异，但对于大多数使用场景，你无需手动运行构建脚本即可完成常规打包。

 ### 打包结果与内容

 打包完成后，你通常会得到一个输出目录或应用，其中包含：
 - Ludork 运行时及其依赖的库。
 - 项目的 `Data/` 目录。
 - 项目的资源目录（图片、音频、着色器、字体等）。
 - 运行时所需的本地化与配置文件。

 你可以通过以下方式测试打包结果：
 - 直接运行生成的可执行文件或启动器。
 - 检查地图、音频和特效是否与编辑器中的预览一致。

 如果发现缺失内容，可以检查：
 - 资源是否存放在预期的文件夹中。
 - 配置字段是否指向正确的路径。

 ### 对打包结果的迭代

 打包通常并非一次性步骤。更常见的循环是：
 - 在编辑器中编辑内容，并通过内置预览反复测试。
 - 当项目达到某个稳定状态时，生成打包版本。
 - 在目标平台上测试该打包版本。
 - 返回编辑器修复问题或润色内容，然后再次打包。

 在整个过程中，保持项目结构清晰、资源分类合理，有助于打包过程顺利进行，并减少最终构建中的意外情况。
