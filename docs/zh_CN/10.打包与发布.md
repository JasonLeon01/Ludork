 # Ludork：2D RPG 工具包 - 帮助手册
 ## 打包与发布

 本章介绍如何将你的游戏项目打包成可分发的版本。Ludork 支持三个目标平台：**Windows**、**macOS (Apple Silicon)** 和 **iOS**。编辑器提供了内置的打包对话框，引导你完成整个流程。

 ### 打包目标

 当你为项目打包时，一般希望：
- 生成一个独立的可执行文件或应用，玩家无需安装编辑器即可运行你的游戏。
- 包含所有必要内容：资源（`Assets/`）、数据（`Data/`）、入口脚本与启动配置。
- 运行时代码被一并纳入分发，玩家无需额外安装 Python 或任何依赖。

 ### 从编辑器发起打包

 典型流程为：
 - 确保项目处于一致状态（地图、配置、公共函数、蓝图等均已保存）。
 - 打开 **打包（Pack）** 对话框，选择输出目录。
 - 在平台选择对话框中选择目标平台，然后开始构建。

 编辑器会根据当前操作系统显示可用的目标平台：
- **Windows** 上可选择：**Windows (win32)**。
- **macOS** 上可选择：**macOS (Apple Silicon)** 或 **iOS**。

 ### Windows 打包

 **前置条件**：
- Python 3.12.0 — 若未安装，编辑器会弹出提示并引导你前往 python.org 下载。
- MSVC 工具链 — 若缺失，编辑器会弹出提示并引导你前往 Visual C++ Build Tools 页面下载。
- Nuitka — 若未安装，编辑器会自动安装。

 **打包过程**：

 编辑器使用 Nuitka 将你的游戏编译为独立的 Windows 可执行文件。你的 `Assets/`、`Data/`、`Main.ini` 以及运行时引擎代码都会被自动包含在产出中。若 `Assets/System/icon.ico` 存在，它会被设置为应用图标。

 **产出**：输出目录中的 `Main.exe`（及其依赖文件），双击即可运行你的游戏。

 ### macOS (Apple Silicon) 打包

 **前置条件**：
- Python 3.12.0 — 若未安装，编辑器会弹出下载提示。
- Xcode 工具链 — 若缺失，编辑器会弹出提示并引导你前往 Xcode 下载页。
- Nuitka — 若未安装，编辑器会自动安装。

 **打包过程**：

 编辑器使用 Nuitka 将你的游戏编译为 macOS `.app` 应用包。`Assets/`、`Data/`、`Main.ini` 以及运行时引擎代码都会被自动包含。若 `Assets/System/icon.icns` 存在，它会被设置为应用图标。

 **产出**：输出目录中的 `Main.app`，双击即可运行你的游戏。

 ### iOS 打包

 iOS 打包与 Windows/macOS 不同——它不直接生成可运行的应用，而是生成一个 **Xcode 项目**，你需要在 Xcode 中完成最终编译与签名。

 **前置条件**：
- **仅限 macOS**：iOS 打包选项仅在 macOS 上可用。
- Xcode 工具链（包括 iOS SDK）— 若缺失，编辑器会弹出提示并引导你前往 Xcode 下载页。

 **打包过程**：

 编辑器会自动生成一个完整的 Xcode 项目，其中已经包含了你的游戏入口、运行时环境与原生扩展的引用。`Assets/System/` 下的图标文件（`icon.icns`、`icon.ico` 或 `icon.png`）会被自动转换为 iOS 应用图标。

 **产出**：`<项目目录>/build/<项目名>/`，包含：
- `<项目名>.xcodeproj/` — 可直接用 Xcode 打开的项目文件。
- `<项目名>/` — 应用源文件与图标。

 生成后，在 Xcode 中打开 `.xcodeproj`，连接 iOS 设备，即可编译并部署到设备上。

 **注意**：iOS 平台由于只能使用 OpenGL ES 1，无法使用 Shader，因此依赖着色器实现的美术效果（如转场特效、色调调整、后期滤镜等）在 iOS 上将不可用或表现与桌面端不同。建议在 iOS 上实际测试画面表现，并根据需要调整或替换这些效果。

 ### 打包结果与内容

 无论选择哪个平台，打包完成后的产物都包含：
- 项目资源：`Assets/`（图片、音频、着色器、字体等）。
- 项目数据：`Data/`（地图、图块集、配置、公共函数、本地化导出等）。
- 启动入口与启动参数：`Entry.py` 与 `Main.ini`（或其编译产物）。
- 运行时引擎代码（自动打入，无需手动处理）。

 你可以通过以下方式测试打包结果：
- **Windows**：直接运行输出目录中的 `Main.exe`。
- **macOS**：双击 `Main.app`。
- **iOS**：在 Xcode 中打开生成的 `.xcodeproj`，连接设备后运行。

 如果发现缺失内容，可以检查：
- `Main.ini` 的 `script` 是否指向存在的入口脚本（默认 `Entry.py`）。
- `Assets/` 与 `Data/` 是否齐全、且资源引用路径是否仍然有效（尽量保持相对路径与目录结构不变）。

 ### 应用图标

 Ludork 支持为每个平台自动设置应用图标。将图标文件放在项目的 `Assets/System/` 目录下：
- **Windows**：`icon.ico`
- **macOS**：`icon.icns`
- **iOS**：`icon.icns`、`icon.ico` 或 `icon.png`（按此优先级自动查找）

 若未提供图标文件，打包产物将使用默认图标。

 ### 对打包结果的迭代

 打包通常并非一次性步骤。更常见的循环是：
 - 在编辑器中编辑内容，并通过内置预览反复测试。
 - 当项目达到某个稳定状态时，生成打包版本。
 - 在目标平台上测试该打包版本。
 - 返回编辑器修复问题或润色内容，然后再次打包。

 在整个过程中，保持项目结构清晰、资源分类合理，有助于打包过程顺利进行，并减少最终构建中的意外情况。
