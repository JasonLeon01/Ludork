# 02. 编辑器基础

本章说明怎样用 Ludork 编辑器打开、创建、保存和理解项目结构。这里描述的是使用方式，不涉及编辑器自身的实现代码。

## 打开项目

启动 Ludork 后，可以在开始页或菜单中选择 `打开项目`，打开项目目录中的 `Main.proj`。`Main.proj` 用来标识一个可被编辑器打开的项目目录。

如果要从内置模板创建新项目，选择 `新建项目`，再选择目标目录并输入项目名。编辑器会复制一份模板项目作为新项目，之后所有编辑都发生在新项目目录中。

## 保存项目

编辑数据后使用 `保存` 写回项目。保存不仅会写入当前打开的数据，也可能同步运行时需要的配置文件。因此在测试游戏前，先保存一次。

如果测试游戏仍显示旧内容，先检查相关地图、蓝图、通用数据、配置和本地化表是否已经保存或导出。

## 典型项目目录

模板项目是一个可运行的 RPG 项目，也是一份新项目起点。典型结构如下：

```text
Project/
  Entry.py
  Main.ini
  Main.proj
  Assets/
  Data/
  Engine/
  Global/
  Source/
  Save/
  Temp/
```

常用目录含义：

- `Assets`：图片、音频、字体、shader、窗口皮肤、光标等资源。
- `Data`：地图、图块集、自动图块、蓝图、通用数据、公共函数和本地化数据。
- `Source`：项目自己的规则脚本，例如场景、窗口、玩家、道具、敌人和节点函数。
- `Engine` 与 `Global`：模板运行时和共享系统，普通项目通常先从编辑器数据和 `Source` 开始改。
- `Save`：运行时存档目录。
- `Temp`：运行时临时目录。

## 模板项目内容

当前模板以 32 像素瓦片为基础，默认游戏窗口内容尺寸为 `640 x 480`，启动地图是 `Map_01.dat`，玩家起点是 `(6, 12)`。

模板中可直接观察到的内容包括：

- 两张地图：`Map_01` 和 `Map_02`。
- 一套普通图块集：`Tileset_01`。
- 三个自动图块：`grass`、`lava` 和 `galaxy`。
- 多类蓝图：Actors、Consumables、Doors、Enemies、Equips、Items、Teleportations。
- 通用数据表：Class、Enemy、Equip、Item、Special、State。
- 公共函数：`PrintTest`。
- 两种本地化导出：`en_GB` 与 `zh_CN`。

## 测试入口

保存项目后，可以使用 `测试游戏` 启动当前项目。编辑器会运行项目入口脚本，加载当前项目的 `Main.ini`、`Data` 和 `Assets`。

测试时先确认游戏是否从期望地图和起点开始，再检查地图、角色、事件、音频、语言和窗口表现是否来自当前保存的数据。
