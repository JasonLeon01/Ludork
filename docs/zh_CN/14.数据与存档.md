# 14. 数据、本地化与存档

本章说明脚本如何访问项目数据、游戏本地化和存档。

## 数据位置

项目把运行时数据放在 `Data/`：

- `Data/Animations`
- `Data/AutoTiles`
- `Data/CommonFunctions`
- `Data/Configs`
- `Data/General`
- `Data/Locale`
- `Data/Maps`
- `Data/Tilesets`

脚本通常通过 `Source.Data` 或配置辅助函数访问这些数据，而不是直接打开文件。

## 加载项目数据

`Source.Data` 持有模板项目的运行时数据注册表。

加载函数：

- `loadAnimations()`
- `loadCommonFunctions()`
- `loadTilesets()`
- `loadAutoTiles()`
- `loadGeneralData()`

获取函数：

- `getAnimation(name)`
- `getTileset(name)`
- `getAutoTile(name)`
- `hasAutoTile(name)`
- `getGeneralData(name)`
- `getGeneralClassData(key)`
- `getGeneralEnemyData(key)`
- `getGeneralSpecialData(key)`
- `getGeneralStateData(key)`
- `getGeneralItemData(key)`
- `getGeneralEquipData(key)`
- `getClass(classPath)`
- `getClassData(classPath)`
- `getCommonFunction(name)`
- `genGraphFromData(data, parent=None, parentClass=None)`
- `genActorFromData(actorData, layerName)`

示例：

```python
from Source import Data

Data.loadTilesets()
tileset = Data.getTileset("Tileset_01")
```

初始场景通常负责加载项目数据。

## 通用数据

`Data/General` 包含物品、装备、敌人、状态、特性、职业等 RPG 记录。

使用：

```python
itemData = Data.getGeneralData("Item")
```

`Source.Item`、`Source.Equip`、`Source.Enemy`、`Source.Battler` 和 `Source.Infos` 中的项目代码会从这些数据构造玩法对象。`Special.dat` 用于敌人特性，例如当前模板中的 `Poisoning`；`State.dat` 用于构建 `StateInfo`，可挂到玩家或敌人身上。

## 配置数据

`Source.System.init()` 读取：

- `Data/Configs/System.json`
- `Data/Configs/Audio.json`

运行时 getter 暴露常用值：

- `getTitle()`
- `getFonts()`
- `getFontSize()`
- `getWindowskinName()`
- `getStartMap()`
- `getStartPos()`
- `getDecisionSE()` 等音效 getter
- `getTitleBGM()`

示例：

```python
from Source import System as GameSystem

startMap = GameSystem.getStartMap()
decisionSE = GameSystem.getDecisionSE()
```

## 本地化数据

`Entry.py` 初始化本地化：

```python
Engine.Locale.init("./Data/Locale")
```

使用：

```python
text = LOC("TITLE_START")
```

`Global.System.setLanguage(language)` 会应用并保存当前语言。`saveLanguage(language)` 只保存，不立即应用。

## 编辑本地化

项目的可编辑本地化源文件位于：

```text
Data/Locale/Locale.xlsx
```

导出的运行时语言文件与它放在同一目录：

```text
Data/Locale/en_GB
Data/Locale/zh_CN
```

表格格式如下：

- 第一行是表头。
- 第一列必须是 `ID`。
- 后续每一列是一个语言 key，例如 `zh_CN` 或 `en_GB`。
- 表头后每一行定义一个文本 ID。
- 可以用不同工作表整理文本，但 ID 应保持唯一且稳定。

示例：

| ID | zh_CN | en_GB |
| --- | --- | --- |
| TITLE_START | 开始游戏 | Start |
| TITLE_EXIT | 退出 | Exit |

修改 `Locale.xlsx` 后，需要导出，运行时语言文件才会更新。如果通过项目的本地化编辑器修改，保存表格时会同时导出运行时文件。如果用外部表格软件修改，请在测试或打包前执行本地化导出。

不同语言中的占位符必须一致：

| ID | zh_CN | en_GB |
| --- | --- | --- |
| GET_ITEM | 获得了 {name} | Got {name} |

不要在另一种语言中把 `{name}` 改成其他名字。运行时格式化要求占位符名称一致。

## 存档

模板项目的存档流程由 `Source.Save`、`Source.GameInstance` 和 `WindowSaveLoad` 管理。

`GameInstance` 保存：

- 当前区域和玩家列表。
- 游戏变量字典。
- 缓存地图文件。
- 被销毁 actor 记录。
- 持久化地形改动。
- 已记录楼层传送点。
- 存档缩略图截图。
- 玩家背包、装备、属性和状态 ID。

地图场景提供存档源：

```python
def _getSaveSource(self) -> GameInstance:
    return self.inst
```

读取存档后，地图场景调用 `applyLoadedGame(inst)` 重新绑定玩家与 UI，重新加载缓存地图，并把玩家放到保存的位置。

## 被销毁的 Actor

持久化被移除的地图 actor：

```python
scene.recordDestroyedActor(actor)
actor.destroy()
```

地图加载时，场景会检查已保存的销毁 actor 标签，并再次销毁匹配 actor。

## 地形改动

持久化地图地形变化时，使用地图或节点函数提供的销毁地形接口：

```python
gameMap.destroyTerrain("Ground", (6, 12), None)
gameMap.destroyTerrainList("Ground", [(6, 12), (7, 12)], None)
```

这些调用会修改当前地图，并通过 `GameInstance.recordTerrainDestruction()` 记录到存档。读取旧存档时，`GameInstance` 同时兼容字典格式和列表格式的地形记录。

## 楼层传送点

`GameInstance.recordTelepoint(mapPath, telepoint)` 保存地图上的可传送位置。`WindowFloorTeleporter` 会读取当前区域内已记录的传送点，显示地图列表和预览，并在确认后调用地图场景切换到目标地图与位置。

## 屏幕截图

地图场景打开菜单前会捕获当前画布，供存档缩略图使用。

使用：

- `Source.System.getSavedScreenImage()`
- `Source.System.setSavedScreenImage(image)`

## 直接文件工具

自定义文件可使用：

```python
from Engine.Utils import File

data = File.getJSONData("./Data/Configs/System.json")
obj = File.loadData("./Data/Maps/Map_01.dat")
File.saveData("./Save/custom.dat", obj)
```

修改存档结构时应保持兼容。优先新增可选字段，不要改变已有字段含义。
