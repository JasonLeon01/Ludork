# 15. 事件节点与公共流程

Ludork 默认模板的交互主要通过编辑器里的可视化事件完成。本章只说明使用方式。

## 蓝图事件

蓝图可维护创建、更新、碰撞、重叠、销毁等事件页。门、楼梯、测试对话对象和部分道具都使用事件图完成交互。

编辑时先选择事件页，再从节点列表添加动作、条件和数据节点，用连线决定执行顺序。保存后进入地图实际触发。

## 公共流程

`Data/CommonFunctions` 中的 `PrintTest` 是 Ludork 默认模板的公共流程示例。公共流程适合放多个蓝图都会复用的逻辑，例如统一提示、统一奖励或调试输出。

## 常用节点组

节点列表按用途分组，常见类别包括容器、数学、移动、魔塔玩法、玩家、存档、场景、状态、字符串、系统和工具。

选择节点时优先看显示名称和说明。参数要使用项目中真实存在的地图名、标签、物品 ID、装备 ID 或本地化文本键。

## 测试建议

每次只改一小段事件，保存后立刻测试。若事件没有触发，先检查蓝图是否放在地图上、标签是否正确、触发方式是否匹配、条件是否永远不成立。
