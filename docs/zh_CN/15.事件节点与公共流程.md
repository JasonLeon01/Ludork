# 15. 事件节点与公共流程

Sample 游戏使用可视化事件图维护蓝图行为和可复用公共流程。本页同时说明编辑器用法和这些图可以调用的运行时函数类别。

## 蓝图事件页

蓝图可以定义创建、更新、碰撞、重叠、移除、遭遇和击败等页面。具体页面取决于角色类和蓝图数据。

常见用途包括：

- 门检查钥匙或条件；
- 楼梯切换地图；
- 物品改变库存或地形；
- 敌人处理战斗结果；
- NPC 显示对话或选项。

## 公共流程

`Data/CommonFunctions` 保存可复用图。默认 Sample 工程包含 `PrintTest.dat`。

公共流程适合共享提示、通用奖励、重复地图逻辑，或需要从多个蓝图调用的调试流程。

## 蓝图装饰器

蓝图函数与类通过 `Sample/Engine/Decorators.py` 中的装饰器暴露给编辑器。`@Meta` 控制显示名与专用参数编辑器；`@ExecSplit`、`@Latent`、`@ReturnType` 决定执行流与数据引脚。详见 `19.蓝图装饰器`。

## 节点函数分组

**Container**

字典和列表操作。

**Math 与 String**

数值、向量、比较、布尔、随机和文本辅助函数。

**Movement**

移动路线和自动寻路。

**Player**

库存、装备、玩家数值、金钱、EXP 和玩家查找。

**Scene 与 Mota**

地图切换、地形编辑、商店、怪物手册、楼层传送和魔塔区域辅助函数。

**State**

战斗者状态、伤害、治疗和事件参数。

**System**

音频、视频、转场、闪屏、震屏和天气。

**Utils**

局部值、游戏变量、动画、公共函数调用、返回标题和打印输出。

## 调用公共函数

游戏代码可以通过工具节点层调用公共函数：

```python
from Source.NodeFunctions.Utils import RunCommonFunction

RunCommonFunction("PrintTest")
```

## 测试事件

一次只改一条事件路径。保存、启动测试游戏、触发地图上的对应角色；如果事件没有运行，检查控制台。
