# 09. 地图游玩流程

Sample 游戏的地图场景组合了地形、角色、输入、UI 窗口、光照、音频、雾、天气、存档和场景切换。

## 开始流程

新游戏进入 `Map_01.dat` 的 `(6, 12)`。先检查玩家移动、相机跟随、HUD 数值、地图音频和雾，再测试具体角色。

## 地图切换

上楼和下楼角色会在地图间传送玩家。它们会查找匹配的传送锚点，并在传送期间保持玩家移动状态安全。

## 楼层传送点

访问过的楼梯可以记录为楼层传送点。楼层传送窗口显示已知楼层和预览信息，然后把玩家送到选中的地图点。

魔塔相关节点辅助函数提供了窗口动作：

```python
from Source.NodeFunctions.Mota import OpenFloorTeleporter, OpenMonsterBook

OpenFloorTeleporter()
OpenMonsterBook()
```

## 地形变化

工具物品可以改变地形。Sample 游戏会记录这些变化，使其在保存和读取后仍然存在。

可视化事件通过场景节点函数使用常见地形操作：

```python
from Source.NodeFunctions.Scene import DestroyTerrain, GetTerrainTile

tile = GetTerrainTile("Ground", [6, 12])
DestroyTerrain("Ground", [6, 12], 0)
```

## 持久化地图角色

运行时地图角色变更可以被记录，用于局部持久化。进入地图时会先应用已记录的新增 Actor，再应用 Actor 位置变更，最后执行已记录的 Actor 销毁移除。这样运行时生成的 Actor 可以先还原，再移动到最后记录的位置；如果之后又被销毁，也会在最后被移除。

拾取过的物品和装备 ID 也会写入存档。对应的详情消息只会在首次获得该 ID 时弹出。

## 交互阻挡

对话、商店、物品、装备、存档、怪物手册和楼层传送窗口打开时会阻挡地图移动。添加自定义窗口时也应遵循同一规则：玩家在窗口中选择时阻挡地图输入，关闭后恢复移动。

## 光照与可读性

环境光、地图灯、角色灯、雾和 UI 窗口可能叠加。黑暗地图必须在 HUD 和阻挡窗口同时可见时测试。
