# 05. 物品

**Item** 标签页对应 `/Data/General/Item.dat`。关联 `ItemInfo`，支持 `onUse`、`onGet` 蓝图事件。

## 字段

**name**、**desc**

背包、商店与消息中的本地化 key。

**icon**

相对 `/Assets/` 的 UI 图标路径。

**usable**

是否可从物品菜单使用。钥匙类通常为 `false`；`EnemyBook`、`Teleport` 等工具设为 `true`。

**price**

商店买卖参考价格。

**cost**

为 `true` 时，使用会消耗背包中一个单位。钥匙等常保持 `true` 以便使用后扣除；按实际需求调整。

**kwargs**

可选的字符串键参数，用于物品专属行为。值通常为字符串，运行时可能被求值（例如 `"True"` → `true`）。

Sample 中已知 key：

| Key | 效果 |
|-----|------|
| `CanOnlyUseAsideTeleporter` | 在 `Teleport` 物品上为 true 时，仅当玩家站在 `Teleporter` 上或与其正交相邻一格时，才能打开楼层传送界面。 |

## 蓝图事件

**onUse**

从物品菜单使用时触发（须 `usable` 为 true）。

**onGet**

物品进入背包时触发。

可在此或读取物品 ID 的窗口代码中实现具体逻辑。

## 与地图蓝图及门关联

`/Data/Blueprints/Items/` 下物品蓝图将 `attrs.ID` 设为成员 key（如 `KEY_Y`）。

彩色门通过门蓝图上的 `needKeyID` 引用相同 ID。修改钥匙 ID 时须同步更新所有引用它的门与蓝图。

## 默认 Sample 成员

| ID | 用途 |
|----|------|
| `KEY_Y`、`KEY_B`、`KEY_R` | 黄、蓝、红钥匙。 |
| `EnemyBook` | 打开怪物手册。 |
| `Teleport` | 打开楼层传送界面。 |
| `BreakWall`、`ClearWall`、`BreakLava`、`BreakIce` | 地形类道具。 |

请先添加通用数据成员，再创建对应物品蓝图或商店条目。
