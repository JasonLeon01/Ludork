# 11. UI 与窗口

运行时 UI 由可绘制控件组成，并加载到场景的 UI 管理器中。

## 加载 UI

在 `SceneBase.onCreate()` 中创建控件，然后加载到 `self._uiManager`。

```python
from Engine.UI import Image
from Global import Manager

self._bg = Image(Manager.loadSystem("GrassBackground.png"))
self._uiManager.loadUI(self._bg)
```

控件会由场景循环自动更新和绘制。

## 坐标系统

UI 位置使用游戏逻辑坐标。`Engine.Scale` 会在渲染时内部应用。

默认 `640 x 480` 游戏中，中心点是 `(320, 240)`。

## Image

`Engine.UI.Image` 显示一张贴图。

```python
from Engine.UI import Image
from Global import Manager

logo = Image(Manager.loadSystem("TitleLogo.png"))
logo.setPosition((120, 48))
self._uiManager.loadUI(logo)
```

## PlainText

`PlainText(font, text, characterSize=None, style=Text.Style.Regular, fillColor=Color.White)` 渲染单一样式文本。

重要方法：

- `setString(text)`
- `getString()`
- `setCharacterSize(size)`
- `getCharacterSize()`
- `setColor(color)`
- `getSize()`
- `getLocalBounds()`
- `getAbsoluteBounds()`

示例：

```python
from Engine.UI import PlainText
from Source import System as GameSystem
from Engine import Color

text = PlainText(GameSystem.getFonts()[0], "Hello", 24, fillColor=Color.White)
text.setPosition((32, 32))
self._uiManager.loadUI(text)
```

## RichText

`RichText(font, text, styleCollection)` 渲染多样式分段文本。

标记使用 `#name#` 表示命名样式，也支持可解析的十六进制颜色标记。

```python
from Engine.UI import RichText, TextStyle
from Engine import Color, Text

styles = {
    "default": TextStyle(characterSize=22, fillColor=Color.White),
    "red": TextStyle(fillColor=Color.Red, style=Text.Style.Bold),
}
message = RichText(font, "You got #red#Potion#default#!", styles)
```

## Window

`Window(rect, windowSkin, repeated=False)` 渲染 windowskin 窗口框。

`rect` 可以写成 `((x, y), (width, height))`。

```python
from Engine.UI import Window
from Global import Manager

skin = Manager.loadSystem(GameSystem.getWindowskinName()).copyToImage()
window = Window(((32, 32), (240, 96)), skin)
self._uiManager.loadUI(window)
```

## 指令窗口

模板项目的 `WindowCommand` 使用字典构建菜单指令：

```python
self._windowCommand = WindowCommand(
    ((0, 0), (320, 96)),
    {
        "Start": {"text": LOC("TITLE_START"), "callback": Scene._startGame},
        "Exit": {"text": LOC("TITLE_EXIT"), "callback": Scene._exitGame},
    },
    columns=2,
)
```

回调会接收被选择对象和关键字数据。

## 消息窗口

模板项目的 `SceneMap` 提供对话辅助方法：

```python
self.showMessage("npc_01", "Guide", "Welcome.")
```

`showMessage(refActorTag, name, message)` 在对话期间禁用玩家移动，并返回 latent 条件。

`showSelection(refActorTag, name, options, allowCancel)` 显示选项并返回选择索引，取消时返回 `-1`。

```python
choice = self.showSelection("npc_01", "Guide", ["Yes", "No"], True)
```

## 菜单与存档窗口

模板地图场景创建了：

- `WindowMenu`
- `WindowItem`
- `WindowEquipSlot`
- `WindowEquipSelect`
- `WindowSaveLoad`
- `WindowShop`
- `WindowEnemyBook`
- `WindowFloorTeleporter`
- `PlayerAttrHUD`

这些是 `Source.Windows` 中的项目级窗口。新增 RPG 窗口时可参考它们。

## 配置窗口与通用控件

标题场景使用 `ConfigWindow` 管理运行时设置。它基于 `WindowSelectable` 和 `ListView`，每行由标签和控件组成：

- `DropBox`：语言、缩放、帧率。
- `CheckBox`：垂直同步、音乐/音效/语音开关。
- `Slider`：音乐/音效/语音音量。

语言和缩放使用 `saveLanguage()`、`saveScale()` 写入 `Main.ini` 并提示重启；帧率、垂直同步和音频开关/音量会通过 `Global.System` 立即应用。

## 激活与阻塞

模板菜单窗口提供 `isBlocking()`、`isOpen()` 等状态。地图场景也会检查消息、商店、怪物手册和楼层传送器是否可见，避免地图移动、对话输入和菜单输入互相冲突。

新增窗口时也应遵循同样规则：可见的模态窗口或菜单窗口应消费输入，并报告自己会阻塞地图操作。
