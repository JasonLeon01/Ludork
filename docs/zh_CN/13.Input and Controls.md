# 13. 输入与操作

Sample 运行时通过共享输入层统一键盘、鼠标、触摸和手柄输入。

## 默认操作

**键盘**

方向键移动玩家或菜单选择。`Enter` 和 `Space` 确认。`Escape` 取消、关闭窗口或打开游戏内菜单。

**鼠标**

左键尝试地图寻路或确认 UI 选择。右键取消、关闭窗口或打开游戏内菜单。

**触摸**

单指点击尝试地图寻路。双指触摸相当于右键取消。

**手柄**

方向轴用于导航或移动。确认和取消按钮遵循运行时输入映射。

## 动作辅助函数

游戏代码应使用动作辅助函数，而不是只检查一个物理按键。

```python
from Engine import Input

if Input.isActionTriggered(Input.getConfirmKeys(), handled=False):
    confirm()
    Input.isActionTriggered(Input.getConfirmKeys(), handled=True)

if Input.isActionTriggered(Input.getCancelKeys(), handled=False):
    cancel()
    Input.isActionTriggered(Input.getCancelKeys(), handled=True)
```

先用 `handled=False` 检测输入。只有动作确实被当前逻辑处理后，才再次调用并传入 `handled=True` 标记消费。这样可以避免某个窗口只因为检测输入就提前吃掉按键。

## 地图点击移动

鼠标左键和触摸可以请求自动寻路。阻挡窗口打开时，地图场景会消费这些输入，避免一次 UI 点击同时移动玩家。

自动寻路会避开没有开启碰撞但实现了 `onOverlap` 的 Actor；手动移动仍然可以踩上去触发交互。

## 输入模式

输入层会记录当前活跃设备是鼠标还是手柄。UI 控件可以据此决定显示鼠标悬停行为还是手柄选择行为。

## 自定义输入建议

新增玩法输入时，请通过 `Engine.Input` 处理，并在动作执行后标记 handled。未执行动作时不要标记。这样菜单、窗口、点击移动和触摸取消不会互相抢输入。
