# 06. 数据、本地化与测试

本页说明编辑器侧的数据维护。它从项目作者角度解释如何使用数据工具以及如何测试结果。

## 本地化表

游戏本地化通过官方本地化工具插件 `Plugins/OfficialLocaleTools` 维护。该插件不属于引擎核心，而是作为参考插件随编辑器分发，提供数据库菜单命令、编辑器内本地化表和导出工具。

**打开表格**

使用数据库菜单中的本地化表命令，或直接打开 `Data/Locale/Locale.xlsx` 进行表格编辑。

**导出**

编辑后运行数据库菜单中的导出命令，刷新 `Data/Locale` 下的语言文件。测试游戏读取的是导出的文件，而不只是工作簿或插件视图。

**Key**

名称、描述、菜单文本和提示应使用稳定 key。

**语言**

每个用户可见 key 都应填写 `zh_CN` 和 `en_GB`。

**占位符**

如果一种语言包含 `{name}` 这样的占位符，所有语言都必须包含同名占位符。

插件结构、菜单 hook 以及 `{LOCALE_KEY}` 灰色提示预览，见 `23.Plugin System`。

## 动画数据

按 `F7` 打开 `动画总览`：左侧列出工程中的动画，右侧内嵌编辑器。在文件浏览器中双击动画文件仍会打开独立的动画窗口。

先在预览中检查图片、声音、时机、缩放、水平翻转（`flipX`）和位置，再在测试游戏中验证动画触发效果。

## 基本测试流程

1. 保存所有编辑过的面板。
2. 文本变化后导出本地化。
3. 启动 `测试游戏`。
4. 开始新游戏或加载相关存档。
5. 触发被修改的地图、角色、物品、窗口或设置。
6. 检查文本或缩放时，切换语言并重新启动测试游戏。
