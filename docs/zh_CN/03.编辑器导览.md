 # Ludork：2D RPG 工具包 - 帮助手册
 ## 编辑器导览

 本章将从使用者的角度，带你快速浏览 Ludork 编辑器的界面结构。这里不会穷举每一个选项，而是帮助你建立一个清晰的心智模型：各个区域大致在哪里、分别负责什么工作、以及如何在工作区中移动与操作。

 ### 主窗口布局

 当你在打开项目的情况下启动 Ludork 编辑器时，会看到一个由三大区域构成的主窗口：
 - 带有菜单、模式切换和全局操作的**顶部栏（Top Bar）**。
 - 显示当前地图或主要内容的**中央工作区（Central Workspace）**。
 - 用于项目浏览、工具和调试的**侧边与底部面板（Side and Bottom Panels）**。

 具体布局会因你打开哪些面板而有所不同，但整体思路始终一致：中间用于编辑，四周提供上下文和工具。

 ### 顶部栏与模式（Top Bar and Modes）

 顶部栏决定了你如何与当前项目交互：
 - **文件与项目操作**：用于打开、保存以及切换项目。
 - **编辑操作**：撤销、重做和剪贴板等常用指令。
 - **模式切换（Mode toggles）**：在不同编辑模式之间切换，例如地图编辑模式与运行时预览模式。
 - **设置与配置入口**：打开各种配置窗口，调整项目或编辑器行为。

 模式切换尤其关键：
 - 在**编辑模式**下，中央工作区表现为地图编辑器，你可以绘制图块、选择区域并放置对象。
 - 在**预览或运行模式**下，工作区则更像游戏运行时的呈现方式，便于从玩家视角观察效果。

 你可以把顶部栏理解为整个编辑会话的“控制面板”。

 ### 中央工作区（Central Workspace）

 中央工作区是你最常停留的地方。在默认布局下，它通常包括：
 - 显示当前正在编辑的地图或内容的**编辑视图（Editor View）**。
 - 用于在大地图上平滑移动的**可滚动区域（Scrollable Area）**。
 - 用于嵌入预览、贴近运行时表现的**游戏视图（Game View）**。

 在该区域，你通常会：
 - 缩放与平移地图或场景。
 - 选择图层或切换不同编辑工具。
 - 放置、擦除或修改图块和其他对象。

 工作区的设计尽量保持简洁，确保地图与场景本身始终是视觉焦点。

 ### 项目与文件面板（Project and File Panels）

 在主窗口的一侧，你通常会看到面向项目的面板，例如：
 - 显示项目文件（地图、图块集、公共函数、配置文件及其他资源）的**文件浏览器（File Explorer）**。
 - 用于快速查看当前选中文件内容的**预览面板（Preview Panel）**。

 这些面板中的常见操作包括：
 - 在项目中定位到某个地图或图块集，并在中央工作区中将其打开。
 - 选择某个配置文件以编辑项目级设置。
 - 快速预览图片、音频或着色器资源，而无需离开编辑器。

 得益于这些面板，你可以在专注于单个资源的同时，随时了解整个项目的大致结构。

 ### 地图与图块集工具（Map and Tileset Tools）

 地图编辑体验围绕两个紧密相关的区域展开：
 - 选择绘制用图块的**图块集区域（Tileset Area）**。
 - 使用这些图块搭建场景的**地图区域（Map Area）**。

 典型的交互包括：
 - 从项目中选择一个图块集，并在专用的图块集面板中查看。
 - 选中单个图块或一块图块区域，用作笔刷。
 - 在地图上绘制、擦除或填充图块。
 - 在不同图层之间切换，例如地面层、装饰层和顶层等。

 对应的图块集编辑器还提供更聚焦的工具，用于：
 - 调整图块集属性，例如图块尺寸与通行规则。
 - 检视源图像中图块的排列方式。

 通过这些工具，你可以在“编辑图块集内容”和“用图块集搭建地图”这两种视角之间快速切换。

 ### 公共函数与节点图（Common Functions and Node Graphs）

 在可视化脚本部分，编辑器提供专门针对**公共函数（Common Functions）**的窗口和面板：
 - 展示项目中所有可用可视化脚本的**公共函数列表（Common Function List）**。
 - 用于摆放和连接节点、定义逻辑的**节点图画布（Node Graph Canvas）**。
 - 帮助管理输入、输出和参数的配套面板。

 从“编辑器导览”的角度看，最重要的是：
 - 你可以从项目列表中打开某个公共函数，从而弹出节点图编辑窗口。
 - 节点图画布的交互方式类似常见的节点编辑器：拖拽节点到视图中，通过连线连接节点，并根据需要调整布局。
 - 你在这里做出的更改会被保存到项目数据中，并可被游戏其他部分复用。

 可以把这些窗口视作 Ludork 的“逻辑编辑器”，与地图编辑器和图块集编辑器并列存在。

 ### 配置与设置窗口（Configuration and Settings Windows）

 除了内容编辑器，Ludork 还提供用于配置全局与项目级设置的窗口，包括：
 - **系统配置（System Configuration）**：设置游戏标题、分辨率、字体、着色器以及其他全局选项。
 - **项目设置（Project Settings）**：调整路径、默认资源或编辑器行为。
 - 如光照配置等**专项面板**，用于微调场景中的光照表现。

 这些窗口通常以对话框或附加窗口形式出现在主编辑器之上。它们的使用频率可能不如地图和图块工具高，但对游戏整体观感和体验的影响非常显著。

 ### 控制台与调试面板（Console and Debug Panels）

 为了帮助你了解项目在背后发生了什么，编辑器集成了：
 - 显示日志输出、警告和错误信息的**控制台面板（Console Panel）**。
 - 用于短暂提示关键信息（例如保存成功或错误）的**Toast 通知**。

 在日常使用中，你可能会：
 - 在运行预览时关注控制台，以便发现缺失资源或配置问题。
 - 当某张地图或公共函数的行为与预期不符时，通过日志来追踪问题。

 这些面板本质上是诊断工具，但对于在不离开编辑器的前提下定位问题非常重要。

 ### 布局与个性化（Layout and Customization）

 编辑器采用现代的、受 Material 风格启发的主题，并支持：
 - 可调整大小的面板和分割条，使你可以为自己关心的区域预留更多空间。
 - 布局持久化，记住你偏好的窗口与面板排列方式。

 随着使用时间的增长，你可以逐步个性化：
 - 默认显示哪些面板。
 - 中央工作区与项目导航、调试面板之间的空间分配。

 目标是让界面适应你的工作流——无论你主要在地图、可视化脚本还是配置中花费时间。

 ### 键盘与鼠标习惯（Keyboard and Mouse Conventions）

 虽然具体快捷键可能会随版本有所变化，但编辑器整体遵循常见桌面软件的习惯：
 - **Ctrl/Cmd + Z / Y**：撤销与重做。
 - **Ctrl/Cmd + C / V**：在支持的列表和编辑器中进行复制与粘贴。
 - **鼠标拖拽（Click-and-drag）**：用于选择图块或在节点图中移动视图/节点。
 - **鼠标滚轮**或专用控制：在支持的视图中进行缩放。

 你通常可以通过菜单条目和工具提示发现具体快捷键。本章主要聚焦界面布局，后续针对特定工具的章节中，必要时会单独强调某些特别高频或有用的快捷键。
