# 15. 节点函数与事件

Ludork 可以在运行时执行节点图数据。模板项目使用节点函数实现类似事件的玩法逻辑和公共函数。

本章只说明脚本侧内容。

## 公共函数

`Source.Data.getCommonFunction(name)` 从 `Data/CommonFunctions` 加载图。

它会查找：

- `Data/CommonFunctions/<name>.dat`
- `Data/CommonFunctions/<name>.json`

示例：

```python
graph = Data.getCommonFunction("PrintTest")
```

`Data.genGraphFromData(data, parent=None, parentClass=None)` 会从已加载数据构造图对象。

## Actor 图

从地图数据生成的 actor 会获得一张图：

```python
actor.setGraph(Data.genGraphFromData(classData["graph"], actor, classModel))
```

随后 actor 事件可通过注册事件名派发到这张图。

## RegisterEvent

使用 `@RegisterEvent` 标记 actor 蓝图事件方法。

```python
from Engine.Gameplay.Actors import Actor

class Door(Actor):
    @RegisterEvent
    def onOverlap(self, other: list[Actor]) -> None:
        pass
```

内置 actor 事件：

- `onCreate`
- `onTick`
- `onLateTick`
- `onFixedTick`
- `onDestroy`
- `onCollision`
- `onOverlap`

状态数据 `Source.Infos.StateInfo` 也使用注册事件，当前提供：

- `onWalk`
- `onHookTriggered`

## ExecSplit

`@ExecSplit(...)` 描述节点逻辑的执行分支。

actor 移动中的例子：

```python
@ExecSplit(success=(True,), fail=(False,))
def MapMove(self, offset) -> bool:
    ...
```

返回值会选择对应分支。

## Latent

`@Latent(...)` 标记返回条件函数的方法。节点图可以等待该条件返回完成值。

基本形态：

```python
@Latent(TimeUp=(True,))
def waitSomething(self) -> Callable[[], bool]:
    def condition() -> bool:
        return ready
    return condition
```

场景定时器、对话和选项都使用这种方式。

## ReturnType

`@ReturnType(name=Type)` 记录命名返回值，供工具和节点执行使用。

```python
@ReturnType(actors=list[Actor])
def getAllActors(self) -> list[Actor]:
    ...
```

## TypeAdapter

`@TypeAdapter` 在函数体执行前适配节点或脚本传入值。

示例：

```python
@TypeAdapter(position=(tuple, Vector2f))
def setLightPosition(self, light, position):
    ...
```

这样节点数据或脚本调用就可以传入简单 tuple。

## Source.NodeFunctions

项目节点函数位于 `Source/NodeFunctions`。

模板项目包含如下分组：

- `Math`
- `String`
- `Container`
- `Utils`
- `System`
- `Scene`
- `Player`
- `State`
- `Save`
- `Movement`
- `Mota`

其中当前模板常用分组职责如下：

- `System`：音效、音乐、视频、转场、闪屏、震动、天气和滤镜。
- `Scene`：切换地图、Game Over、记录传送点、读写/持久化地形、打开商店。
- `Player`：物品、装备、属性、HP、金币、经验和玩家查询。
- `State`：状态宿主、事件参数、任意 battler 属性和跨对象状态增删。
- `Movement`：actor 路线和寻路移动。
- `Mota`：怪物手册、楼层传送器和当前区域。

新增可由节点调用的玩法函数时，可参考这些文件。

## 事件安全

节点函数可能由图执行调用，而不是直接由场景方法调用。应保持小而明确：

- 验证重要输入。
- 优先使用运行时 API，不直接改私有状态。
- 用 `System.setScene()` 切换场景。
- 用 `SceneMap.showMessage()` 和 `showSelection()` 处理阻塞对话。
- 用 `GameMap` 方法查找和生成 actor。

## 示例：适合节点使用的函数

```python
from Engine import ReturnType
from Global import System
from Source.Scenes.SceneMap import Scene as SceneMap

@ReturnType(scene=SceneMap)
def getMapScene() -> SceneMap:
    scene = System.getScene()
    assert isinstance(scene, SceneMap)
    return scene
```

返回值名称应清晰，因为它会出现在节点和文档元数据中。
