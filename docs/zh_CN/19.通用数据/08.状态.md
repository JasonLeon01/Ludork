# 08. 状态

**State** 标签页对应 `/Data/General/State.dat`。关联 `StateInfo`，支持 `onWalk`、`onHookTriggered` 蓝图事件。

## 字段

**name**、**desc**

界面与提示的本地化 key。

**icon**

可选，相对 `/Assets/` 的状态图标路径。

**stackable**

为 `true` 时，再次施加同一状态会叠加层数而非替换。Sample 中 `Poisoned` 可叠加，`Weak` 不可。

## 蓝图事件

**onWalk**

宿主战斗者每走一步时调用。

**onHookTriggered**

游戏代码显式触发状态钩子时调用。

战斗伤害已不再经过状态蓝图事件；上述事件主要用于地图行走或自定义脚本逻辑。

## 运行时施加

游戏与节点代码通过 `Battler.addState` 按 ID 施加状态。运行时从对应通用数据成员构建 `StateInfo`，并在 `stackable` 为 true 时维护层数。

## 默认 Sample 成员

| ID | 说明 |
|----|------|
| `Poisoned` | 可叠加；战斗中按层数增加受到伤害。 |
| `Weak` | 生效期间有效 ATK 降低 `2 × 层数`。 |

## 使用说明

- 新增状态时请复制现有记录，保留 `stackable` 与图槽结构。
- 节点函数或 Python 中使用的状态 ID 须与本页成员 key 一致。
- 图标路径规则与物品、装备相同，均相对 `/Assets/`。
