# 01. 概览

通用数据（General Data）存放可复用的 RPG 记录：职业、敌人、装备、物品、战斗特殊能力与状态。蓝图通过 `ID` 引用这些记录；名称、数值、图标与事件逻辑在运行时从通用数据解析。

## 打开编辑器

从 **数据库** 菜单打开 `通用数据`，或按 `F10`。每个标签页对应 `/Data/General/` 下的一个文件（例如 `Enemy.dat`）。

## 文件结构

每个 `.dat` 文件包含：

- **params**：该类型下所有成员共用的字段定义（名称、类型、默认值）。
- **members**：以 key 为 `ID` 的一条条记录。
- **linkedType**（可选）：将标签页关联到 `Source/Infos/*.py` 中的类，使成员可挂载蓝图事件图。

成员记录中还可能包含 `_graph`，用于保存成员级事件节点图。该数据通过编辑器中的 **编辑蓝图** 维护，不会作为普通属性字段显示。

## 编辑器布局

**左侧面板**

- 列出当前标签页下的所有成员 ID。
- `+` 添加成员；右键成员可更改 ID、删除，或在已设置关联类型时打开蓝图图。

**右侧面板**

- 显示当前选中成员的字段。
- `ID` 为只读；通过成员右键菜单重命名。
- 底部 `+` 为整个类型添加新字段（会同步到所有成员）。
- 字段旁的 `-` 从类型定义及所有成员中删除该字段。

**标签栏**

- 右键标签可新建数据类型、重命名、设置或清除关联类型、删除该类型。

## 字段类型

| 类型 | 编辑控件 | 说明 |
|------|----------|------|
| `string` | 单行文本 | 常存放 `{KEY_Y}` 这类本地化 key。 |
| `int` | 整数输入框 | 战斗属性、价格、计数等。 |
| `float` | 浮点输入框 | 小数值。 |
| `bool` | 勾选框 | 如 `usable`、`stackable` 等标志。 |
| `file` | 浏览按钮 | 相对 `/Assets/` 的路径。 |
| `list` | 列表行编辑器 | 有序字符串列表，如掉落物 ID。 |
| `dict` | 键值行编辑器 | 键与值均为字符串；战斗逻辑可能对值求值。 |

添加字段时请设置新成员的默认值。字段名应与游戏代码读取的属性名一致（例如 `icon`、`price`、`attrPlus`）。

## 关联类型与蓝图事件

关联类型（如 `EnemyInfo`、`ItemInfo`）将标签页连接到 `Source/Infos/` 下的类。编辑器会扫描该类上的 `@RegisterEvent` 方法，并为每个成员提供 **编辑蓝图**。

事件图保存在成员的 `_graph` 中，用于数据层逻辑（`onUse`、`onDefeat`、`onWalk` 等），无需重复编写地图 Actor 蓝图。

## 与地图蓝图关联

敌人、装备、物品的地图蓝图 `attrs.ID` 须与通用数据成员 key 一致。蓝图负责地图表现（纹理、碰撞、摆放）；通用数据负责数值、文本、图标与 Info 层事件。

请先建立通用数据记录，再在蓝图中使用相同 `ID`。

## 本地化

`name`、`desc` 通常填写 `{大括号}` 包裹的本地化 key，而非直接写显示文本。请在本地化表中补全对应 key 并导出后再测试。

## 保存与测试

编辑通用数据后使用 **保存**。启动 **测试游戏**，通过战斗、背包、消耗钥匙的门等路径触发相关记录以验证改动。

## 默认 Sample 标签页

Sample 工程预置六个标签页：

- `Class` — 玩家职业与装备槽。
- `Enemy` — 敌人数值、掉落与战斗特殊能力。
- `Equip` — 可穿戴装备与属性加成。
- `Item` — 钥匙、道具与可使用物品。
- `Special` — 可复用的战斗能力定义。
- `State` — 战斗者状态效果。
