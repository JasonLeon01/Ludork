# 01. 介绍

Ludork 是一套 2D RPG 工具链，由编辑器、默认 Sample 游戏和少量运行时框架组成。本手册参考经典帮助文件的写法：每页先说明功能用途，再列出相关命令、数据、属性与检查方法。

## 手册分区

**编辑器手册**

`02` 到 `06` 章说明编辑器的使用方法。这些章节不展开运行时代码，只讲可见工作流：打开工程、编辑数据、选择素材、保存与测试。

**Sample 游戏参考**

`07` 到 `17` 章说明默认 Sample 游戏。这些章节可以提到数据文件、运行时行为、节点函数，以及有助于解释游戏侧用法的少量代码示例。

**蓝图脚本参考**

`18` 章说明 `@Meta`、`@ExecSplit`、`@Latent` 等装饰器及参数默认值约定；`19` 章说明通用数据编辑；`20` 章说明默认 Sample 蓝图的常用制作技巧；`21` 章列出蓝图可见 API 签名。

## Ludork 默认工程

默认工程是一份魔塔风格 RPG 模板，包含地图、自动图块、玩家蓝图、敌人、门、钥匙、消耗品、装备、怪物手册、楼层传送、商店、存档、设置、天气、雾、音效和标题音乐。

## 默认值

**工程标记**

`Main.proj` 用于标记一个文件夹是 Ludork 工程。

**启动脚本**

`Main.ini` 在 Sample 工程中指向 `Entry.py`。

**语言**

Sample 工程包含 `zh_CN` 与 `en_GB`。英文文档和英文语言 key 使用 `en_GB`。

**显示**

默认内容尺寸为 `640 x 480`，图块尺寸为 `32`，默认缩放为 `2.0`。

**新游戏位置**

新游戏从 `Map_01.dat` 的图块坐标 `(6, 12)` 开始。

## 阅读方式

如果你在制作内容，先读编辑器章节，并遵循“保存后测试”的节奏。如果你在修改 Sample 游戏行为，则使用后面的参考章节寻找对应数据或运行时入口。

当页面要求验证改动时，指的是通过 `测试游戏` 运行真实工程，而不是只确认编辑器字段已经改变。
