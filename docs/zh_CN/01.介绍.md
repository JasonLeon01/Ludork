# 01. 介绍

本手册说明 Ludork 游戏运行时以及项目脚本的用法，面向需要修改游戏逻辑、场景、角色、窗口、数据读取、音频和画面效果的项目作者。

某个功能如果既可以由数据完成，也可以由脚本完成，本手册只说明脚本侧的写法。

## 阅读方式

本手册采用偏参考手册的写法：

- 概念章节说明功能属于哪一层。
- 类与方法章节列出常用接口和用途。
- 示例章节展示 Sample 项目中的常见脚本形态。
- 注意事项说明运行时约定。

多数示例默认当前工作目录是项目根目录，例如 `Sample/`。

## 运行时总览

Ludork 游戏项目是一个 Python 项目目录。Sample 项目结构如下：

```text
Sample/
  Entry.py
  Main.ini
  Assets/
  Data/
  Engine/
  Global/
  Source/
  Save/
  Temp/
```

`Entry.py` 是游戏进程入口。它加载本地化数据，初始化节点图 latent 系统，初始化 `Global.System`，创建第一个场景，然后持续运行当前场景直到游戏退出。

`Engine` 包含底层类型、输入、UI 控件、装饰器、文件工具、节点图执行、玩法基础对象，以及 SFML/PySF 导出内容。

`Global` 包含窗口、场景栈、摄像机、地图、天气、雾、资源管理器、视频播放和场景基类等共享系统。

`Source` 是 Sample RPG 的游戏规则层，包含标题场景、地图场景、玩家、战斗对象、物品装备、存档、窗口、数据访问和节点函数。

## 主要包

### Engine

需要核心对象和工具时导入 `Engine`：

```python
import Engine
from Engine import Vector2u, Vector2f, Color, Direction
from Engine import Input
```

重要全局值：

- `Engine.CellSize`：瓦片像素尺寸，Sample 使用 `32`。
- `Engine.GameSize`：游戏逻辑分辨率。
- `Engine.Scale`：显示缩放。
- `Engine.GameRunning`：主循环标记。

### Global

需要游戏全局系统时导入 `Global`：

```python
from Global import System, SceneBase, GameMap, Camera
from Global import Manager
```

`Global.System` 负责窗口状态、场景切换、转场、屏幕效果、语言、帧率和音频设置。

### Source

需要 Sample 项目的游戏规则时导入 `Source`：

```python
from Source import Data
from Source.GameInstance import GameInstance
from Source.Scenes.SceneMap import Scene as SceneMap
```

`Source` 中的脚本通常是每个游戏会修改的部分。`Engine` 与 `Global` 更接近运行时基础。

## 脚本约定

Ludork 脚本使用 Python 3.12 语法和类型标注。项目中方法名通常沿用 lower camel case。

很多运行时方法可以接收 tuple 或 list，装饰器会把它们转换为 SFML 向量类型：

```python
actor.setMapPosition((3, 5))
plainText.setOrigin((16, 8))
```

这只是调用便利。进入运行时状态后，应把这些值当作 `Vector2u`、`Vector2f`、`IntRect` 等引擎对象使用，而不是原始 tuple。

## 运行项目

在项目根目录运行：

```bash
python Entry.py
```

如果安装了 `debugpy`，Sample 会监听 `localhost:2333`。没有安装时游戏会照常继续。
