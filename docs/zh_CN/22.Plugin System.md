# 22. 插件系统

插件用于在不修改编辑器主体源码的情况下扩展 Ludork 编辑器。插件可以添加菜单命令、右键菜单项、右侧面板、底部标签页、顶栏控件、编辑器本地化文本，以及自定义工程数据类型。

官方本地化工具现在就是一个内置示例插件，位置为 `Plugins/OfficialLocaleTools`。

## 目录结构

插件位于 `main.py` 同级目录下，不放在具体游戏工程内。

```text
Ludork/
  Plugins/
    MyPlugin/
      Plugin.json
      Plugin.py
      locale.json
      icons/
```

`Plugin.json` 和 `Plugin.py` 必须存在。`locale.json`、图标、辅助模块和其他资源可以按需添加。

插件文件夹名必须与 `Plugin.json` 中的 `name` 完全一致。无效插件会在编辑器启动时跳过，并显示警告。

## 加载模型

编辑器启动时，Ludork 会扫描 `Plugins/*/Plugin.json`，校验每个 manifest，合并可选的 `locale.json`，再用 `runpy.run_path` 执行 `Plugin.py`。

`Plugin.py` 会收到一个 `EDITOR` 对象：

| 字段 | 含义 |
|---|---|
| `EDITOR.main_window` | 当前主窗口；窗口尚未就绪时可能为 `None`。 |
| `EDITOR.project_path` | 当前工程根目录。 |
| `EDITOR.plugin_dir` | 插件目录的绝对路径。 |
| `EDITOR.plugin_config` | 已解析的 `Plugin.json` 内容。 |

插件目录只会在加载 `Plugin.py` 期间临时加入 `sys.path`，加载结束后会恢复原始 `sys.path`。如果插件要使用自身的辅助模块，建议在加载时直接 import；如果必须延迟 import，请结合 `EDITOR.plugin_dir` 和 `importlib` 显式加载。

打包编辑器时，`Plugins/` 视为运行时资产目录，会复制到可执行输出目录旁边。不要把它作为编辑器 Python package 加入 Nuitka 编译输入，因为插件入口脚本是通过 `runpy` 动态加载的。

## Manifest

```json
{
  "name": "MyPlugin",
  "version": "1.0.0",
  "author": "Your Name",
  "description": "Short description.",
  "minEditorVersion": "0.1.0",
  "types": ["submenu", "rightclick", "panel", "tab", "toolbar", "datatype"],
  "hooks": {
    "submenu": []
  }
}
```

必填字段是 `name`、`version`、`types` 和 `hooks`。支持的 `types` 如下：

| Type | 用途 |
|---|---|
| `submenu` | 向 `FILE`、`EDIT`、`GAME`、`DATABASE` 或 `HELP` 菜单添加动作。 |
| `rightclick` | 向编辑器右键菜单添加动作。 |
| `panel` | 添加右侧编辑面板。 |
| `tab` | 添加底部标签页。 |
| `toolbar` | 添加顶栏控件。 |
| `datatype` | 注册新的工程数据目录和文件扩展名。 |

## 菜单动作

```json
{
  "hooks": {
    "submenu": [
      {
        "parentMenu": "DATABASE",
        "label": "MY_TOOL",
        "localeKey": "MY_PLUGIN_TOOL",
        "shortcut": "Ctrl+Shift+T",
        "separatorBefore": true
      }
    ]
  }
}
```

可以实现具名 hook：

```python
from PyQt5 import QtWidgets

def hook_submenu_my_tool(window):
    action = QtWidgets.QAction(ELOC("MY_PLUGIN_TOOL"), window)
    action.triggered.connect(lambda checked=False: run_tool(window))
    return action
```

也可以使用共享 fallback：

```python
def on_triggered(window, hook_type, config):
    if hook_type == "submenu" and config.get("label") == "MY_TOOL":
        run_tool(window)
```

`label` 会转换成蛇形命名，所以 `MY_TOOL` 对应 `hook_submenu_my_tool`。

## 右键菜单

```json
{
  "hooks": {
    "rightclick": [
      {
        "location": "fileExplorer",
        "condition": "hit",
        "label": "INSPECT_FILE",
        "localeKey": "MY_PLUGIN_INSPECT_FILE",
        "position": "after",
        "anchor": "DELETE"
      }
    ]
  }
}
```

具名 hook 签名：

```python
def hook_rightclick_fileExplorer(window, condition, item_data):
    ...
```

fallback 签名：

```python
def on_triggered(window, hook_type, config, condition, item_data):
    ...
```

`condition` 通常是 `hit`、`empty` 或 `always`。`item_data` 由菜单来源决定，可能是数据 key、文件路径、actor 数据字典、图层名、表格单元格或 `None`。

可用位置如下：

| Location | 区域 |
|---|---|
| `mapList` | 左侧地图列表。 |
| `layerTab` | 地图图层标签页。 |
| `editorPanel_light` | 光照模式下的地图画布。 |
| `editorPanel_actor` | Actor 模式下的地图画布。 |
| `blueprintGraphList` | 蓝图事件图列表。 |
| `fileExplorer` | 工程文件浏览器。 |
| `actorQueue` | Actor 放置队列。 |
| `animationAssets` | 动画资源区域。 |
| `commonFunction` | 公共函数列表。 |
| `tilesetList` | Tileset 列表。 |
| `autotileList` | Autotile 列表。 |
| `generalDataParamLabel` | 通用数据参数标签。 |
| `generalDataMember` | 通用数据成员列表。 |
| `generalDataTab` | 通用数据标签页。 |
| `localeSheetTab` | 官方本地化编辑器的 sheet 标签页。 |
| `localeTable` | 官方本地化编辑器的表格。 |
| `consoleFilter` | 控制台筛选下拉菜单。 |

对于支持命中判断的位置，插件也可以声明 `location_hit` 或 `location_empty`，直接绑定某个条件。

## 面板、标签页和工具栏

面板会插入到右侧编辑器栈：

```json
{
  "hooks": {
    "panel": {
      "tabLabel": "MY_PANEL",
      "localeKey": "MY_PLUGIN_PANEL",
      "editModes": [0, 1, 2]
    }
  }
}
```

```python
def hook_panel(window):
    return MyPanel(window)
```

`editModes` 中 `0` 表示图块模式，`1` 表示光照模式，`2` 表示 Actor 模式。

底部标签页使用：

```python
def hook_tab(window):
    return MyBottomTab(window)
```

工具栏 hook 可以返回任意 `QWidget`：

```python
def hook_toolbar_my_button(window):
    button = QtWidgets.QToolButton(window)
    button.setText(ELOC("MY_PLUGIN_BUTTON"))
    return button
```

## 自定义数据类型

插件可以注册存放在 `Data/{typeName}` 下的数据类型。

```json
{
  "types": ["datatype"],
  "hooks": {
    "datatype": {
      "typeName": "Quests",
      "extension": ".quest",
      "defaultFormat": "json"
    }
  }
}
```

```python
def hook_datatype():
    return {
        "loader": load_quest_file,
        "editor": create_quest_editor
    }
```

如果没有提供 loader，`defaultFormat: "json"` 会使用 JSON 加载，`defaultFormat: "dat"` 会使用 Ludork 的二进制数据加载器。

加载后的插件数据会参与编辑器数据快照、修改检查、重命名/删除跟踪，以及保存和引用维护。重新初始化游戏数据前会先清空插件数据，因此工程重载后不应保留旧数据或重复数据。

## 插件本地化

在 `Plugin.py` 同级添加 `locale.json`。推荐使用语言优先格式：

```json
{
  "en_GB": {
    "MY_PLUGIN_TOOL": "My Tool"
  },
  "zh_CN": {
    "MY_PLUGIN_TOOL": "我的工具"
  }
}
```

插件本地化 key 会在安装 hook 前合并进编辑器本地化表，因此插件代码和 manifest 都可以使用 `ELOC("MY_PLUGIN_TOOL")`。

## 官方示例

`Plugins/OfficialLocaleTools` 是参考插件。它提供：

- `DATABASE` 菜单中的本地化表打开与导出命令。
- `F11` 本地化编辑器和 `F12` 导出快捷键。
- 插件内的 UI 代码 `OfficialLocaleToolsLocaleEditor.py`。
- 插件内的 Excel 导入/导出代码 `OfficialLocaleToolsLocaleIO.py`。
- 菜单和对话框文本所需的 `locale.json`。

迁移编辑器工具到插件时，可以把这个插件作为实际模板。
