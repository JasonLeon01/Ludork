# 05. 蓝图与角色

蓝图决定地图上对象的外观、属性、交互和可视化事件。Ludork 默认模板已按用途分好目录。

## 当前分类

- `Actors`：玩家 Braver、对话测试对象、自动寻路测试对象。
- `Consumables`：红蓝宝石和血瓶。
- `Doors`：通用门、黄门、蓝门、红门。
- `Enemies`：史莱姆和骷髅士兵。
- `Equips`：基础剑和盾。
- `Items`：黄/蓝/红钥匙、怪物手册、楼层传送、破墙镐、地震卷轴、冰冻徽章和破冰镐。
- `Teleportations`：上楼和下楼传送点。

## 常用属性

编辑蓝图时常见字段包括图片路径、默认裁切区域、碰撞开关、移动速度、材质、光源、战斗属性、物品价格、图标、描述和持有数量变化。

玩家和敌人会使用生命、攻击、防御、经验、金币等数值。装备会影响玩家属性，消耗品会直接改变玩家状态或资源。

## 可视化事件

蓝图可以在编辑器中维护事件图。门会在碰撞时检查开门条件，楼梯会切换地图，道具会在使用时触发效果，测试对象会展示对话或自动寻路行为。

改事件后，保存项目并进入测试游戏，从地图上实际触发一次，确认音效、动画、物品消耗、地图变化和存档表现都正确。

## 放置到地图

在地图的 `角色模式` 下选择蓝图并放置。放置后检查位置、图层、标签和实例属性。需要被保存追踪的对象一定要有稳定标签。
