# 05. 蓝图与角色

蓝图定义可放置在地图上的角色。在编辑器中，可以把蓝图理解为带有外观、属性、组件和可视化事件的可复用对象模板。

## 打开蓝图

从文件浏览器或关联数据面板打开蓝图。在 **游戏** 菜单中选择 `新建蓝图`，或按 `F4` 创建新蓝图文件。实验已在地图中使用的行为时，建议先编辑副本。

## 蓝图分类

**Actors**

玩家与测试角色，例如 Braver、对话测试和寻路测试。

**Consumables**

会立即改变玩家数值或库存的拾取物。

**Doors**

普通门和彩色门。

**Enemies**

放置在地图上的敌人角色。

**Equips**

装备拾取物，例如初始剑和盾。

**Items**

钥匙、怪物手册、楼层传送物品和改变地形的工具物品。

**Teleportations**

楼层流程使用的上楼和下楼角色。

## 常见属性

**图片和裁剪**

选择精灵图片和默认裁剪。替换图片后在地图中测试对齐。

**碰撞**

控制角色是否参与接触、重叠和地图阻挡。

**移动**

速度和移动相关字段由玩家、敌人和寻路角色使用。

**灯光和材质**

用于地图光照和渲染效果。

**战斗数值**

玩家和敌人蓝图可包含 HP、攻击、防御、EXP、金钱、等级、职业、状态、特殊能力和奖励。

**图标、价格和描述**

由物品、装备、商店和菜单窗口使用。

## 事件页

蓝图可以包含创建、更新、碰撞、重叠、移除、遭遇或击败等可视化事件页。从节点列表添加节点，按顺序连接，然后保存并在地图上测试。

派生 Actor 或对应的 Python 行为类不建议重写 `update`、`lateUpdate`、`fixedUpdate` 等 update 类型方法；这些方法通常由基类用于调度和维护通用状态。持续逻辑应优先放在 `onTick`、`onLateTick`、`onFixedTick` 等 tick 钩子或对应事件页中，这样可以避免某些基类逻辑需要 `super()` 却被遗漏。

## 放置检查

放置蓝图后，检查位置、图层、标签、碰撞、实例属性覆盖和存档持久化。任何会被移除或再次访问的角色都应拥有稳定标签。

在事件图中动态生成 Actor 时，普通默认值使用 `CreateActorFromBPPath`。如果本次生成需要用 dict 覆盖类属性默认值，使用 `CreateActorFromBPPathWithDefaults`；若随后通过 `RecordAddedActor` 记录该 Actor，这些覆盖值会随存档恢复。
