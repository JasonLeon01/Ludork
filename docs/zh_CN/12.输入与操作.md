# 12. 输入与操作

Sample 运行时通过共享输入层统一键盘、鼠标、触摸和手柄输入。

## 默认操作

**键盘**

方向键移动玩家或菜单选择。`Enter` 和 `Space` 确认。`Escape` 取消、关闭窗口或打开游戏内菜单。

**鼠标**

左键尝试地图寻路或确认 UI 选择。右键取消、关闭窗口或打开游戏内菜单。

**触摸**

单指点击尝试地图寻路。双指触摸相当于右键取消。

**手柄**

方向轴用于导航或移动。确认和取消按钮遵循运行时输入映射。

## 动作辅助函数

游戏代码应使用动作辅助函数，而不是只检查一个物理按键。

```python
from Engine import Input

if Input.isActionTriggered(Input.getConfirmKeys(), handled=True):
    confirm()

if Input.isActionTriggered(Input.getCancelKeys(), handled=True):
    cancel()
```

## 地图点击移动

鼠标左键和触摸可以请求自动寻路。阻挡窗口打开时，地图场景会消费这些输入，避免一次 UI 点击同时移动玩家。

## 输入模式

输入层会记录当前活跃设备是鼠标还是手柄。UI 控件可以据此决定显示鼠标悬停行为还是手柄选择行为。

## 自定义输入建议

新增玩法输入时，请通过 `Engine.Input` 处理，并在消费后标记 handled。这样菜单、窗口、点击移动和触摸取消不会互相抢输入。
