# 12. 打包与实用注意

本章收集打包要求和常见脚本模式。

## 打包游戏项目

游戏项目打包会使用 Nuitka 编译项目的 `Entry.py`，并包含项目运行时目录：

- `Assets`
- `Data`
- `Main.ini`
- `pysf`
- `Engine`
- `Global`
- `Source`

仓库根目录的 `pack.bat` 和 `pack.sh` 用于打包 Ludork 工具链应用本身。游戏项目应使用项目打包流程，它的目标是已打开项目的 `Entry.py`。

## 打包所需 Python

从工具链中测试项目时，项目脚本可以通过 `runpy` 执行，因此玩家只是在工具链中测试游戏时，不需要额外安装本机 Python 环境。

打包不同。桌面打包流程使用 Nuitka，而 Nuitka 必须由用户机器上的真实 Python 解释器启动。由于 Nuitka 和工具链限制，这个 Python 解释器无法自动一起带入打包工具链。

桌面打包时，请在打包机器上安装本机 Python `3.12.0`：

- Windows：打包流程会查找 `py -3.12`，并要求版本为 `3.12.0`。
- macOS：打包流程会查找 `python3.12`，并要求版本为 `3.12.0`。

如果缺少 Python，请先安装再打包。Nuitka 可以在该 Python 环境中自动安装，但 Python 可执行文件本身必须已经存在。

## 原生工具链要求

桌面打包还需要原生编译工具链：

- Windows：MSVC Build Tools，提供 `cl.exe`。
- macOS：Xcode 命令行构建工具。

iOS 工程生成仅支持 macOS，并需要 Xcode 和 iPhoneOS SDK。

## 视频播放的 PyAV 要求

`Global.playVideo()` 和 `Global.Video` 依赖 PyAV，导入名为 `av`。

Ludork 工具链自身打包后的运行时会包含 PyAV，但玩家打包自己的游戏项目时，PyAV 不会默认带上。通过编辑器打包 Windows/macOS 桌面版本时，可在打包弹窗中勾选“包含 PyAV（视频播放）”。打包流程会在需要时把 PyAV 安装到用于打包的 Python 3.12.0 环境，并让 Nuitka 包含 `av`。

iOS 打包无法包含 PyAV，因为移动端无法安装 PyAV。

如果手动打包，且游戏播放视频，请在用于打包的 Python 环境中安装 PyAV：

```bash
python -m pip install av
```

如果使用自定义 Nuitka 命令，请显式包含 `av`：

```bash
python -m nuitka --include-module=av Entry.py
```

缺少 PyAV 时，游戏可能仍然能完成打包，但脚本导入 `av` 时视频播放会失败。

## 开始新游戏

```python
from Global import System, Manager
from Source.GameInstance import GameInstance
from Source.Scenes.SceneMap import Scene as SceneMap
from Source import System as GameSystem

Manager.playSE(GameSystem.getDecisionSE())
Manager.stopMusic("BGM")

scene = SceneMap()
scene.setInst(GameInstance())
System.setScene(scene)
```

## 移动玩家

```python
from Engine import Vector2i

player = self.inst.getPlayer()
if not player.isMoving():
    player.MapMove(Vector2i(1, 0))
```

使用单格偏移。`MapMove()` 会把每个分量限制到 `-1`、`0` 或 `1`。

## 显示对话

```python
self.showMessage("guide", LOC("NPC_GUIDE"), LOC("WELCOME_MESSAGE"))
```

当 `refActorTag` 匹配 actor 时，消息窗口可以定位到该 actor 附近。

## 显示选项

```python
result = self.showSelection(
    "guide",
    LOC("NPC_GUIDE"),
    [LOC("CHOICE_YES"), LOC("CHOICE_NO")],
    True,
)
```

结果是选项索引。取消返回 `-1`。

## 打开商店

```python
condition = self.openShop(["KEY_Y", "EnemyBook"], canSell=True)
```

节点函数 `Scene.OpenShop(items, canSell)` 会返回 latent 条件，商店关闭后继续执行后续节点。

## 切换地图

```python
self.gotoMapAndPos("Map_02.dat", (4, 8), blockTransition=False)
```

如果地图文件已经加载，Sample 会更新保存的地图位置，而不会重新加载所有地图资源。

## 按标签查找 Actor

```python
actor = self.getGameMap().getActorByTag("chest_01")
if actor:
    chest = actor
```

tag 适合用于事件目标、对话锚点和持久化 actor 记录。

## 永久销毁 Actor

```python
self.recordDestroyedActor(chest)
chest.destroy()
```

地图场景下次加载地图时会重新应用被销毁 actor 记录。

## 永久改变地形

```python
gameMap = self.getGameMap()
gameMap.destroyTerrain("Ground", (6, 12), None)
```

`destroyTerrain()` 会同时修改当前地图和存档数据。只想临时修改当前地图时，使用 `setTerrainTile()`。

## 记录楼层传送点

```python
from Engine import Vector2u

self.inst.recordTelepoint("Map_02.dat", Vector2u(4, 8))
self.showFloorTeleporter()
```

楼层传送器按当前区域读取已记录的传送点。

## 添加灯光

```python
from Global import Light
from Engine import Vector2f, Color

gameMap = self.getGameMap()
gameMap.addLight(Light(Vector2f(128, 128), Color(255, 210, 120, 255), 160.0, 1.0))
```

## 播放音效并闪屏

```python
from Engine import Color
from Global import Manager, System
from Source import System as GameSystem

Manager.playSE(GameSystem.getGetSE())
System.flashScreen(Color(255, 255, 255, 180), 0.25)
```

## 开始天气

```python
from Global import System

System.setWeather("Rain", 50, 80)
```

离开地图或事件结束时清除：

```python
System.clearWeather()
```

## 切到 Game Over

```python
from Source.Scenes import GameOver
from Global import System

System.setScene(GameOver())
```

节点图中也可以使用 `Scene.GameOver()`。

## 在场景中使用输入

```python
from Engine import Input

def onTick(self, deltaTime: float) -> None:
    if Input.isActionTriggered(Input.getCancelKeys(), handled=True):
        self._windowMenu.open()
```

当前对象消费动作时，应使用 `handled=True`。

## 实用注意

- 游戏特定代码放在 `Source`。
- 场景和屏幕操作使用 `Global.System`。
- 资源和音频使用 `Global.Manager`。
- 项目数据使用 `Source.Data`。
- actor 查找、生成和寻路使用 `GameMap` 方法。
- tuple 参数只是输入便利，运行时保存值是引擎对象。
- 着色器被禁用的平台上，注意处理返回 `None`。
