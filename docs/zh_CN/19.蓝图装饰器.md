# 19. 蓝图装饰器

Sample 运行时通过 `Sample/Engine/Decorators.py` 提供一组蓝图装饰器。它们在编辑器加载 Python 脚本时被读取，用于控制节点在蓝图编辑器中的显示、参数编辑方式，以及执行流/返回值引脚。

装饰器定义在 `Engine` 包中，运行时由 `Engine/__init__.py` 注入为全局名称，因此在 `Source/`、`Global/` 等脚本中可以直接书写 `@Meta(...)`，无需显式 import。

## 装饰器一览

| 装饰器 | 作用对象 | 用途 |
| --- | --- | --- |
| `@Meta(...)` | 函数、方法、类 | 编辑器元数据：显示名、描述、专用参数编辑器 |
| `@ExecSplit(...)` | 函数、方法 | 声明执行流出口（如 `default`、`Success`） |
| `@Latent(...)` | 函数、方法 | 声明延迟/异步出口，等待条件满足后继续 |
| `@ReturnType(...)` | 函数、方法 | 声明数据输出引脚名称与类型 |
| `@TypeAdapter(...)` | 函数、方法 | 在调用前自动转换参数类型 |
| `@RegisterEvent` | 函数、方法 | 将函数注册为蓝图事件页入口 |
| `@InvalidVars(...)` | 类 | 在蓝图属性编辑器中隐藏指定类变量 |
| `@RectRangeVars(...)` | 类 | 用图片范围选择器编辑矩形字段 |

带 `@ExecSplit`、`@Latent` 或 `@ReturnType` 的函数还会获得 `_refLocal`，从而出现在蓝图节点选择器中。仅带 `@Meta` 的函数不会自动暴露为节点。

## `@Meta` 编辑器元数据

`@Meta(...)` 为函数、方法或类附加 `_meta` 字典。编辑器当前识别这些 key：

- `DisplayName` 和 `DisplayDesc`：节点显示名和描述，通常写成 `LOC("KEY")`。编辑器在构建节点标题和函数选择器搜索缓存时会求值这些表达式。
- `DropBox`：节点参数下拉选项，例如 `{"attr": ["pitch", "volume"]}`。
- `Rely`：字段或参数之间的编辑依赖，例如 `{"target": ["source", True]}`，或字典形式 `{"target": {"source": "source", "value": expectedValue}}`（也接受 `key` / `var` 代替 `source`）。
- `PathVars`：以 `Assets` 下目录为根的路径字段或节点参数，例如 `[("texturePath", "Characters")]`。只写变量名字符串（如 `["texturePath"]`）时，默认根目录为 `Characters`。
- `ColourVars`：颜色字段或参数，例如 `["lightColour"]`。值仍保存 RGBA 元组，但在编辑器中显示颜色方块。
- `Vector2Vars`、`Vector2fVars`、`Vector2iVars`、`Vector2uVars`、`PairVars`、`PairFloatVars`、`PairIntVars`：二维字段或节点参数，在编辑器中显示为 x/y 数值编辑器。
- `Vector3Vars`、`Vector3fVars`、`Vector3iVars`、`Vector3uVars`：三维字段或节点参数，在编辑器中显示为 x/y/z 数值编辑器。
- `MoveRouteVars`：移动路线节点参数，在编辑器中显示为地图参考路线编辑器。值保存为相对 `(dx, dy)` 步进。
- `Transfer`：地图坐标节点参数，在编辑器中显示为地图格子选择对话框。格式：`[("posVarName", "mapVarName")]`。每个元组将坐标参数与另一个参数关联，后者的当前值用于在选择器中预选地图。确认选择后，若关联的地图参数为空或指向不存在的地图，会自动填入当前选中的地图 key。
- `InvalidVars`：在蓝图属性编辑器中隐藏的类变量，例如 `["defaultRect"]`。等价于类装饰器 `@InvalidVars(...)`。
- `RectRangeVars`：使用图片范围选择器编辑的矩形字段，例如 `{"defaultRect": "texturePath"}`。等价于类装饰器 `@RectRangeVars(...)`。

示例：

```python
@Meta(
    DisplayName='LOC("PLAY_SOUND")',
    DisplayDesc='LOC("PLAY_SOUND_DESC")',
    PathVars=[("soundFileName", "Sounds")],
)
@ExecSplit(default=(None,))
def PlaySound(soundFileName: str, applyFilter: bool = False) -> None:
    ...
```

## `@ExecSplit` 执行流出口

`@ExecSplit` 声明函数完成后的执行引脚。键名为出口标签，值为元组（保留给运行时扩展，通常写 `(None,)` 或具体分支索引）。

常见写法：

```python
@ExecSplit(default=(None,))
def DictAdd(dict_: Dict, key: Any, value: Any) -> None:
    ...

@ExecSplit(Success=(0,), Failed=(1,))
def RemoveItem(itemID: str, count: int = 1) -> int:
    ...

@ExecSplit(TRUE=(0,), FALSE=(1,))
def IF(condition: bool = False) -> int:
    ...
```

## `@Latent` 延迟出口

`@Latent` 用于需要等待条件满足后再继续的节点。函数通常返回一个可调用对象；蓝图在对应出口上等待该条件变为 `True`。

```python
@Latent(Finished=(True,))
def RequestTransition(transitionName: str = "", transitionTime: float = 1.0) -> Callable[[], bool]:
    ...

@Latent(Closed=(True,))
def OpenShop(items: List[str], canSell: bool = True) -> Callable[[], bool]:
    ...
```

## `@ReturnType` 数据输出

`@ReturnType` 为纯数据节点声明输出引脚。键为引脚名，值为类型注解（用于文档与编辑器提示）。

```python
@ReturnType(value=bool)
def DictContains(dict_: Dict, key: Any) -> bool:
    ...

@ReturnType(index=int)
def ListFind(list_: List[Any], value: Any) -> int:
    ...
```

## `@TypeAdapter` 参数类型适配

`@TypeAdapter` 在函数调用前按映射表转换参数。每个参数对应一个元组：

- `(originType, targetType)`：使用 `targetType` 作为转换器；
- `(originType, targetType, converter)`：使用指定 `converter` 函数。

`originType` 可以是单个类型，也可以是类型列表。当传入值为 `list` 或 `tuple` 时，转换器以解包方式调用。

```python
@TypeAdapter(position=((list, tuple), Vector2i, Vector2i))
def setMapPosition(self, position: Vector2i) -> None:
    ...
```

## `@RegisterEvent` 事件注册

`@RegisterEvent` 将函数标记为蓝图事件页入口，并自动附加 `default` 执行出口。可带括号或不带括号使用。

```python
@RegisterEvent
def onCreate(self) -> None:
    ...
```

## 类装饰器

`@InvalidVars("fieldA", "fieldB")` 和 `@RectRangeVars(defaultRect="texturePath")` 也可直接用于类体。它们与 `@Meta(InvalidVars=...)` / `@Meta(RectRangeVars=...)` 效果相同。

## 参数默认值

带 Python 签名默认值的节点参数仍会显示在蓝图编辑器中。未连线的引脚使用源码中的默认值。

默认值应使用 Python 基础值（`None`、`bool`、`int`、`float`、`str`，或由这些值组成的容器）。如果默认值要指向引擎或编辑器对象，应写成字符串，并由函数内部显式处理。

`Source.NodeFunctions` 中大量函数已为常用参数补充默认值，例如：

- `String` 模块中字符串参数默认 `""` 或 `"0"`；
- `Math` 模块中向量分量默认 `0`，比较运算默认 `0` / `1`；
- `Player` 模块中数量类参数默认 `1`；
- `System` 模块中 `PlaySound` / `PlayMusic` 的 `applyFilter` 默认 `False`，`FlashScreen` 颜色通道默认 `255`。

完整签名见 `18.API说明/02.NodeFunctions/`。

## 组合顺序

推荐从外到内的装饰器顺序为：`@Meta` → `@ExecSplit` / `@Latent` / `@ReturnType` → `@TypeAdapter`（若需要）。`@Meta` 只影响编辑器展示，不改变运行时行为；执行流与返回类型装饰器决定节点在蓝图图中的引脚结构。
