# 02. 项目与运行时结构

本章说明游戏项目如何启动，以及运行时各个包如何协作。

## 项目根目录

游戏从项目根目录运行。运行时脚本中的路径通常相对于该目录：

```text
./Main.ini
./Data/Configs/System.json
./Assets/System/icon.png
./Assets/System/cursor.png
./Assets/System/WS-prefix100-original2.png
./Save/
```

游戏脚本建议放在 `Source/` 中，除非你正在扩展运行时本身。

## Main.ini

`Main.ini` 保存创建窗口前需要读取的进程级选项。

`[Main]` 中常见字段：

- `script`：主脚本文件名，通常是 `Entry.py`。
- `language`：本地化 key，例如 `zh_CN` 或 `en_GB`；为空时使用系统语言。
- `scale`：显示缩放。
- `frameRate`：目标帧率。
- `verticalSync`：是否启用垂直同步。
- `musicOn`、`soundOn`、`voiceOn`：音频分类开关。
- `musicVolume`、`soundVolume`、`voiceVolume`：`0` 到 `100` 的分类音量。

这些值由 `Global.System.init()` 读取，并可通过系统 setter 保存回文件。

## 启动流程

`Entry.py` 的运行步骤如下：

1. 导入 `Engine`、`Global` 和 `Source`。
2. 如果存在 `WINDOWHANDLE`，启动 stdin 调试线程，允许嵌入式宿主发送表达式或语句。
3. 如果安装了 `debugpy`，监听 `localhost:2333`。
4. 从 `Data/Locale` 初始化游戏本地化。
5. 初始化节点图 latent 支持。
6. 通过 `Global.System` 读取 `Main.ini`。
7. 将场景操作绑定到当前线程。
8. 设置第一个场景为 `Source.Scenes.Init()`。
9. 调用 `Source.System.init()` 创建窗口并读取核心配置。
10. 在游戏应继续运行时，重复执行当前场景的 `main()`。
11. 退出时保存 FPS 历史并打印退出信息。

简化形态：

```python
Engine.Locale.init("./Data/Locale")
Engine.NodeGraph.initLatent()
Global.System.init(iniFile, iniFilePath)
Global.System.bindSceneOperationThread()
Global.System.setScene(Source.Scenes.Init())
Source.System.init()

while Global.System.shouldLoop():
    Global.System.applyPendingSceneReplace()
    scene = Global.System.getScene()
    if scene:
        scene.main()
```

## Engine 包

`Engine` 导出 SFML/PySF 类和 Ludork 工具。需要向量、颜色、输入、UI、节点装饰器、文件工具或玩法基础对象时使用它。

示例：

```python
from Engine import Vector2u, Vector2f, Color
from Engine import Input
from Engine.UI import PlainText
```

常用值：

- `Engine.CellSize`
- `Engine.GameSize`
- `Engine.Scale`
- `Engine.GameRunning`

常用工具：

- `Engine.Cast(type_, value)`
- `Engine.AssertType(obj, annotation)`
- `Engine.Eval(expr)`
- `Engine.OppositeDirection(direction)`

## Global 包

`Global` 提供游戏全局服务：

- `System`：窗口、画布、设置、场景栈、转场、闪屏、震动、天气、雾。
- `SceneBase`：场景基类。
- `GameMap`：瓦片地图、角色、碰撞、灯光和寻路。
- `Camera`：地图视图与渲染纹理。
- `Manager`：资源、音频、字体、贴图、着色器和时间。
- `playVideo()`：阻塞式视频播放辅助函数。

示例：

```python
from Global import System, SceneBase
from Global import Manager
```

## Source 包

`Source` 是 Sample 游戏自己的规则层。真实项目可以替换或重组它。

重要模块：

- `Source.System`：读取 `Data/Configs/System.json` 和 `Audio.json`。
- `Source.Data`：加载动画、图块集、自动图块、公共函数和通用数据。
- `Source.GameInstance`：保存玩家和持久游戏状态。
- `Source.Scenes`：初始化、标题、地图和 Game Over 场景。
- `Source.Windows`：Sample RPG 窗口，包括菜单、存读档、物品、装备、商店、怪物手册、楼层传送和配置窗口。
- `Source.NodeFunctions`：可供节点调用的项目函数。

## 初始场景

初始场景是 `Source.Scenes.SceneInit.Scene`。它通常加载项目数据，然后切换到标题场景。

标题场景开始新游戏时会创建 `GameInstance()`，再切换到 `SceneMap`。

## 线程注意

场景逻辑运行在线程中，渲染和场景替换保持在场景操作线程。切换场景时使用 `System.setScene()`，不要直接修改内部场景列表。

如果从其他线程调用 `System.setScene()`，替换会排队，并由 `System.applyPendingSceneReplace()` 在绑定线程中应用。
