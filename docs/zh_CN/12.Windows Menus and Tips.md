# 12. 窗口、菜单与提示

Sample 游戏提供了一组供标题流程、地图流程、事件和存档使用的窗口。

## 标题窗口

**命令窗口**

显示 `开始`、`继续`、`设置` 和 `退出`。

**存档读取窗口**

由 `继续` 用于读取已有存档槽。

**设置窗口**

修改运行时设置，并通过配置系统写回。

## 地图窗口

**HUD**

显示玩家关键数值。奖励、装备变化、物品使用、受伤、治疗和状态变化后应更新。

**菜单**

在地图上通过取消输入打开。提供物品、装备、存档和返回标题等命令。

**物品窗口**

显示可用物品并处理物品使用。

**装备窗口**

显示装备槽和可用装备。

**存档窗口**

把当前进度写入存档文件夹。

## 焦点组与窗口导航

Sample 运行时把 `active` 和键盘焦点分开处理。`active=True` 表示窗口或焦点组可以成为键盘、手柄操作的落点；`active=False` 表示它当前不可选中。焦点本身全局唯一，由场景的 `FocusManager` 管理。

**焦点组**

相关窗口会被放进 `FocusGroup`。方向键在当前窗口内部先移动光标；到达边界且存在方向邻组时，焦点可以跳到相邻组。如果中间组不可进入（例如 `active=False`），方向导航会继续沿同方向查找下一个可进入组。

**确认式子窗口**

装备、存档、商店和楼层传送这类窗口不会仅靠方向键进入下一级列表。它们需要确认当前命令后显式转移焦点，取消时再退回上一级：

- 装备菜单确认后焦点进入装备槽；确认装备槽后进入可选装备；在可选装备上取消会回到装备槽。
- 存档菜单确认 `Load` 或 `Save` 后焦点进入存档列表；在存档列表上取消会回到 `Load / Save` 命令栏。
- 商店和楼层传送也遵循“确认进入、取消返回”的规则。

**鼠标与焦点**

可见窗口可以响应鼠标悬停，悬停到可接收焦点的选择窗口时会同步键盘光标。点击、滚轮、触摸确认仍受 `active` 限制，避免非当前操作层误确认。

## 事件窗口

**消息窗口**

显示对话和选项。事件提供参考位置时，它可以相对地图角色定位。

**商店窗口**

根据给定物品列表处理购买和出售。

**怪物手册**

显示当前地图敌人并预览战斗伤害。

**楼层传送**

显示已记录楼层和地图预览。

## 从游戏逻辑打开窗口

尽量使用现有场景或节点辅助函数：

```python
from Source.NodeFunctions.Scene import OpenShop
from Source.NodeFunctions.Mota import OpenMonsterBook

OpenShop(["Bottle_200", "Sword_A"], True)
OpenMonsterBook()
```

## 窗口规则

阻挡窗口应消费取消和点击输入，停止地图点击移动，并在关闭后恢复玩家移动。

自定义窗口如果要参与跨窗口导航，应在场景中注册 `FocusGroup`，为相邻窗口设置方向邻接，并在确认、取消或业务状态变化时调用窗口的焦点请求方法。不要依赖 Z-order 来决定谁接收键盘输入。
