# 18. 测试与常见问题

当 Sample 工程表现不符合预期时，可使用本页作为简明检查清单。

## 标准测试路线

1. 保存工程。
2. 如果文本变化，导出本地化。
3. 启动 `测试游戏`。
4. 开始新游戏。
5. 移动、点击寻路、打开菜单，并触发附近角色。
6. 测试门、楼梯、敌人、物品、商店、怪物手册和楼层传送。
7. 保存，返回标题，再从该存档继续。
8. 切换语言，重新启动，并检查可见文本。

## 新内容没有出现

**可能原因**

编辑过的面板没有保存，打开了错误工程，素材移动后没有重新选择，或修改的蓝图不是地图上放置的那个。

**检查**

从编辑器打开地图或数据文件，确认值，重新保存，然后重启 `测试游戏`。

## 文本缺失

**可能原因**

本地化 key 不存在，某个语言为空，或导出的语言文件已过期。

**检查**

确认 `zh_CN` 和 `en_GB` 都包含该 key，保持占位符一致，导出本地化，并重启测试游戏。

## 角色没有反应

**可能原因**

角色没有碰撞，触发页面不对，标签不匹配，条件不可能成立，或阻挡窗口消费了输入。

**检查**

在小地图上测试该角色，确认标签，并临时添加打印或消息事件来确认页面是否运行。

## 存档看起来不对

**可能原因**

存档是在改动前创建的，角色标签变化，地形编辑使用了不同地图或图层名，或自定义存档字段没有默认值。

**检查**

使用新存档和旧存档分别测试。确认位置、库存、装备、状态、已移除角色、改变过的地形和楼层点。

## 音频或效果失败

**可能原因**

音频开关或音量关闭，文件在错误文件夹，地图使用了另一个声音设置，或效果时间太短不明显。

**检查**

先用简单事件测试声音，再在最终地图流程中测试。

## 插件工具没有出现

**可能原因**

插件文件夹名与 `Plugin.json` 的 `name` 不一致，`types` 包含不支持的值，`minEditorVersion` 高于当前编辑器版本，`Plugin.py` 加载时报错，或插件把依赖自身目录的 import 延迟到了临时 `sys.path` 恢复之后。

**检查**

确认插件位于 `Plugins/{PluginName}`，查看启动阶段警告；插件内部模块应在加载时 import，或通过 `EDITOR.plugin_dir` 显式加载。修改 `datatype` hook 后，请重新加载工程，让插件数据先清空再重新载入。

## 控制台线索

使用编辑器控制台过滤器区分普通消息、警告和错误。调试事件时，短打印动作通常比猜测分支更快。
