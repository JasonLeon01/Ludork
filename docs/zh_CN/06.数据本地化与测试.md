# 06. 数据、本地化与测试

本页说明编辑器侧的数据维护。它从项目作者角度解释如何使用数据工具以及如何测试结果。

## 通用数据

从数据库菜单打开 `通用数据`。

**职业**

定义玩家职业数据，例如装备槽。

**敌人**

定义敌人显示文本、战斗数值、奖励、掉落和特殊能力。

**装备**

定义槽位、图标、价格、描述和数值加成。

**物品**

定义物品显示文本、图标、价格、是否可使用、库存消耗和相关行为字段。

**特殊能力与状态**

定义敌人和战斗者使用的可复用战斗标记与状态记录。

新增记录时，先复制一个相似的现有记录。这样必填字段更完整，也方便比较。

## 本地化表

打开内置本地化编辑器或本地化工作簿。

**Key**

名称、描述、菜单文本和提示应使用稳定 key。

**语言**

每个用户可见 key 都应填写 `zh_CN` 和 `en_GB`。

**占位符**

如果一种语言包含 `{name}` 这样的占位符，所有语言都必须包含同名占位符。

**导出**

编辑后导出语言文件。测试游戏读取的是导出的文件，而不只是工作簿视图。

## 动画数据

通过动画编辑器打开动画数据。先在预览中检查图片、声音、时机、缩放和位置，再在测试游戏中验证动画触发效果。

## 基本测试流程

1. 保存所有编辑过的面板。
2. 文本变化后导出本地化。
3. 启动 `测试游戏`。
4. 开始新游戏或加载相关存档。
5. 触发被修改的地图、角色、物品、窗口或设置。
6. 检查文本或缩放时，切换语言并重新启动测试游戏。
