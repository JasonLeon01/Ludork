# 09. 场景

场景是游戏流程的主要单位。标题画面、地图玩法、全屏菜单和加载画面都适合作为场景。

## SceneBase

所有场景都应继承 `Global.SceneBase`。

```python
from Global import SceneBase

class Scene(SceneBase):
    def onCreate(self) -> None:
        pass
```

`SceneBase` 提供生命周期钩子、UI 管理器、定时器、动画列表、通用提示和内部场景循环。

## 生命周期

可重写的方法：

- `onCreate()`：首次进入场景前调用一次。
- `onEnter()`：场景成为当前场景时调用。
- `onTick(deltaTime)`：逻辑线程中的每帧更新。
- `onFixedTick(fixedDelta)`：固定步长更新，默认 `1 / 60`。
- `onLateTick(deltaTime)`：显示前的后期更新。
- `onQuit()`：离开场景循环时调用。
- `onDestroy()`：场景从场景栈移除时调用。

最小场景示例：

```python
from Global import SceneBase, System
from Engine import Color

class Scene(SceneBase):
    def onEnter(self) -> None:
        System.setTransition(transitionTime=0.5)

    def onTick(self, deltaTime: float) -> None:
        if someCondition:
            System.flashScreen(Color.White, 0.25)
```

## 场景栈

`Global.System` 管理当前场景栈。

重要方法：

- `System.setScene(scene)`：替换当前场景。
- `System.pushScene(scene)`：压入一个场景。
- `System.popScene()`：移除栈顶场景。
- `System.getScene()`：获取当前场景。
- `System.getSceneList()`：获取完整场景栈。
- `System.exit()`：移除所有场景并结束游戏循环。

示例：

```python
from Global import System
from Source.Scenes.SceneTitle import Scene as SceneTitle

System.setScene(SceneTitle())
```

## 定时器

`SceneBase.addTimer(key, interval, task=None, params=None)` 添加一次性定时器，并返回一个条件函数。

```python
def showReady(self) -> None:
    self.addCommonTip("Ready")

self.addTimer("ready-tip", 1.0, self.showReady)
```

该方法支持 latent，因此节点图逻辑可以等待定时器结束。

## 场景动画

场景可以拥有 `Global.Animation` 实例。

方法：

- `addAnim(anim)`
- `getAnims()`
- `removeAnim(anim)`
- `clearAnims()`

场景动画由 `SceneBase` 自动更新和绘制。

## 通用提示

`addCommonTip(text)` 在当前场景显示浮动提示。

```python
self.addCommonTip(LOC("ITEM_GET"))
```

`GameMap.addCommonTip(text)` 会转发给所属场景。

## UI 管理器

每个场景拥有 `self._uiManager`。用它加载控件和窗口：

```python
self._uiManager.loadUI(self._windowCommand)
```

模板标题场景在 `onCreate()` 中加载背景、指令窗口、存读档窗口和设置窗口。

## 标题场景模式

模板标题场景执行以下工作：

1. 在 `onEnter()` 开始转场。
2. 在 `onCreate()` 创建背景与指令 UI。
3. 播放标题 BGM。
4. 选择开始时创建 `GameInstance()`。
5. 切换到 `SceneMap`。
6. 打开存读档窗口或配置窗口时暂停标题指令窗口。
7. 在 `onQuit()` 和 `onDestroy()` 停止标题 BGM。

场景切换示例：

```python
from Global import System, Manager
from Source.GameInstance import GameInstance
from Source.Scenes.SceneMap import Scene as SceneMap

Manager.stopMusic("BGM")
nextScene = SceneMap()
nextScene.setInst(GameInstance())
System.setScene(nextScene)
```

## Game Over 场景

模板项目中的 `Source.Scenes.GameOver` 是一个全屏场景：

- 进入时播放较长转场。
- 绘制黑色背景和居中的 `GAME OVER` 文本。
- 按确认键或鼠标左键返回标题场景。

敌人战斗判定失败或节点函数 `Scene.GameOver()` 都会切换到该场景。

## 地图场景模式

模板地图场景拥有玩家、HUD、消息窗口、菜单、存读档窗口、物品/装备窗口、商店窗口、怪物手册、楼层传送器和当前 `GameMap`。

重要方法：

- `setInst(inst)`：设置当前 `GameInstance`。
- `loadMap(mapPath)`：加载地图数据并生成 `GameMap`。
- `gotoMapAndPos(mapPath, pos=None, *, blockTransition)`：切换地图并设置玩家位置；`blockTransition` 为 `True` 时跳过地图 transition。
- `showMessage(refActorTag, name, message)`：显示对话。
- `showSelection(refActorTag, name, options, allowCancel)`：显示选择项。
- `openShop(items, canSell)`：打开商店并返回可等待的关闭条件。
- `showEnemyBook()`：打开当前地图怪物手册。
- `showFloorTeleporter()`：打开已记录楼层传送点窗口。
- `recordDestroyedActor(actor)`：持久化被销毁的 actor 标签。
- `applyLoadedGame(inst)`：加载存档后重新绑定玩家、UI 和地图状态。

新增玩法场景时，可优先参考地图场景。
