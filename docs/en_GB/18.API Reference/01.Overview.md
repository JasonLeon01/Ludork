# 01. Overview

This folder documents the blueprint-visible API collected from handwritten Python scripts. Generated blueprint data classes are not included.

## Reading Rules

- Original names use the Python module path and function name.
- Display names and descriptions are resolved from `Sample/Data/Locale/Locale.xlsx` when the API declares `LOC("...")` metadata.
- If no localisation metadata exists, the blueprint node picker falls back to the function name; this manual does the same.
- Class sections list public class variables declared directly in the handwritten class body, then the API methods exposed from that class.
- `Source.NodeFunctions` is documented separately because those functions appear as standalone node groups.

## Pages

Each Markdown file inside a category folder documents one handwritten Python source file.

- `02.NodeFunctions/`: standalone node functions.
- `03.Source Class APIs/`: Sample gameplay classes and info/event classes.
- `04.Global APIs/`: scene, map, camera, UI manager, and global runtime helpers.
- `05.Engine Gameplay APIs/`: engine actor and tilemap APIs.
