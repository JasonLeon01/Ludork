# 01. Overview

This folder documents the blueprint-visible API collected from handwritten Python scripts. Generated blueprint data classes are not included.

## Reading Rules

- Original names use the Python module path and function name.
- Display names and descriptions are resolved from `Sample/Data/Locale/Locale.xlsx` when the API declares `LOC("...")` metadata.
- If no localisation metadata exists, the blueprint node picker falls back to the function name; this manual does the same.
- Class sections list public class variables declared directly in the handwritten class body, blueprint-editable component fields where applicable, then the API methods exposed from that class.
- `Source.NodeFunctions` is documented separately because those functions appear as standalone node groups.

## Editor Metadata

Python functions and classes can use `@Meta(...)` to attach editor-facing hints. The editor currently recognises these keys:

- `DisplayName` and `DisplayDesc`: localised node names and descriptions, usually written as `LOC("KEY")`.
- `DropBox`: combo-box options for node parameters, for example `{"attr": ["pitch", "volume"]}`.
- `Rely`: edit dependencies between fields or parameters, for example `{"target": ["source", True]}`.
- `PathVars`: path fields or node parameters rooted under `Assets`, for example `[("texturePath", "Characters")]`.
- `ColourVars`: colour fields or parameters, for example `["lightColour"]`. Values store RGBA tuples but display through a colour swatch editor.
- `Vector2Vars`, `Vector2fVars`, `Vector2iVars`, `Vector2uVars`, `PairVars`, `PairFloatVars`, `PairIntVars`: two-component fields or node parameters displayed as x/y numeric editors.
- `Vector3Vars`, `Vector3fVars`, `Vector3iVars`, `Vector3uVars`: three-component fields or node parameters displayed as x/y/z numeric editors.
- `MoveRouteVars`: movement-route node parameters displayed with a map reference route editor. Values are saved as relative `(dx, dy)` steps.
- `Transfer`: map coordinate node parameters displayed with a map tile picker dialog. Format: `[("posVarName", "mapVarName")]`. Each tuple pairs the coordinate parameter with another parameter that supplies the map key used to pre-select the map in the picker. If the map parameter is empty or refers to a non-existent map when the dialog is confirmed, it is automatically filled with the selected map key.
