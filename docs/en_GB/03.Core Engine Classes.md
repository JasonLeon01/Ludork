# 03. Core Engine Classes

This chapter lists the basic objects and helpers most game scripts use.

## Vector and Rect Types

The runtime uses SFML-style vector classes.

Common types:

- `Vector2i`: integer position.
- `Vector2u`: unsigned integer position or size.
- `Vector2f`: floating-point position.
- `Vector4f`: floating-point four-component value.
- `IntRect`: rectangle made from position and size vectors.
- `FloatRect`: floating-point rectangle.

Create rectangles with vector values:

```python
from Engine import IntRect, Vector2i

rect = IntRect(Vector2i(0, 0), Vector2i(32, 32))
```

Do not use the older four-number rectangle style.

## Colour

`Color` stores RGBA components from `0` to `255`.

```python
from Engine import Color

white = Color.White
flash = Color(255, 255, 255, 160)
```

## Direction

`Direction` describes four-way grid movement:

- `Direction.DOWN`
- `Direction.LEFT`
- `Direction.RIGHT`
- `Direction.UP`

`OppositeDirection(direction)` returns the reverse direction.

```python
from Engine import Direction, OppositeDirection

back = OppositeDirection(Direction.LEFT)
```

## Type Adaptation

Many runtime methods are decorated with `TypeAdapter`. This allows scripts to pass tuples or lists where vector objects are expected.

```python
actor.setOrigin((16, 16))
actor.setScale((1.0, 1.0))
```

This is a call convenience only. After the call, runtime state is stored as engine types.

## Cast

`Cast(targetType, value)` returns `value` typed as `targetType` for static analysis and readability.

```python
from Engine import Cast
from Source.Scenes.SceneMap import Scene as SceneMap
from Global import System

scene = Cast(SceneMap, System.getScene())
```

`Cast` does not construct a new object.

## AssertType

`AssertType(obj, type_)` recursively checks runtime values against Python type annotations.

```python
from Engine import AssertType

AssertType(["Potion", "Key"], list[str])
```

It supports common containers and unions.

## BPBase

`BPBase` is the base for blueprint-style objects such as actors. It stores and executes a node graph attached to an instance.

Actors generated from map data receive a graph through `actor.setGraph(...)`. Events such as `onCreate` and `onCollision` can then be dispatched by `Actor.BlueprintEvent(...)`.

## Decorators

The runtime exposes several decorators as builtins after `Engine` is imported:

- `@RegisterEvent`: marks a method as a blueprint event.
- `@Latent(...)`: describes latent return conditions for node execution.
- `@ExecSplit(...)`: describes branch-like execution results.
- `@ReturnType(...)`: records named return values.
- `@TypeAdapter(...)`: adapts argument values.
- `@PathVars(...)`, `@RectRangeVars(...)`, `@InvalidVars(...)`: metadata for data and visual scripting tools.

Game scripts normally use these decorators when adding node-callable functions or actor events.

## File Helpers

`Engine.Utils.File` contains data helpers used by runtime loading.

Common functions:

- `getJSONData(path)`: load UTF-8 JSON.
- `loadData(path)`: load project binary `.dat` data.
- `saveData(path, data)`: save project binary data.

Example:

```python
from Engine.Utils import File

systemData = File.getJSONData("./Data/Configs/System.json")
mapData = File.loadData("./Data/Maps/Map_01.dat")
```

## Math Helpers

`Engine.Utils.Math` contains conversion helpers and small numeric utilities. It is often used when converting between tuple-like values, engine vectors, and rectangles.

Example:

```python
from Engine.Utils import Math

position = Math.ToVector2f((320, 240))
```

## Locale Helper

After `Engine.Locale.init("./Data/Locale")`, localised text can be read with `LOC("KEY")`.

Example:

```python
title = LOC("TITLE_START")
```

The sample project stores locale data in `Data/Locale`.
