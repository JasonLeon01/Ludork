# 09. Map Play Flow

The Sample game map scene combines terrain, actors, input, UI windows, lighting, audio, fog, weather, saves, and scene transitions.

## Start Flow

A new game enters `Map_01.dat` at `(6, 12)`. Check player movement, camera follow, HUD values, map audio, and fog before testing individual actors.

## Map Transfer

Upstairs and downstairs actors transfer the player between maps. They search for matching teleport anchors and keep the player's movement state safe during the transfer.

## Floor Teleport Points

Visited stairs can be recorded as floor teleport points. The floor teleport window shows known floors and preview information, then sends the player to the selected map point.

The Mota node helpers expose the tower-specific window actions:

```python
from Source.NodeFunctions.Mota import OpenFloorTeleporter, OpenMonsterBook

OpenFloorTeleporter()
OpenMonsterBook()
```

## Terrain Changes

Utility items can alter terrain. The Sample game records these changes so they remain after saving and loading.

Typical terrain-changing functions are exposed to visual events through scene node functions:

```python
from Source.NodeFunctions.Scene import DestroyTerrain, GetTerrainTile

tile = GetTerrainTile("Ground", [6, 12])
DestroyTerrain("Ground", [6, 12], 0)
```

## Interaction Blocking

Dialogue, shop, item, equipment, save, enemy book, and floor teleport windows block map movement while open. When custom windows are added, they should follow the same rule: block map input while the player is making a window choice, then restore movement after closing.

## Lighting and Readability

Ambient light, map lights, actor lights, fog, and UI windows can overlap. Always test dark maps with the HUD and a blocking window visible.
