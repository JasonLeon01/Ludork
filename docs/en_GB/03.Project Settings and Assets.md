# 03. Project Settings and Assets

This page explains how to use editor-facing settings and asset selection. It does not describe runtime implementation; use it as a checklist for project authors.

## Game Settings

Open `Game Settings` from the game menu, or press `F2`, to edit values stored in `Main.ini`.

**Script**

The startup script is normally `Entry.py`. Leave this unchanged unless the project has a custom entry script.

**Language**

Choose a language that exists in `Data/Locale`. The default documentation language is `en_GB`; the default Sample project also includes `zh_CN`.

**Scale**

Changes the window scale. Restart the test game after changing scale so all UI layout is re-created at the intended size.

**Frame rate and vertical sync**

Frame rate sets the target update rate. Vertical sync helps stabilise display timing. Test movement and animation after changing either value.

**Audio switches and volumes**

Music, sound, and voice can be switched separately. Volumes use `0` to `100`.

## System Config

Open `System Config` from the database menu, or press `F6`.

**Title**

The game window title and title-screen identity.

**Game size**

The logical content size. The default is `640 x 480`.

**Fonts**

Select font files from the project font assets. Test both Chinese and English text after changing fonts.

**Icon, cursor, and window skin**

Select images from `Assets/System`.

**Cell size**

The tile size used by maps and movement. The default is `32`.

**Start map and start position**

Choose the map and tile coordinate used for a new game. The coordinate is in map cells, not pixels.

## Audio Config

Open the audio config panel from the database or system tools.

**Interface sounds**

Cursor, confirm, cancel, error, save, and load sounds should be short and clear.

**Map feedback**

Door, stairs, item pickup, equipment, and shop sounds should match the action timing.

**Title music**

The title BGM is chosen from `Assets/Musics`.

## Asset Folders

**System**

Icon, cursor, window skin, and title background.

**Tilesets and Autotiles**

Terrain images used by map editing.

**Characters and Icons**

Actor sprites, pickup sprites, item icons, equipment icons, and state icons.

**Sounds, Voices, and Musics**

Sound effects, dialogue voice clips, and music. Voice files used by dialogue nodes belong under `Assets/Voices`.

**Fogs, Transitions, Animations, Shaders**

Visual effect assets.

## Replacement Check

After replacing an asset, select it again in every panel that references it, save, and test the screen or map where it appears.
