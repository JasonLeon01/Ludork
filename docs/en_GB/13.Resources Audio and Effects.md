# 13. Resources, Audio, and Effects

`Global.Manager` is the main resource facade for game scripts.

## Textures

`Manager.loadTexture(subFolder, filename, sRGB=False, area=None, smooth=False)` loads from `Assets/<subFolder>`.

Convenience functions:

- `loadCharacter(filename)`
- `loadSystem(filename)`
- `loadTileset(filename)`
- `loadAutotile(filename)`
- `loadFog(filename)`
- `loadTransition(filename)`

Example:

```python
from Global import Manager

texture = Manager.loadSystem("GrassBackground.png")
transition = Manager.loadTransition("001-Blind01.png")
```

If a texture filename has no extension, `.png` is added.

## Fonts

Functions:

- `loadFont(filename)`
- `getFont(fontName)`
- `getFontList()`
- `getFontFilename(fontName)`
- `hasFont(fontName)`

`Source.System.init()` loads default fonts from `Data/Configs/System.json` and assigns `Engine.UI.DefaultFont`.

## Sound Effects

`Manager.playSE(filename, filter=None)` plays a sound from `Assets/Sounds`.

```python
Manager.playSE(GameSystem.getDecisionSE())
```

`Manager.stopSound()` stops active sound effects.

Sound playback respects `Global.System.getSoundOn()` and sound volume settings.

## Music

`Manager.playMusic(musicType, filename, filter=None)` plays from `Assets/Musics`.

```python
music = Manager.playMusic("BGM", "Theme.ogg")
if music is not None:
    music.setLooping(True)
```

`Manager.stopMusic(musicType)` stops one music channel. The template project uses names such as `BGM` and `BGS`.

Music playback respects `Global.System.getMusicOn()` and music volume settings.

## Audio Filters

The template map scene builds `Filters.SoundFilter` and `Filters.MusicFilter` from map data.

Common filter fields:

- `loop`
- `offset`
- `pitch`
- `pan`
- `volume`
- `loopPoint` for music

Use `setBgmFilter(attr, value)` and `setBgsFilter(attr, value)` on the map scene to modify active map music.

## Shaders

Functions:

- `Manager.loadShader(shaderPath, shaderType=None)`
- `Manager.loadFullShaderWithGeo(vertPath, geoPath, fragPath)`
- `Manager.loadGeoShader(vertPath, geoPath, fragPath)`

Shaders load from `Assets/Shaders`. On iOS, shader features may be skipped and return `None`.

Post-processing:

```python
shader = Manager.loadShader("Flash.frag")
Global.System.addGraphicsShader(shader)
Global.System.removeGraphicsShader(shader)
Global.System.removeAllGraphicsShaders()
```

## Transitions

`Global.System.setTransition(transitionResource=None, transitionTime=1.0)` starts a transition.

```python
System.setTransition(Manager.loadTransition("012-Random04.png"), 0.5)
```

`System.freezeTransitionBackground()` freezes the current frame for the next transition background.

## Flash and Shake

Screen flash:

```python
from Engine import Color

System.flashScreen(Color(255, 255, 255, 200), 0.4)
```

Screen shake:

```python
System.startShake(power=4.0, speed=10.0, duration=0.5)
```

Useful checks:

- `System.isFlashing()`
- `System.stopFlash()`
- `System.isShaking()`
- `System.stopShake()`

## Weather

`System.setWeather(weatherType, power, maxCount)` starts weather. `weatherType` may be a weather enum, a name, or a supported label.

```python
System.setWeather("Rain", 60, 80)
System.clearWeather()
```

Weather is updated and drawn by `Global.System` and `GameMap`.

## Fog

Map fog is applied from map data:

```python
System.applyFogFromMapData(mapData)
System.clearFog()
```

The map scene clears and reapplies fog whenever a map loads.

## Video

Use `Global.playVideo(videoPath, mute=False, skipable=False)` for blocking video playback.

```python
from Global import playVideo

playVideo("./Assets/System/Opening.mp4", skipable=True)
```

Video playback depends on the PyAV package, imported as `av`. The Ludork toolkit build includes `av` for its own runtime, but a packaged game project does not automatically have PyAV unless the packaging Python environment has it and the package is included in the build.

If your game calls `Global.playVideo()` or uses `Global.Video`, install PyAV in the Python environment used for packaging:

```bash
python -m pip install av
```

When using a custom Nuitka command, also make sure the build includes `av`, for example with `--include-module=av`.
