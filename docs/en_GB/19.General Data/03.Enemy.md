# 03. Enemy

The **Enemy** tab maps to `/Data/General/Enemy.dat`. It is linked to `EnemyInfo` and supports the `onDefeat` blueprint event.

## Fields

**name**, **desc**

Locale keys for the enemy book and battle UI.

**MAXHP**, **ATK**, **DEF**

Core battle stats used by `Battler` damage calculations.

**EXP**, **GOLD**

Rewards granted when the player wins a battle against this enemy.

**drops**

List of item IDs dropped on defeat. The Sample records leave this empty; entries are plain strings matching `/Data/General/Item.dat` member keys.

**special**

Dictionary mapping a **Special** record ID to an optional value string. Presence of the key enables that special; the value is passed into battle logic when a numeric parameter is needed.

Examples from the Sample set:

| Enemy | Special entry | Effect |
|-------|---------------|--------|
| `Skeleton` | `Poisoning: "1"` | Applies poisoning behaviour tied to the `Poisoning` special. |
| `Slime` | `Weaken: "2"` | Applies weaken with stack value `2`. |
| `Rock` | `Hard: ""` | Raises effective defence during combat. |
| `Mage` | `Magic: ""` | Attacks ignore defender defence. |
| `Knight` | `Compete: ""`, `MultiHit: "2"` | Attack competes with the opponent's ATK; each round hits twice. |

For `MultiHit`, the value string is read as an integer hit count (default `1` when empty).

## Blueprint Events

**onDefeat**

Runs after the enemy is defeated. Use it for mechanism puzzles that open doors through the event bus—see `20.Usage Tips/03.Enemies` and the `ConditionEnemies` blueprint examples.

## Linking Map Blueprints

Enemy blueprints under `/Data/Blueprints/Enemies/` set `attrs.ID` to a member key here (for example `Slime`). Stats and text come from General Data; `texturePath` and `defaultRect` stay on the blueprint.

## Default Sample Members

`Skeleton`, `Slime`, `Rock`, `Mage`, `Bat`, `Knight`.

When adding a new enemy, copy a similar record, set a unique ID, adjust stats and `special`, then create or duplicate a map blueprint with the same `ID`.
