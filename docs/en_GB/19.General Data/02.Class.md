# 02. Class

The **Class** tab maps to `/Data/General/Class.dat`. It has no linked type and no blueprint events; it defines player class metadata used when resolving equipment slots.

## Fields

**name**

Display name locale key, for example `{WARRIOR}`.

**desc**

Description locale key.

**slot**

Dictionary of equipment slot names. Keys are slot identifiers used by equips and the equipment window; values are usually empty strings in the Sample project. The default `Warrior` record defines `weapon`, `shield`, and `accessory`.

## Default Sample

| ID | Purpose |
|----|---------|
| `Warrior` | Default player class with three equipment slots. |

## Usage Notes

- Equip records reference slot names through their `slot` field; a slot name must exist on the player's class before that equip can be worn.
- When adding a new class, copy `Warrior` and adjust `slot` keys to match the slots your equipment set expects.
- The player's runtime `CLASS` attribute (on a **Player** record) should match a member ID in this tab.
