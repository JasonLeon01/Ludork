The Sample game loads resources from project asset folders and applies audio and visual effects through the runtime systems.

## Images

**System images**

Icon, cursor, title background, and window skin.

**Character images**

Player, enemies, doors, stairs, pickups, and NPCs.

**Icons**

Items, equipment, and states.

When replacing actor images, test default crop, animation frames, collision size, and map alignment.

## Fonts

The default font is `HarmonyOS_Sans_SC_Regular.ttf`. After replacing fonts, test English, Chinese, numbers, item descriptions, save details, and dialogue.

## Audio

The audio config separates common UI sounds from map feedback sounds and title music. Node functions can play sound, voice dialogue, or music from events:

```python
from Source.NodeFunctions.Scene import ShowVoiceMessageByTag
from Source.NodeFunctions.System import PlaySound, PlayMusic

PlaySound("item1.ogg", True)
PlayMusic("To Walk the Unseen Path.mp3", True)
ShowVoiceMessageByTag("Guide", "Stay close and listen.", "guide_01.ogg", refActorTag="GuideNPC")
```

Sound effects are loaded from `Assets/Sounds`, voice clips from `Assets/Voices`, and music from `Assets/Musics`. Voice clips use the voice switch and voice volume settings. Voice is a single dialogue slot: starting a new voice clip stops and replaces the previous one. `ShowVoiceMessageByTag` uses `refActorTag` to position the dialogue window; use `ShowVoiceMessage` when the voice should also be spatially anchored to an actor.

## Filters

BGM and BGS filters can adjust offset, pitch, pan, volume, and loop point. Use filter editing for map ambience changes rather than replacing the original file for every variation.

## Visual Effects

Transitions, flash, shake, weather, fog, lighting, animations, and shaders all affect readability.

```python
from Global.Weather import WeatherType
from Source.NodeFunctions.System import FlashScreen, ChangeScreenTone, ClearScreenTone, ScreenShake, SetWeather

FlashScreen(255, 255, 255, 160, 0.2)
ChangeScreenTone(-68, -68, 0, 0, 1.0)
ClearScreenTone(0.5)
ScreenShake(4.0, 8.0, 0.5)
SetWeather(WeatherType.RAIN, 40, 80)
```

`ChangeScreenTone` uses RMXP-style parameters: red, green, and blue range from `-255` to `255`, gray ranges from `0` to `255`, and `duration` is the number of seconds spent fading to the target tone. `ClearScreenTone` fades back to the normal tone over the given duration.

## Animation Data

Animations combine image assets, audio assets, timing, position, rotation, scale, and horizontal flip (`flipX`). Press `F7` to open `Animation Overview` for editing; double-click an animation file in the file explorer to open the standalone animation window. After editing animation data, test it where it is actually triggered, not only in the editor preview.
