# 04. Tilesets, Autotiles, and Maps

This page covers map-authoring work in the editor. It explains what to change and how to verify it, without requiring runtime code knowledge.

## Tileset Data

Open `Tilesets Data` from the database menu, or press `F8`.

**Tileset image**

Choose the source image for regular tiles. The default Sample tileset is `Tileset_01.dat`, using `Assets/Tilesets/mota.png`.

**Passability**

Passability decides whether the player, enemies, and click pathing can enter a tile. After changing passability, test movement from several directions.

**Four-direction passability**

Directional passability blocks only selected directions. Use it for one-way edges, ledges, narrow gates, and tiles whose collision does not match a simple square.

**Material**

Material settings affect render and movement properties such as lighting, reflection, opacity, blocking, and speed. Use the map preview and a test game to confirm that the result is readable.

## Autotiles

Autotiles are stored separately from regular tilesets. The default Sample project includes `grass.dat`, `lava.dat`, and `galaxy.dat`.

When replacing an autotile image, open a map that uses it and verify edge stitching on corners, islands, and long borders.

## Map Editing Modes

**Tile Mode**

Paint terrain, layers, and autotiles. Use this for map shape and navigation.

**Actor Mode**

Place blueprints such as doors, enemies, stairs, items, NPCs, and test actors.

**Light Mode**

Place light sources used by the map lighting system.

## Map Properties

Use the map edit dialog to change map name, size, ambient light, BGM, BGS, and fog settings. Save and test the map after changing audio or fog because these are easiest to judge in motion.

## Start Point

To change the opening position, first make the target tile passable, then update the start map and start position in `System Config` (`F6`).

## Actor Tags

Actors placed on a map can have tags. Use stable readable tags for doors, stairs, NPCs, chests, shops, and special triggers. Tags are used by events, saves, floor teleporting, and window previews.
