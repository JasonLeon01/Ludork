# 04. Tilesets, Autotiles, and Maps

This chapter explains how to maintain tilesets, autotiles, and map content in the editor.

## Tilesets Data

`Tilesets Data` maintains `Data/Tilesets`. The current template tileset is `Tileset_01.dat`, linked to `Assets/Tilesets/mota.png`.

A tileset can store:

- Tileset image file.
- Tile passability.
- Four-direction passability.
- Material data, such as light blocking, mirror state, reflection strength, opacity, and movement speed rate.

Passability and four-direction passability affect actor movement, collision, and pathfinding. Material data affects map rendering.

## Autotiles

Autotile data is stored in `Data/AutoTiles`. The current template includes `grass.dat`, `lava.dat`, and `galaxy.dat`.

When creating or adjusting an autotile, put its image in `Assets/Autotiles` and keep the fixed autotile image layout required by the tool. While painting maps, normal tiles and autotiles can both be used as terrain on the same layer.

## Map List

Maps are stored in `Data/Maps`. The current template contains `Map_01` and `Map_02`.

After opening a map, switch modes based on what you want to edit:

- `Tile Mode`: paint normal tiles or autotiles, and maintain layers and terrain.
- `Actor Mode`: place blueprint instances, then set position, tag, and instance properties.
- `Light Source Mode`: place map lights, used together with ambient light and self-lit actors.

## Layers and Terrain

Maps can contain multiple layers. Before painting terrain, select the target layer, then select either a normal tile or an autotile.

One cell on one layer should keep only one terrain value: either a normal tile or an autotile. Use erase or an empty selection when you need to clear terrain.

## Start Map and Start Position

At runtime, a project starts from the `startMap` and `startPos` values in `Data/Configs/System.json`.

To change the new-game start point, edit those fields in `System Config` and make sure the target map has been saved. `startPos` uses map-cell coordinates, not screen pixels.

## Map Actor Tags

Actor tags on maps are important. Dialogues, lookups, event positioning, and some node functions use tags to refer to actors in the map.

Give dialogue objects, doors, stairs, and special triggers stable, readable tags. Do not treat tags as display names; they are closer to internal event anchors.

## Lights

`Light Source Mode` places map lights. Blueprint `lightComp` settings make an actor emit its own light.

If a map uses ambient light, map lights, and self-lit actors together, test dark areas, bright areas, moving actors, and window UI readability.
