# 04. Tilesets, Autotiles, and Maps

This chapter covers Ludork default template tilesets, autotiles, and maps.

## Tilesets

The current regular tileset is `Tileset_01.dat`, using `Assets/Tilesets/mota.png`. Tileset data controls passability, four-direction passability, and material appearance.

Passability affects player movement, enemies, and click pathing. Material settings affect lighting, blocking, reflection, opacity, and movement speed.

## Autotiles

The Ludork default template includes `grass.dat`, `lava.dat`, and `galaxy.dat`. Their images are stored in `Assets/Autotiles`.

Regular tiles and autotiles can be painted on the same map layer. After changing an autotile, open maps that use it and verify edge stitching.

## Maps

Maps are stored in `Data/Maps`. The Ludork default template includes `Map_01.dat` and `Map_02.dat`.

Switch editing modes based on the task:

- `Tile Mode`: paint terrain, layers, and autotiles.
- `Actor Mode`: place doors, enemies, stairs, items, and test objects.
- `Light Mode`: place map lights for use with ambient light and glowing blueprints.

## Start Point

New games start on `Map_01.dat` at `(6, 12)`. To change the opening position, first confirm the target cell is passable, then update the start map and start position in System Config.

## Tags

Actor tags on maps are used by dialogue placement, event targets, save records, and floor teleporting. Give doors, stairs, NPCs, chests, and special triggers stable readable tags.
