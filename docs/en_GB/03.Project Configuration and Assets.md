# 03. Project Configuration and Assets

This chapter explains how to maintain project configuration and resource files in the editor. Configuration controls how a project starts, how the window appears, and which default resources are loaded.

## Game Settings

`Game Settings` maintains the runtime options stored in `Main.ini`. Common fields include:

- Entry script, usually `Entry.py`.
- Language, such as `en_GB` or `zh_CN`.
- Frame-rate limit and vertical sync.
- Music, sound, and voice switches.
- Music, sound, and voice volumes.

These settings are read early during runtime startup. After changing them, save the project and test again.

## System Config

`System Config` maintains `Data/Configs/System.json`. Common fields include:

- `title`: window title.
- `gameSize`: logical game resolution.
- `fonts` and `fontSize`: font files and default font size.
- `icon`: window icon.
- `cursor`: runtime cursor image.
- `windowskinName`: window skin.
- `cellSize`: tile size.
- `startMap`: new-game start map.
- `startPos`: new-game start position.

File fields select resources from the matching `Assets` or `Data` folder. Prefer the file selector over manually typing paths.

## Audio Config

Audio configuration maintains `Data/Configs/Audio.json`. The template already defines cursor, decision, cancel, shop, save, load, gate, stair, item pickup, and equipment sounds, plus the title music.

When replacing audio, put files in `Assets/Sounds` or `Assets/Musics`, then reselect the matching field. Common audio formats include `.ogg`, `.wav`, `.mp3`, `.aac`, and `.flac`.

## Asset Folders

Common asset folders:

- `Assets/System`: icons, cursor, window skin, and system images.
- `Assets/Tilesets`: normal tileset images.
- `Assets/Autotiles`: autotile images.
- `Assets/Characters`: actor-related images.
- `Assets/Icons`: item, equipment, and state icons.
- `Assets/Sounds`: sound effects.
- `Assets/Musics`: music.
- `Assets/Shaders`: shaders used by map materials, weather, fog, transitions, flashes, and other effects.
- `Assets/Transitions`, `Assets/Fogs`, and `Assets/Animations`: transition, fog, and animation resources.

Resource fields in configuration, blueprints, and general data usually store paths relative to `Assets`, such as `System/icon.png` or `Sounds/cursor.ogg`.

## Adding, Renaming, and Deleting

After adding an asset, reselect the related field or save the project so configurations, blueprints, general data, or maps reference the latest file name.

Before renaming or deleting assets, check whether project data still references them. Resource paths are part of the data; if a path changes but references are not updated, the runtime will not find the file.
