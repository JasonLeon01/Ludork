# 07. Ludork Default Template Content

This page describes the structure of the default Sample game. Unlike the editor chapters, this page may mention project files because the Sample game reads them directly.

## Project Root

**Main.proj**

Marks the folder as a Ludork project.

**Main.ini**

Stores startup and user-facing game settings:

```ini
[Main]
script = Entry.py
language = zh_CN
scale = 2.0
framerate = 120
verticalsync = True
```

**Entry.py**

Starts the Sample game runtime.

## Data Folder

**Maps**

`Data/Maps` contains map files such as `Map_01.dat` and `Map_02.dat`.

**Tilesets and AutoTiles**

Terrain definitions used by maps.

**Blueprints**

Actor templates grouped by purpose.

**General**

Class, enemy, equipment, item, special, and state records.

**Configs**

System and audio configuration files.

**Locale**

The workbook and exported language files.

**Animations**

Animation data read by attack and event effects.

**CommonFunctions**

Reusable visual event graphs.

## Assets Folder

The Sample game loads images, audio, fonts, fogs, transitions, animation assets, and shaders from `Assets`. Data files usually store paths relative to an asset category.

## Save Folder

`Save` stores player progress. The Sample game records player state, map state, removed actors, changed terrain, floor points, and save screenshots.

## Runtime Rule

The editor is the preferred way to maintain project data. When the Sample game side is customised by code, keep data names, tags, and localisation keys stable so existing maps and saves still make sense.
