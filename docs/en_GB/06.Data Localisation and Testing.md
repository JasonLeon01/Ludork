# 06. Data, Localisation, and Testing

This page covers editor-side data maintenance. It explains how to use the data tools and how to test the result from the project author's point of view.

## General Data

Open `General Data` from the database menu.

**Classes**

Define player class data such as equipment slots.

**Enemies**

Define enemy display text, battle values, rewards, drops, and specials.

**Equipment**

Define slots, icons, prices, descriptions, and stat bonuses.

**Items**

Define item display text, icons, prices, usability, inventory cost, and related behaviour fields.

**Specials and states**

Define reusable battle flags and state records used by enemies and battlers.

When adding new records, copy a similar existing record first. This keeps required fields present and makes comparison easier.

## Localisation Table

Open the in-app localisation editor or the locale workbook.

**Keys**

Use stable keys for names, descriptions, menu text, and tips.

**Languages**

Keep `zh_CN` and `en_GB` filled for every user-facing key.

**Placeholders**

If one language contains a placeholder such as `{name}`, every language must contain the same placeholder name.

**Export**

Export the locale files after editing. The test game reads the exported files, not only the workbook view.

## Animation Data

Open animation data through the animation editor. Check image, sound, timing, scale, and placement in the preview, then verify the animation in the test game.

## Basic Test Flow

1. Save all edited panels.
2. Export localisation when text changed.
3. Start `Test Game`.
4. Begin a new game or load a relevant save.
5. Trigger the map, actor, item, window, or setting that was changed.
6. Switch language and restart the test game when checking text or scale.
