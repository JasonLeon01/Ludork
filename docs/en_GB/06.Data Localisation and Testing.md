# 06. Data, Localisation, and Testing

This chapter covers Ludork default template general data, localisation, and basic test flow.

## General Data

General data is stored in `Data/General`. The Ludork default template includes job, enemy, equipment, item, special, and state data.

- Job data controls default equipment slots.
- Enemy data controls name, description, HP, attack, defence, EXP, gold, drops, and specials.
- Equipment data controls slot, icon, price, and value bonuses.
- Item data controls name, description, icon, price, usability, and use cost.
- Special and state data support enemy abilities, player states, and battle behavior.

## Localisation

The editable localisation workbook is `Data/Locale/Locale.xlsx`. Exported language files are `en_GB` and `zh_CN` in the same folder.

General data, blueprints, and window text can use text keys. After changing a key, check that both English and Chinese columns contain matching text. Text with placeholders such as `{name}` must keep the same placeholder names in every language.

## Animations

Animation data is stored in `Data/Animations`. The Ludork default template includes attack-related animation data and uses images and sounds from `Assets/Animations`. After replacing animation assets, test alignment, duration, sound timing, and scale.

## Test Checklist

- Save all edited content.
- Use `Test Game` to reach the title screen.
- Start a new game and check map, player position, and language.
- Trigger doors, stairs, items, enemies, menus, and saves.
- Switch language and confirm text is not missing and placeholders are correct.
