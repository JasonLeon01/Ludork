# 06. Data, Localisation, and Testing

This page covers editor-side data maintenance. It explains how to use the data tools and how to test the result from the project author's point of view.

## Localisation Table

Game localisation is maintained through the official locale tools plugin at `Plugins/OfficialLocaleTools`. The plugin is not part of the engine core; it is shipped as a reference plugin that adds database-menu commands, an in-editor locale table, and export tools.

**Open the table**

Use the locale-table command from the database menu, or open `Data/Locale/Locale.xlsx` directly when you prefer spreadsheet editing.

**Export**

After editing, run the export command from the database menu so `Data/Locale` language files are refreshed. The test game reads the exported files, not only the workbook or plugin view.

**Keys**

Use stable keys for names, descriptions, menu text, and tips.

**Languages**

Keep `zh_CN` and `en_GB` filled for every user-facing key.

**Placeholders**

If one language contains a placeholder such as `{name}`, every language must contain the same placeholder name.

For plugin structure, menu hooks, and inline `{LOCALE_KEY}` hint preview, see `23.Plugin System`.

## Animation Data

Press `F7` to open `Animation Overview`: the left side lists project animations, and the right side embeds the editor. Double-clicking an animation file in the file explorer still opens the standalone animation window.

Check image, sound, timing, scale, horizontal flip (`flipX`), and placement in the preview, then verify the animation in the test game.

## Basic Test Flow

1. Save all edited panels.
2. Export localisation when text changed.
3. Start `Test Game`.
4. Begin a new game or load a relevant save.
5. Trigger the map, actor, item, window, or setting that was changed.
6. Switch language and restart the test game when checking text or scale.
