# 09. Scenes

Scenes are the main units of game flow. Title screens, map gameplay, menus that take over the whole screen, and loading screens are all good scene candidates.

## SceneBase

All scenes should inherit from `Global.SceneBase`.

```python
from Global import SceneBase

class Scene(SceneBase):
    def onCreate(self) -> None:
        pass
```

`SceneBase` supplies lifecycle hooks, a UI manager, timers, animation handling, common tips, and the internal scene loop.

## Lifecycle

Override these methods:

- `onCreate()`: called once, before the scene first enters.
- `onEnter()`: called each time the scene becomes active.
- `onTick(deltaTime)`: logic update on the scene logic thread.
- `onFixedTick(fixedDelta)`: fixed-step update, default step is `1 / 60`.
- `onLateTick(deltaTime)`: render-thread late update before drawing is displayed.
- `onQuit()`: called when leaving the scene loop.
- `onDestroy()`: called when the scene is removed from the scene stack.

Minimal scene:

```python
from Global import SceneBase, System
from Engine import Color

class Scene(SceneBase):
    def onEnter(self) -> None:
        System.setTransition(transitionTime=0.5)

    def onTick(self, deltaTime: float) -> None:
        if someCondition:
            System.flashScreen(Color.White, 0.25)
```

## Scene Stack

`Global.System` manages the active scene stack.

Important methods:

- `System.setScene(scene)`: replace the current scene.
- `System.pushScene(scene)`: push a scene on top.
- `System.popScene()`: remove the top scene.
- `System.getScene()`: get the current scene.
- `System.getSceneList()`: get the full scene stack.
- `System.exit()`: remove all scenes and end the game loop.

Example:

```python
from Global import System
from Source.Scenes.SceneTitle import Scene as SceneTitle

System.setScene(SceneTitle())
```

## Timers

`SceneBase.addTimer(key, interval, task=None, params=None)` adds a one-shot timer and returns a condition function.

```python
def showReady(self) -> None:
    self.addCommonTip("Ready")

self.addTimer("ready-tip", 1.0, self.showReady)
```

The method is latent-aware, so node graph logic can wait until the timer expires.

## Scene Animations

Scenes can own `Global.Animation` instances.

Methods:

- `addAnim(anim)`
- `getAnims()`
- `removeAnim(anim)`
- `clearAnims()`

Scene animations are updated by `SceneBase` and drawn automatically.

## Common Tips

`addCommonTip(text)` displays a floating tip overlay in the current scene.

```python
self.addCommonTip(LOC("ITEM_GET"))
```

`GameMap.addCommonTip(text)` forwards the call to its parent scene.

## UI Manager

Every scene owns `self._uiManager`. Use it to load controls and windows:

```python
self._uiManager.loadUI(self._windowCommand)
```

The template title scene loads its background, command window, save/load windows, and config window in `onCreate()`.

## Title Scene Pattern

The template title scene:

1. Starts a transition in `onEnter()`.
2. Creates background and command UI in `onCreate()`.
3. Plays title BGM.
4. Starts a new `GameInstance()` when Start is selected.
5. Switches to `SceneMap`.
6. Pauses the title command window while save/load or configuration windows are open.
7. Stops title BGM in `onQuit()` and `onDestroy()`.

Scene switching example:

```python
from Global import System, Manager
from Source.GameInstance import GameInstance
from Source.Scenes.SceneMap import Scene as SceneMap

Manager.stopMusic("BGM")
nextScene = SceneMap()
nextScene.setInst(GameInstance())
System.setScene(nextScene)
```

## Game Over Scene

The template project's `Source.Scenes.GameOver` is a full-screen scene:

- It starts a long transition when entered.
- It draws a black background and centred `GAME OVER` text.
- Confirm input or left click returns to the title scene.

Enemy battle failure and the node function `Scene.GameOver()` both switch to this scene.

## Map Scene Pattern

The template map scene owns the player, HUD, message window, menu, save/load window, item/equipment windows, shop window, enemy book, floor teleporter, and current `GameMap`.

Important methods:

- `setInst(inst)`: assign the active `GameInstance`.
- `loadMap(mapPath)`: load map data and generate a `GameMap`.
- `gotoMapAndPos(mapPath, pos=None, *, blockTransition)`: switch map, place the player, and run the map transition unless `blockTransition` is `True`.
- `showMessage(refActorTag, name, message)`: display dialogue.
- `showSelection(refActorTag, name, options, allowCancel)`: display a choice list.
- `openShop(items, canSell)`: open a shop and return a close-condition callable.
- `showEnemyBook()`: open the current-map enemy book.
- `showFloorTeleporter()`: open the visited-floor teleporter window.
- `recordDestroyedActor(actor)`: persist destroyed actor tags.
- `applyLoadedGame(inst)`: rebind player, UI, and map state after loading a save.

Use the map scene as the main reference when adding gameplay scenes.
