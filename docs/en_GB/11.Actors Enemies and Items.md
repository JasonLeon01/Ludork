# 11. Actors, Enemies, and Items

This page describes the default RPG objects in the Sample game and the runtime concepts behind them.

## Player

The default player actor uses battle attributes such as `MAXHP`, `HP`, `ATK`, `DEF`, `EXP`, `GOLD`, `LEVEL`, and `CLASS`. It also owns inventory, equipment, states, and recorded floor points.

Node functions can modify the player from visual events:

```python
from Source.NodeFunctions.Player import AddItem, AddGold, HealPlayer

AddItem("Key_Y", 1)
AddGold(10)
HealPlayer(100)
```

## Enemies

Enemy records provide name, description, HP, attack, defence, EXP, gold, drops, and specials. Enemy blueprints place those records on the map.

The default enemy set includes slime, skeleton soldier, rock, mage, bat, and knight records.

## Battle Specials

**Hard**

Raises effective defence against an attacker.

**Magic**

Ignores defender defence when calculating the attacker's damage.

**MultiHit**

Multiplies damage per attack round by the configured hit count.

**Compete**

Raises effective attack to at least the opponent's attack.

The battle seam is the `Battler` helper. Custom battle code should use `getATK`, `getDEF`, `getDamagePerRound`, and `getDamage` instead of duplicating the formula.

```python
damageType, damage = enemy.getDamage(player)
roundDamage = player.getDamagePerRound(enemy)
```

## Doors and Keys

Yellow, blue, and red keys match coloured doors. A door should play feedback, check cost or conditions, open visually, update passability, and record removal when it should stay open after loading.

## Items and Equipment

Consumables usually change values immediately. Utility items open windows or alter terrain. Equipment changes player values through slots and should be checked in the equipment window, HUD, enemy book, and battle result.
