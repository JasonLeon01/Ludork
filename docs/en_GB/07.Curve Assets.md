# 07. Curve Assets

Curve assets store keyed interpolation data for fades, motion, opacity, and other time-varying values. They live under `Data/Curves` and are edited in the curve window, then sampled at runtime through `Engine.Curve`.

## Creating and Opening Curves

**New curve**

Choose `New Curve` from the database menu, or press `F11`. Pick a path under `Data/Curves`. The editor creates a JSON or `.dat` file with two default keys.

**Open an existing curve**

Double-click a curve file in the file explorer, or open it from the project tree under `Data/Curves`.

**Save**

Curve edits follow the normal project save flow. Unsaved curve changes are not visible to `Test Game`.

## Curve Window

The curve window has three main areas:

**Asset properties**

- `name`: display name stored in the asset.
- `defaultValue`: value returned when the curve has no keys.
- `preInfinity` and `postInfinity`: extrapolation before the first key and after the last key. Supported modes are `constant` and `linear`.

**Canvas**

The horizontal axis is time; the vertical axis is value. Use the canvas to add, move, and select keys. Cubic segments expose arrive and leave tangent handles.

**Key inspector**

When a key is selected, edit `time`, `value`, `interpolation`, `arriveTangent`, and `leaveTangent`.

Supported interpolation modes are:

| Mode | Behaviour |
| --- | --- |
| `constant` | Holds the start key value until the next key time. |
| `linear` | Interpolates linearly between values. |
| `cubic` | Uses Hermite tangents for smooth segments. |

Use `Fit View` to frame all keys. Select a key and press `Delete` to remove it.

## File Layout

Curve files are stored under `Data/Curves`. Subfolders are allowed; the runtime key is the path relative to `Data/Curves` without the extension.

Example keys from the default Sample project:

| Key | Purpose |
| --- | --- |
| `UI/SelectionRectOpacity` | Selection rectangle opacity pulse |
| `WindowMessageFadeIn` | Message window fade-in |
| `WindowMessageFadeOut` | Message window fade-out |
| `Global/CommonTipAlphaIn` | Common tip fade-in |
| `Global/CommonTipAlphaOut` | Common tip fade-out |
| `Global/CommonTipRise` | Common tip upward motion |
| `Global/DamageTextSpeed` | Floating damage text motion |

A minimal JSON curve looks like this:

```json
{
    "type": "curve",
    "name": "SelectionRectOpacity",
    "defaultValue": 255.0,
    "preInfinity": "constant",
    "postInfinity": "constant",
    "keys": [
        {
            "time": 0.0,
            "value": 255.0,
            "interpolation": "linear",
            "arriveTangent": 0.0,
            "leaveTangent": 0.0
        }
    ]
}
```

## Runtime Usage

Curves are loaded from `Data/Curves` during game startup. Sample code resolves them by key:

```python
from Engine import Curve
from Source import Data

curve = Data.getCurve("UI/SelectionRectOpacity")
opacity = curve.evaluate(elapsedTime)
duration = curve.getDuration()
```

`Curve.evaluate(time)` samples the keyed data. `getDuration()` returns the span between the first and last key. When a curve has no keys, `evaluate` returns `defaultValue`.

The default Sample project already uses curves for UI fades, common tips, selection rectangles, and damage text. Search for `getCurve` under `Sample/` when tracing a specific effect.

## Authoring Checks

1. Save the project after editing curves.
2. Start `Test Game` and trigger the UI or effect that references the curve key.
3. If timing feels wrong, adjust key times first, then interpolation and tangents.
4. When renaming or moving a curve file, update every runtime key reference and save again.
