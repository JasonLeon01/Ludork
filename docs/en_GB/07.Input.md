# 07. Input

`Engine.Input` wraps keyboard, mouse, touch, and joystick events. It is updated once per frame by the scene loop:

```python
Input.update(System.getWindow())
```

Game scripts usually read input in `onTick()` or window update methods.

## Key Names

Common aliases:

```python
from Engine import Input

Key = Input.Key
Scan = Input.Scan
Mouse = Input.Mouse
JoystickButton = Input.JoystickButton
```

Use `Key` for logical keys and `Scan` for physical scan codes.

## Action Helpers

Default action key lists:

- `Input.getConfirmKeys()`
- `Input.getCancelKeys()`
- `Input.getUpKeys()`
- `Input.getDownKeys()`
- `Input.getLeftKeys()`
- `Input.getRightKeys()`

Example:

```python
if Input.isActionTriggered(Input.getCancelKeys(), handled=True):
    self._windowMenu.open()
```

`handled=True` consumes the trigger so another system does not act on the same press.

## Keyboard

Important functions:

- `isKeyTriggered(key, alt=False, ctrl=False, shift=False, system=False, handled=False)`
- `isTextEntered()`
- `getEnteredText()`
- `blockKeyboard()`
- `unblockKeyboard()`
- `isKeyboardBlocked()`

Example:

```python
if Input.isKeyTriggered(Input.Key.F5, handled=True):
    self.quickSave()
```

## Mouse

Important functions:

- `isMouseButtonTriggered(button, handled=False)`
- `isMouseBlocked()`
- `blockMouse()`
- `unblockMouse()`

Common button usage:

```python
if Input.isMouseButtonTriggered(Input.Mouse.Button.Right, handled=True):
    self._windowMenu.open()
```

The sample map scene uses right-click to open the menu and left-click/touch for map click movement.

## Touch

Important functions:

- `isTouchBegan(handled=False)`
- `isTouchEnded()`
- `isTouchMoved()`
- `isTouchActive()`
- `isTouchTriggered(handled=False)`
- `getTouchPosition()`
- `getTouchBeganPosition()`
- `getTouchEndedPosition()`
- `getTouchMovedDelta()`
- `blockTouch()`
- `unblockTouch()`

Touch state tracks the primary finger.

## Joystick

Important functions:

- `isAnyJoystickButtonTriggered(button, handled=False)`
- `getJoystickButtonPressed(joystickId, button, handled)`
- `getJoystickButtonReleased(joystickId, button, handled)`
- `isJoystickAxisMoved()`
- `getJoystickAxisMoved(joystickId, handled)`
- `isJoystickConnected()`
- `isJoystickDisconnected()`

Default confirm/cancel/navigation helpers include joystick buttons and axes.

## Blocking All Input

Use:

```python
Input.blockInput()
Input.unblockInput()
```

This blocks or unblocks keyboard, mouse, joystick, and touch together.

## Action Mappings

`registerActionMapping(obj, actionName, actionKeys, callable_, triggerOnHold=False)` registers a callback. `unregisterActionMapping(obj, actionName)` removes it.

Example:

```python
Input.registerActionMapping(
    self,
    "open_menu",
    Input.getCancelKeys(),
    lambda obj, delta: obj.openMenu(),
)
```

Use direct polling for simple scene code and action mappings for reusable controls.

## Trigger, Hold, and Repeat

`isActionTriggered()` and `isKeyTriggered()` support three trigger modes, controlled by
the call parameters:

| Mode | Parameters | Behaviour |
|---|---|---|
| **Trigger** | `repeatInterval=0` (default) | Fires once on the first press. Remains consumed until the key is released. |
| **Hold** | `triggerOnHold=True` (in `registerActionMapping` only) | Fires every frame while the key is held. |
| **Repeat** | `repeatDelay > 0` and `repeatInterval > 0` | Fires once on the first press, then after `repeatDelay` seconds fires every `repeatInterval` seconds while held. |

### Repeat Mode in Detail

Add two keyword arguments to `isActionTriggered`:

- `repeatDelay` — seconds to wait after the first press before repeat starts.
- `repeatInterval` — seconds between successive repeat fires.

Typical values for menu navigation:

```python
_REPEAT_DELAY = 0.4   # wait 0.4 s before repeating
_REPEAT_INTERVAL = 0.1  # fire every 0.1 s afterwards

if Input.isActionTriggered(
    Input.getUpKeys(),
    handled=True,
    repeatDelay=_REPEAT_DELAY,
    repeatInterval=_REPEAT_INTERVAL,
):
    self.index = (self.index - 1) % itemCount
```

Faster repeat (e.g. for slider adjustment):

```python
if Input.isActionTriggered(
    Input.getLeftKeys(),
    handled=True,
    repeatDelay=0.4,
    repeatInterval=0.05,  # twice as fast as menu navigation
):
    slider.adjust(-1)
```

### When to Use Each Mode

- **Trigger** — one-shot actions: confirm, cancel, open menu, toggle.
- **Hold** — continuous input: player movement, camera panning.
- **Repeat** — list navigation: menus, selectable windows, slider bars.

The sample `WindowSelectable` uses Repeat for all four direction keys,
and `ConfigWindow` uses a faster Repeat for slider adjustment.
All windows inheriting from `WindowSelectable` get repeat navigation
automatically through `super().onKeyDown()`.

## Joystick Axis Navigation

Direction helpers (`getUpKeys()`, `getDownKeys()`, `getLeftKeys()`, `getRightKeys()`)
include joystick axis tuples:

```python
# getUpKeys() includes:
(Joystick.Axis.Y, -10.0, float.__lt__),   # left stick up
(Joystick.Axis.PovY, 50.0, float.__gt__),  # d-pad up
```

Each tuple is `(axis, threshold, comparator)`. The axis fires when its
value satisfies the comparator against the threshold. Navigation axes
use a 10-unit deadzone; POV (d-pad) axes use a 50-unit threshold.

Repeat mode also works for joystick buttons. Joystick axes use a
separate "just pressed" mechanism and do not participate in repeat
timing (they already fire per-frame while tilted).
