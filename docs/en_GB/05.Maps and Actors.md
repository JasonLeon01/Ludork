# 05. Maps and Actors

This chapter describes the runtime map and actor model. It does not cover how map data is authored.

## GameMap

`Global.GameMap` represents a loaded map. It owns tile layers, actors, lights, particles, camera, passability cache, and pathfinding helpers.

Common methods:

- `getPlayer()`
- `setPlayer(player)`
- `getAllActors()`
- `getActorByTag(tag)`
- `getActorsByPosition(position)`
- `getActorsByRange(position, radius)`
- `spawnActor(actor, layer)`
- `destroyActor(actor)`
- `isPassable(actor, targetPosition)`
- `findPath(start, goal)`
- `getCamera()`
- `getTilemap()`
- `getSize()`
- `getTerrainTile(layerName, position)`
- `getTerrainTilePositions(layerName, tileID)`
- `setTerrainTile(layerName, position, tileID)`
- `setTerrainTiles(layerName, positions, tileID)`
- `destroyTerrain(layerName, position, tileID)`
- `destroyTerrainList(layerName, positions, tileID)`
- `addLight(light)`
- `setAmbientLight(color)`
- `addCommonTip(text)`

Get the current map from the sample map scene:

```python
from Global import System
from Source.Scenes.SceneMap import Scene as SceneMap

scene = Cast(SceneMap, System.getScene())
gameMap = scene.getGameMap()
```

## Loading a Map

The sample map scene loads data from `Data/Maps`:

```python
scene.gotoMapAndPos("Map_01.dat", (3, 5), blockTransition=False)
```

`gotoMapAndPos()` reloads the map when the file changes, stores map state in `GameInstance`, and runs the map transition unless the third argument is `True`.

## Actors

`Engine.Gameplay.Actors.Actor` is the base class for map actors.

Class variables often set by actor subclasses:

- `collisionEnabled`: whether the actor blocks movement.
- `tickable`: whether tick events are dispatched.
- `speed`: movement speed in pixels per second.
- `texturePath`: character texture path.
- `defaultRect`: texture rectangle.
- `defaultTranslation`, `defaultRotation`, `defaultScale`, `defaultOrigin`: transform defaults.

Basic movement methods:

- `MapMove(offset)`: move one tile by an offset such as `(1, 0)`.
- `setRoute(routine)`: assign a list of tile offsets to follow.
- `getRoute()`
- `isMoving()`
- `isInRoute()`
- `stop()`
- `setMoveEnabled(enabled)`
- `getMoveEnabled()`

Example:

```python
from Engine import Vector2i

player.MapMove(Vector2i(1, 0))
player.setRoute([Vector2i(0, 1), Vector2i(1, 0)])
```

## Actor Events

Actors define blueprint-style events:

- `onCreate()`
- `onTick(deltaTime)`
- `onLateTick(deltaTime)`
- `onFixedTick(fixedDelta)`
- `onDestroy()`
- `onCollision(other)`
- `onOverlap(other)`

Enable `tickable` when the actor should receive per-frame events.

```python
class Chest(Actor):
    collisionEnabled = True
    tickable = False

    def onOverlap(self, other: list[Actor]) -> None:
        self.destroy()
```

## Collision and Overlap

When an actor with collision enabled tries to move:

1. The target tile is checked against map boundaries.
2. Tile passability and four-direction passability are checked.
3. Occupying collision-enabled actors are checked.
4. `onCollision` is sent when movement is blocked by another actor.
5. `onOverlap` is sent after movement completes into a shared cell.

`GameMap.getCollision(actor, targetPosition)` and `GameMap.getOverlaps(actor)` expose these checks.

## Terrain Reads and Writes

The current `GameMap` can read and replace terrain tiles on tile layers:

```python
tile = gameMap.getTerrainTile("Ground", (6, 12))
gameMap.setTerrainTile("Ground", (6, 12), None)
```

`tileID` may be a normal tile ID, an autotile key, or `None`. `setTerrainTile()` only changes the current in-memory map; `destroyTerrain()` and `destroyTerrainList()` also record the change in `GameInstance`, so save files keep the terrain edits.

## Tags

Map data may assign a tag to each actor. Tags are useful for dialogue anchors and event lookup:

```python
npc = gameMap.getActorByTag("old_man")
if npc:
    npc.setVisible(False)
```

The sample `showMessage(refActorTag, name, message)` uses the tag to position the message window near an actor.

## Lights

`Global.Light` stores:

- `position`
- `color`
- `radius`
- `intensity`

Example:

```python
from Global import Light
from Engine import Vector2f, Color

light = Light(Vector2f(160, 96), Color(255, 220, 160, 255), 180.0, 1.0)
gameMap.addLight(light)
```

Use `setAmbientLight(color)` for the map's base lighting.

## Pathfinding

`findPath(start, goal)` returns a list of map positions.

```python
path = gameMap.findPath(player.getMapPosition(), targetPos)
```

For actor movement, convert positions into step offsets before assigning a route.

## Camera

Each map has a `Camera`. By default, the player is set as the camera parent when `setPlayer(player)` is called.

Use `getCamera()` when script logic needs the current view position or map-to-screen relationship.

## Battle and States

The sample player and enemies use the `Source.Battler` mixin for battle attributes. Core values live in the `infoComp` component, such as `HP`, `MAXHP`, `ATK`, `DEF`, `EXP`, and `GOLD`; the player also has `LEVEL` and `CLASS`.

States are represented by `Source.Infos.StateInfo` and attached to battlers:

- `addState(stateID)`
- `removeState(stateID)`
- `hasState(stateID)`
- `getStateIDs()`
- `triggerStateWalk()`
- `triggerStateHook(stateID)`

`StateInfo` exposes the node events `onWalk` and `onHookTriggered`. In the current sample, enemies with the `Poisoning` special add the `Poisoned` state to the player after battle; when the player finishes moving, the poisoned state's `onWalk` event is triggered.
