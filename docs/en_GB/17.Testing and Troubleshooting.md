# 17. Testing and Troubleshooting

Use this chapter to quickly check Ludork default template changes.

## Suggested Flow

1. Save the project.
2. Start `Test Game`.
3. Start a new game from the title screen.
4. Move near doors, stairs, enemies, items, and test actors, then trigger each.
5. Open item, equipment, save, enemy book, floor teleport, and config windows.
6. Save, return to title, and use Continue to load.
7. Switch language and re-enter the game to check text.

## New Content Does Not Appear

Confirm that the relevant map, blueprint, general data, system config, audio config, or localisation workbook was saved. For text changes, confirm language exports were updated.

## Missing Assets

Check that the file is in the correct `Assets` subfolder, then select it again in the data panel that references it. Path case and extension should match the real file.

## Doors or Items Do Nothing

Check that the blueprint is placed on the map, collision is enabled, the tag is correct, the player actually touches it, and item counts satisfy the condition.

## Save Problems

Test with a new save first, then load it and confirm position, items, equipment, states, terrain changes, and removed objects are restored. Do not edit `Save` files while testing.

## Audio or Effect Problems

Check volume and switches first, then confirm audio files, effect assets, and map settings. After replacing assets, restarting the test game helps rule out cached content.
