## Class `Source.Infos.StateInfo.StateInfo`

State data + logic layer. A `StateInfo` is the data + blueprint container for a battler status effect (poisoned, burning, blessed, etc). Each active state is owned by exactly one `Battler` (the host). State blueprints expose walking behaviour and one developer-triggered hook; combat resolution is handled directly by the battle system. Defines state-related blueprint events: `onWalk`, `onHookTriggered`. Independent of Actor; can be used standalone in inventory/shop UI.
