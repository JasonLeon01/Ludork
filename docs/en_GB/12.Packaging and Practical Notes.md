# 12. Packaging and Practical Notes

This chapter collects packaging requirements and common script patterns.

## Packaging a Game Project

Game project packaging compiles the project's `Entry.py` with Nuitka and includes the project runtime folders:

- `Assets`
- `Data`
- `Main.ini`
- `pysf`
- `Engine`
- `Global`
- `Source`

The repository-level `pack.bat` and `pack.sh` are for packaging the Ludork toolkit application itself. A game project package should be made from the project packaging flow, which targets the opened project's `Entry.py`.

## Python Requirement for Packaging

Testing a project from the toolkit can run the project script through `runpy`, so the player does not need to install a separate local Python environment just to test from the toolkit.

Packaging is different. The desktop pack flow uses Nuitka, and Nuitka must be launched by a real Python interpreter on the user's machine. Because of Nuitka and toolchain limitations, that Python interpreter cannot be bundled into the packaging toolchain automatically.

For desktop packaging, install local Python `3.12.0` on the packaging machine:

- Windows: the pack flow looks for `py -3.12` and requires version `3.12.0`.
- macOS: the pack flow looks for `python3.12` and requires version `3.12.0`.

If Python is missing, install it before packaging. Nuitka may be installed automatically into that Python environment, but the Python executable itself must already be available.

## Native Toolchain Requirement

Desktop packaging also needs a native compiler toolchain:

- Windows: MSVC Build Tools, providing `cl.exe`.
- macOS: Xcode command-line build tools.

iOS project generation is macOS-only and requires Xcode plus the iPhoneOS SDK.

## PyAV Requirement for Video

`Global.playVideo()` and `Global.Video` require PyAV, imported as `av`.

The toolkit's own packaged runtime includes PyAV, but a packaged game project does not receive it by default. If your game plays videos, install PyAV in the Python environment used for packaging:

```bash
python -m pip install av
```

When using a custom Nuitka command, include `av` explicitly:

```bash
python -m nuitka --include-module=av Entry.py
```

Without PyAV, the game may still package successfully, but video playback will fail when the script imports `av`.

## Start a New Game

```python
from Global import System, Manager
from Source.GameInstance import GameInstance
from Source.Scenes.SceneMap import Scene as SceneMap
from Source import System as GameSystem

Manager.playSE(GameSystem.getDecisionSE())
Manager.stopMusic("BGM")

scene = SceneMap()
scene.setInst(GameInstance())
System.setScene(scene)
```

## Move the Player

```python
from Engine import Vector2i

player = self.inst.getPlayer()
if not player.isMoving():
    player.MapMove(Vector2i(1, 0))
```

Use one-cell offsets. `MapMove()` clamps each component to `-1`, `0`, or `1`.

## Show Dialogue

```python
self.showMessage("guide", LOC("NPC_GUIDE"), LOC("WELCOME_MESSAGE"))
```

When `refActorTag` matches an actor, the message window can position itself near that actor.

## Show a Selection

```python
result = self.showSelection(
    "guide",
    LOC("NPC_GUIDE"),
    [LOC("CHOICE_YES"), LOC("CHOICE_NO")],
    True,
)
```

The result is an option index. Cancel returns `-1`.

## Change Map

```python
self.gotoMapAndPos("Map_02.dat", (4, 8))
```

If the map file is already loaded, the sample updates the saved map position without reloading all map resources.

## Find Actors by Tag

```python
actors = self.getGameMap().getAllActorsByTag("chest_01")
if actors:
    chest = actors[0]
```

Use tags for event targets, dialogue anchors, and persistent actor records.

## Destroy an Actor Permanently

```python
self.recordDestroyedActor(chest)
chest.destroy()
```

The map scene reapplies destroyed actor records the next time the map loads.

## Add a Light

```python
from Global import Light
from Engine import Vector2f, Color

gameMap = self.getGameMap()
gameMap.addLight(Light(Vector2f(128, 128), Color(255, 210, 120, 255), 160.0, 1.0))
```

## Play a Sound and Flash the Screen

```python
from Engine import Color
from Global import Manager, System
from Source import System as GameSystem

Manager.playSE(GameSystem.getGetSE())
System.flashScreen(Color(255, 255, 255, 180), 0.25)
```

## Start Weather

```python
from Global import System

System.setWeather("Rain", 50, 80)
```

Clear it when leaving the map or event:

```python
System.clearWeather()
```

## Use Input in a Scene

```python
from Engine import Input

def onTick(self, deltaTime: float) -> None:
    if Input.isActionTriggered(Input.getCancelKeys(), handled=True):
        self._windowMenu.open()
```

Always use `handled=True` when the current object consumes the action.

## Practical Notes

- Keep game-specific code in `Source`.
- Use `Global.System` for scene and screen operations.
- Use `Global.Manager` for resources and audio.
- Use `Source.Data` for project data.
- Use `GameMap` methods for actor lookup, spawning, and pathfinding.
- Treat tuple parameters as convenience input; stored runtime values are engine objects.
- Handle optional shader returns on platforms where shaders are disabled.
