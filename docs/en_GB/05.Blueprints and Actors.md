# 05. Blueprints and Actors

Blueprints define the actors that can be placed on maps. In the editor, think of a blueprint as a reusable object template with appearance, properties, components, and visual events.

## Opening Blueprints

Open a blueprint from the file explorer or from linked data panels. Choose `New Blueprint` from the **Game** menu, or press `F4`, to create a new blueprint file. Edit a copy when experimenting with behaviour that is already used on a map.

## Blueprint Categories

**Actors**

Player and test actors, such as Braver, dialogue tests, and pathing tests.

**Consumables**

Pickup objects that immediately change player values or inventory.

**Doors**

General doors and coloured doors.

**Enemies**

Enemy actors placed on maps.

**Equips**

Equipment pickups such as the starter sword and shield.

**Items**

Keys, enemy book, floor teleport item, and terrain-changing utility items.

**Teleportations**

Upstairs and downstairs actors used by the floor flow.

## Common Properties

**Image and crop**

Choose the sprite image and default crop. Test alignment on the map after replacing images.

**Collision**

Controls whether the actor participates in touch, overlap, and map blocking.

**Movement**

Speed and movement-related fields are used by player, enemy, and pathing actors.

**Light and material**

Used by map lighting and render effects.

**Battle values**

Player and enemy blueprints may include HP, attack, defence, EXP, gold, level, job, states, specials, and rewards.

**Icon, price, and description**

Used by item, equipment, shop, and menu windows.

## Event Pages

Blueprints can hold visual event pages such as creation, update, collision, overlap, removal, encounter, or defeat. Add nodes from the node list, connect them in order, then save and test on a map.

## Placement Check

After placing a blueprint, check position, layer, tag, collision, instance overrides, and save persistence. Any actor that can be removed or revisited should have a stable tag.

When spawning actors from event graphs, use `CreateActorFromBPPath` for normal blueprint defaults. Use `CreateActorFromBPPathWithDefaults` when the spawned instance needs class default overrides supplied as a dict; if the actor is recorded with `RecordAddedActor`, those overrides are restored from saves.
