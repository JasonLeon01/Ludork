# 05. Blueprints and Actors

Blueprints define map objects: appearance, properties, interaction, and visual events. The Ludork default template already groups them by purpose.

## Current Categories

- `Actors`: Braver, dialogue test object, and pathing test object.
- `Consumables`: red/blue gems and recovery bottles.
- `Doors`: general, yellow, blue, and red doors.
- `Enemies`: slime and skeleton soldier.
- `Equips`: basic sword and shield.
- `Items`: yellow/blue/red keys, enemy book, floor teleport, Pickaxe, Earthquake Scroll, Snowflake, and Ice Axe.
- `Teleportations`: upstairs and downstairs points.

## Common Properties

Blueprints commonly use image path, default crop, collision, speed, material, light, battle values, price, icon, description, and inventory cost.

Players and enemies use HP, max HP, attack, defence, EXP, gold, level, and job. Equipment changes player values, while consumables directly affect state or resources.

## Visual Events

Blueprints can maintain visual event graphs in the editor. Doors check opening conditions on contact, stairs move to another map, items trigger effects on use, and test actors show dialogue or pathing behavior.

After editing an event, save and trigger it in a map. Check sound, animation, item cost, map changes, and save behavior.

## Placing on Maps

Use `Actor Mode` on a map, choose a blueprint, and place it. Then check position, layer, tag, and instance properties. Any object tracked by saves should have a stable tag.
