# 06. UI and Windows

The runtime UI system is built from drawable controls loaded into a scene's UI manager.

## Loading UI

Create controls in `SceneBase.onCreate()` and load them into `self._uiManager`.

```python
from Engine.UI import Image
from Global import Manager

self._bg = Image(Manager.loadSystem("GrassBackground.png"))
self._uiManager.loadUI(self._bg)
```

Controls are updated and rendered by the scene loop.

## Coordinate System

UI positions use logical game coordinates. `Engine.Scale` is applied internally when controls render.

For a default `640 x 480` game, the centre is `(320, 240)`.

## Image

`Engine.UI.Image` displays a texture.

```python
from Engine.UI import Image
from Global import Manager

logo = Image(Manager.loadSystem("TitleLogo.png"))
logo.setPosition((120, 48))
self._uiManager.loadUI(logo)
```

## PlainText

`PlainText(font, text, characterSize=None, style=Text.Style.Regular, fillColor=Color.White)` renders a single text style.

Important methods:

- `setString(text)`
- `getString()`
- `setCharacterSize(size)`
- `getCharacterSize()`
- `setColor(color)`
- `getSize()`
- `getLocalBounds()`
- `getAbsoluteBounds()`

Example:

```python
from Engine.UI import PlainText
from Source import System as GameSystem
from Engine import Color

text = PlainText(GameSystem.getFonts()[0], "Hello", 24, fillColor=Color.White)
text.setPosition((32, 32))
self._uiManager.loadUI(text)
```

## RichText

`RichText(font, text, styleCollection)` renders segments with multiple styles.

Markers use `#name#` for named styles and hex colour markers when possible.

```python
from Engine.UI import RichText, TextStyle
from Engine import Color, Text

styles = {
    "default": TextStyle(characterSize=22, fillColor=Color.White),
    "red": TextStyle(fillColor=Color.Red, style=Text.Style.Bold),
}
message = RichText(font, "You got #red#Potion#default#!", styles)
```

## Window

`Window(rect, windowSkin, repeated=False)` renders a windowskin frame.

`rect` may be passed as `((x, y), (width, height))`.

```python
from Engine.UI import Window
from Global import Manager

skin = Manager.loadSystem(GameSystem.getWindowskinName()).copyToImage()
window = Window(((32, 32), (240, 96)), skin)
self._uiManager.loadUI(window)
```

## Command Windows

The sample `WindowCommand` builds menu commands from a dictionary:

```python
self._windowCommand = WindowCommand(
    ((0, 0), (320, 96)),
    {
        "Start": {"text": LOC("TITLE_START"), "callback": Scene._startGame},
        "Exit": {"text": LOC("TITLE_EXIT"), "callback": Scene._exitGame},
    },
    columns=2,
)
```

Callbacks receive the selected object and keyword data.

## Message Window

The sample `SceneMap` exposes dialogue helpers:

```python
self.showMessage("npc_01", "Guide", "Welcome.")
```

`showMessage(refActorTag, name, message)` disables player movement while dialogue is open and returns a latent condition.

`showSelection(refActorTag, name, options, allowCancel)` displays choices and returns the selected index, or `-1` when cancelled.

```python
choice = self.showSelection("npc_01", "Guide", ["Yes", "No"], True)
```

## Menu and Save Windows

The sample map scene creates:

- `WindowMenu`
- `WindowItem`
- `WindowEquipSlot`
- `WindowEquipSelect`
- `WindowSaveLoad`
- `WindowShop`
- `WindowEnemyBook`
- `WindowFloorTeleporter`
- `PlayerAttrHUD`

These are project-level windows in `Source.Windows`. Use them as examples when adding new RPG windows.

## Configuration Window and Common Controls

The title scene uses `ConfigWindow` to manage runtime settings. It is based on `WindowSelectable` and `ListView`, with each row combining a label and a control:

- `DropBox`: language, scale, and frame rate.
- `CheckBox`: vertical sync and music/sound/voice switches.
- `Slider`: music/sound/voice volumes.

Language and scale use `saveLanguage()` and `saveScale()` to write `Main.ini` and show a restart tip; frame rate, vertical sync, and audio switches/volumes are applied immediately through `Global.System`.

## Active and Blocking Windows

The sample menu windows expose state such as `isBlocking()` and `isOpen()`. The map scene also checks whether message, shop, enemy-book, and floor-teleporter windows are visible, so map movement, dialogue input, and menu input do not fight each other.

When adding new windows, follow the same rule: a visible modal or menu window should consume input and report that it blocks map actions.
