# 07. Project and Runtime Structure

This chapter explains how a game project starts and how the runtime packages work together.

## Project Root

The game is run from the project root. Paths in runtime scripts are usually relative to that root:

```text
./Main.ini
./Data/Configs/System.json
./Assets/System/icon.png
./Assets/System/cursor.png
./Assets/System/WS-prefix100-original2.png
./Save/
```

Keep game scripts inside `Source/` unless you are extending the runtime itself.

## Main.ini

`Main.ini` stores process-level options read before the game window is created.

Common fields in the `[Main]` section:

- `script`: the main script filename, normally `Entry.py`.
- `language`: locale key, such as `zh_CN` or `en_GB`; empty means use the system locale.
- `scale`: display scale.
- `frameRate`: target frame rate.
- `verticalSync`: whether vertical sync is enabled.
- `musicOn`, `soundOn`, `voiceOn`: audio category switches.
- `musicVolume`, `soundVolume`, `voiceVolume`: category volume values from `0` to `100`.

The values are loaded by `Global.System.init()` and may be saved again when changed through system setters.

## Entry Sequence

`Entry.py` performs the following runtime steps:

1. Imports `Engine`, `Global`, and `Source`.
2. Starts a stdin debug worker when `WINDOWHANDLE` is present, allowing an embedded host to send expressions or statements.
3. Listens on `localhost:2333` through `debugpy` when that package is available.
4. Initialises game locale data from `Data/Locale`.
5. Initialises latent node graph support.
6. Loads `Main.ini` through `Global.System`.
7. Binds scene operations to the current thread.
8. Sets the first scene to `Source.Scenes.Init()`.
9. Calls `Source.System.init()` to create the window and load core settings.
10. Repeats the active scene's `main()` method while the game should keep running.
11. Saves FPS history and prints the exit message when the loop ends.

The simplified shape is:

```python
Engine.Locale.init("./Data/Locale")
Engine.NodeGraph.initLatent()
Global.System.init(iniFile, iniFilePath)
Global.System.bindSceneOperationThread()
Global.System.setScene(Source.Scenes.Init())
Source.System.init()

while Global.System.shouldLoop():
    Global.System.applyPendingSceneReplace()
    scene = Global.System.getScene()
    if scene:
        scene.main()
```

## Engine Package

`Engine` exports SFML/PySF classes and Ludork helpers. Use it when you need vectors, colours, input, UI controls, node decorators, file helpers, or gameplay primitives.

Examples:

```python
from Engine import Vector2u, Vector2f, Color
from Engine import Input
from Engine.UI import PlainText
```

Useful constants:

- `Engine.CellSize`
- `Engine.GameSize`
- `Engine.Scale`
- `Engine.GameRunning`

Useful helpers:

- `Engine.Cast(type_, value)`
- `Engine.AssertType(obj, annotation)`
- `Engine.Eval(expr)`
- `Engine.OppositeDirection(direction)`

## Global Package

`Global` contains game-wide services:

- `System`: window, canvas, settings, scene stack, transitions, flash, shake, weather, fog.
- `SceneBase`: base class for scenes.
- `GameMap`: tilemap, actors, collision, lights, and pathfinding.
- `Camera`: map view and render texture.
- `Manager`: resources, audio, fonts, textures, shaders, and time.
- `playVideo()`: blocking video playback helper.

Example:

```python
from Global import System, SceneBase
from Global import Manager
```

## Source Package

`Source` is the project's own game-rule layer. It may be replaced or reorganised by a real project.

Important modules:

- `Source.System`: reads `Data/Configs/System.json` and `Audio.json`.
- `Source.Data`: loads animations, tilesets, autotiles, common functions, and general data.
- `Source.GameInstance`: holds player and persistent game state.
- `Source.Scenes`: title, initialisation, map, and game-over scenes.
- `Source.Windows`: template RPG windows, including menu, save/load, item, equipment, shop, enemy book, floor teleporter, and configuration windows.
- `Source.NodeFunctions`: node-callable project functions.

## Initial Scene

The initial scene is `Source.Scenes.SceneInit.Scene`. It normally loads project data and switches to the title scene.

The title scene can start a new game by creating `GameInstance()` and switching to `SceneMap`.

## Threading Note

Scene logic runs on a logic thread, while rendering and scene replacement are kept on the scene operation thread. Use normal scene APIs such as `System.setScene()` instead of replacing internal scene lists directly.

When `System.setScene()` is called from another thread, the replacement is queued and applied by `System.applyPendingSceneReplace()`.
