# 14. Data, Localisation, and Saves

This chapter explains how data changes affect progress and displayed text.

## Data References

Maps, blueprints, general data, and localisation reference one another. After changing a key, file name, or asset path, check every map, blueprint, window, and data table that uses it.

When adding new records, copy an existing similar record first, then rename and adjust values. This keeps fields complete and makes comparison with the Ludork default template easier.

## Text Display

Items, enemies, states, menus, and tips can all use localisation. To test language changes, switch language from the title config window, re-enter the game, then check title, menus, dialogue, items, enemy book, and save details.

If text appears as a key or is blank, the localisation table is usually missing that entry, or the exported language file is stale.

## Save Content

Saves record player position, values, items, equipment, states, current map, removed objects, changed terrain, floor teleport points, and a screenshot.

For save testing, save current progress, return to title and continue, then trigger the same doors, items, enemies, and terrain changes again.

## Cleaning Test Saves

When you need a fresh new-game test, back up or clear `Save` first. Do not remove progress you still need.
