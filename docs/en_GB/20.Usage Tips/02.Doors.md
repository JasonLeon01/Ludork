# 02. Doors

Doors are obstacles and puzzle elements on a Mota map. Default examples are under `/Data/Blueprints/Doors/`; mechanism doors are under `/Data/Blueprints/Doors/ConditionDoors/`.

## Doors that open on contact

`BP_Door_General` opens as soon as the player collides with it. If you need the same behaviour with a different look, derive a new blueprint from it and change `texturePath` (and `defaultRect` if required).

## Coloured key doors

Yellow, blue, and red key doors are provided as `BP_Door_Y`, `BP_Door_B`, and `BP_Door_R`. They inherit from `BP_Door_General` and use `needKeyID` to name the key required from `/Data/General/Item.dat`.

To add, for example, a green door, derive from `BP_Door_Y`, set `needKeyID` to the matching key ID, and update the texture rect to suit your tileset.

## Mechanism doors

Doors in `/Data/Blueprints/Doors/ConditionDoors/` do not consume keys. They open when an event-bus variable reaches a target value. For a new mechanism door, derive from `BP_Door_ConditionExample` and adjust:

- `eventBusName`: the event bus to listen on.
- `eventBusVariableName`: the linked game variable name.
- `targetConditionValue`: the value required before the door opens.

The example registers on `onCreate` and compares the variable when it changes. Copy its node graph when wiring a new door of this kind.
