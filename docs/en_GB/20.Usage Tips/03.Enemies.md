# 03. Enemies

Enemy blueprints live under `/Data/Blueprints/Enemies/`. In most cases you only need an `ID` that matches a record in `/Data/General/Enemy.dat`; stats, names, and descriptions are then resolved from General data.

## Standard enemies

Take `BP_Enemy_Slime` as an example: `attrs.ID` is set to `Slime`, which links to the record of the same name in `/Data/General/Enemy.dat`. When duplicating an example, keep `ID` aligned with General data and adjust presentation fields such as `texturePath` and `defaultRect` as needed.

## Mechanism enemies (defeat to open doors)

When an enemy must trigger a mechanism door on defeat, refer to the examples in `/Data/Blueprints/Enemies/ConditionEnemies/` (for example `Slime2FCondition.json`).

These blueprints use the `onDefeat` event graph together with the event bus and game variables: defeating the enemy increments or writes a variable that pairs with the target door's `eventBusName`, `eventBusVariableName`, and `targetConditionValue`. Derive from a similar enemy blueprint and edit the bus and variable names in `onDefeat` so they match the door you intend to open.
