# 15. Event Nodes and Common Flows

The Sample game uses visual event graphs for blueprint behaviour and common reusable flows. This page describes both the editor usage and the runtime functions that those graphs can call.

## Blueprint Event Pages

Blueprints can define pages such as creation, update, collision, overlap, removal, encounter, and defeat. The exact pages depend on the actor class and blueprint data.

Common uses include:

- doors checking keys or conditions;
- stairs transferring maps;
- items changing inventory or terrain;
- enemies resolving battle results;
- NPCs showing dialogue or selections.

## Common Flows

`Data/CommonFunctions` contains reusable graphs. Open `Common Functions` from the database menu, or press `F9`. The default Sample project includes `PrintTest.dat`.

Use common flows for shared tips, common rewards, repeated map logic, or debugging flows that should be callable from several blueprints.

## Blueprint Decorators

Blueprint functions and classes are exposed to the editor through decorators in `Sample/Engine/Decorators.py`. `@Meta` controls display names and specialised parameter editors; `@ExecSplit`, `@Latent`, and `@ReturnType` define execution and data pins. See `18.Blueprint Decorators` for the full reference.

## Node Function Groups

**Container**

Dictionary and list operations.

**Math and String**

Numeric, vector, comparison, boolean, random, and text helpers.

**Movement**

Movement routes and automatic pathing.

**Player**

Inventory, equipment, player values, gold, EXP, and player lookup.

**Scene and Mota**

Map transfer, terrain edits, shops, enemy book, floor teleport, and tower region helpers.

**State**

Battler states, damage, healing, and event arguments.

**System**

Audio, video, transition, flash, shake, and weather.

**Utils**

Local values, game variables, animations, common function calls, return-to-title, and print output.

## Calling Common Functions

Game code can invoke a common function through the utility node layer:

```python
from Source.NodeFunctions.Utils import RunCommonFunction

RunCommonFunction("PrintTest")
```

## Testing Events

Change one event path at a time. Save, start the test game, trigger the exact actor on a map, and check the console if the event does not run.
