# 15. Event Nodes and Common Flows

Ludork default template interactions are mainly maintained through visual events in the editor. This chapter only covers how to use them.

## Blueprint Events

Blueprints can maintain event pages for creation, updates, collision, overlap, and removal. Doors, stairs, dialogue test actors, and some items use event graphs for interaction.

Choose an event page, add actions, conditions, and data nodes from the node list, then connect them to decide order. Save and trigger the result on a map.

## Common Flows

`PrintTest` in `Data/CommonFunctions` is the Ludork default template common-flow example. Common flows are useful for logic reused by multiple blueprints, such as shared tips, shared rewards, or test output.

## Common Node Groups

The node list is grouped by purpose. Common groups include container, math, movement, tower gameplay, player, save, scene, state, string, system, and utilities.

When choosing nodes, prefer the displayed name and description. Parameters should use real map names, tags, item IDs, equipment IDs, or localisation keys from the project.

## Test Advice

Change one small event section at a time, save, then test immediately. If an event does not trigger, check that the blueprint is on the map, the tag is correct, the trigger type matches, and the condition can become true.
