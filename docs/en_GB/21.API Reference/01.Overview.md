# 01. Overview

This folder documents the blueprint-visible API collected from handwritten Python scripts and C++ extension methods that expose blueprint metadata through their Python wrappers. Generated blueprint data classes are not included.

## Reading Rules

- Original names use the public Python module path and function name. Some entries are implemented by C++ extension methods behind that Python surface.
- Display names and descriptions are resolved from `Sample/Data/Locale/Locale.xlsx` when the API declares `LOC("...")` metadata.
- If no localisation metadata exists, the blueprint node picker falls back to the function name; this manual does the same.
- Class sections list public class variables declared directly in the handwritten class body, blueprint-editable component fields where applicable, then the API methods exposed from that class.
- `Source.NodeFunctions` is documented separately because those functions appear as standalone node groups.
- Python default values in function signatures are reflected in each entry heading; unconnected pins use the source default. See `18.Blueprint Decorators` for decorators and `@Meta` metadata.
- Parameters or class variables marked as General Data/animation selections use `GeneralDataVars` combo boxes in the editor. The documentation identifies the corresponding data source; values remain plain strings.

## Pages

Each Markdown file inside a category folder documents one handwritten Python source file.

- `02.NodeFunctions/`: standalone node functions.
- `03.Source Class APIs/`: Sample gameplay classes and info/event classes.
- `04.Global APIs/`: scene, map, camera, UI manager, and global runtime helpers.
- `05.Engine Gameplay APIs/`: engine actor and tilemap APIs.
