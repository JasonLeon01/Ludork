# 00. BPBase

Source file: `Sample/Engine/BPBase.py`

## Class `Engine.BPBase`

Blueprint system base class. Provides static methods for dispatching blueprint events through node graphs, applying GeneralData attributes, and managing the info-layer graph fallback mechanism. All `Actor` and `InfoBase` subclasses inherit from `BPBase`.

**Static API**

#### `BlueprintEvent(obj: object, objType: type, eventName: str, kwargs: Dict[str, Any] = None) -> None`

Original: `Engine.BPBase.BlueprintEvent`

Dispatch a blueprint event on the given object. Resolution order: actor-layer `_graph` → info-layer `_infoGraph` → parent class graph or direct method call.

**Parameters**
- `obj` – Target object instance
- `objType` – Expected type for isinstance check
- `eventName` – Name of the event to trigger (e.g. "onUse", "onCreate")
- `kwargs` – Arguments passed to the event

**Exec Outputs:** routed to matching event handler

---

#### `ApplyGeneralData(obj: object, data: Dict[str, Any], paramsType: Dict[str, Any]) -> None`

Original: `Engine.BPBase.ApplyGeneralData`

Apply key-value pairs from GeneralData onto an object's attributes. Handles type coercion for basic types (int, float, bool, string, list, tuple). Keys starting with '_' are skipped.

**Parameters**
- `obj` – Target object to set attributes on
- `data` – Member data dictionary from GeneralData
- `paramsType` – Parameter schema with type/defaultValue definitions

**Exec Outputs:** `default`

---

#### `ExecuteParentEvent(obj: object, cls: type, eventName: str, ...) -> None`

Original: `Engine.BPBase.ExecuteParentEvent`

Execute a blueprint event handler defined in the parent class of the given object, falling back through the inheritance hierarchy.

**Exec Outputs:** `default`

---

#### `ExecuteInfoGraph(obj: object, eventName: str, kwargs: Dict[str, Any] = None) -> None`

Original: `Engine.BPBase.ExecuteInfoGraph`

Explicitly dispatch an event through the info-layer graph only, bypassing the actor-layer graph.

**Exec Outputs:** `default`
