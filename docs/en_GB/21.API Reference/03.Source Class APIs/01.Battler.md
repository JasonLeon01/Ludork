# 01. Battler

Source file: `Sample/Source/Battler.py`

## Class `Source.Battler.Battler`

Mixin providing combat stats and state management. Attach to any Actor via multiple inheritance to give it battle capabilities. Manages a list of active `StateInfo` objects whose blueprint events can drive non-combat behaviours such as walking effects and explicit hooks.

**Class variables**

- `infoComp`: type `BattlerInfoComponent`; default `BattlerInfoComponent()`

**Blueprint API**

The `state` and `stateKey` parameters documented below use `GeneralDataVars` combo boxes populated from `State` General Data when edited as unconnected blueprint inputs. The methods still accept a connected `StateInfo` instance where their signatures allow it.

#### `addState(state: Union[str, StateInfo], stacks: int) -> None`

Original: `Source.Battler.Battler.addState`

Apply a state to this battler with the given stack count. Accepts either a state ID (string) or a pre-built `StateInfo`. When given an ID the corresponding `StateInfo` is built from GeneralData. If the state is already active and its GeneralData marks it as stackable, the stack count is increased instead.

**Parameters**
- `state` – State ID selected from `State` General Data, or a connected `StateInfo` instance.
- `stacks` – Stack count to apply or add.

**Exec Outputs:** `default`

---

#### `removeState(state: Union[str, StateInfo]) -> None`

Original: `Source.Battler.Battler.removeState`

**Parameters**

#### `reduceStateStacks(state: Union[str, StateInfo], stacks: int = 1) -> None`

Original: `Source.Battler.Battler.reduceStateStacks`

Reduce the stack count of an active state. If the stack count reaches `0`, the state is removed.

**Parameters**
- `state` – State ID selected from `State` General Data, or a connected `StateInfo` instance.
- `stacks` – Stack count to reduce.

**Exec Outputs:** `default`
