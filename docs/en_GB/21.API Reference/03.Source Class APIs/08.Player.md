# 08. Player

Source file: `Sample/Source/Player.py`

## Class `Source.Player.Player`

Player-controlled character combining `Character`, `PlayerInfo`, and `Battler`.

**Class variables**

- `ID`: type `str`; default `'FILL_IT_BY_YOURSELF'` — `GeneralDataVars` combo populated from `Player` General Data.
- `infoComp`: type `PlayerInfoComponent`; default `PlayerInfoComponent()`

**Blueprint API**

The `itemID` and `equipID` parameters below use `GeneralDataVars` combo boxes populated from `Item` and `Equip` General Data. `slotID` remains a normal string because equipment slots are defined by the selected class record rather than a General Data member list.

#### Inventory

- `addItem(itemID: str, count: int = 1) -> None` — Add items. Exec output: `default`.
- `removeItem(itemID: str, count: int = 1) -> bool` — Remove items. Exec outputs: `success`, `failed`.
- `getItemCount(itemID: str) -> int` — Return the owned item count. Output: `count`.
- `hasItem(itemID: str) -> bool` — Check whether at least one item is owned. Output: `value`.

#### Equipment

- `addEquip(equipID: str, count: int = 1) -> None` — Add equipment to the bag. Exec output: `default`.
- `removeEquip(equipID: str, count: int = 1) -> bool` — Remove equipment from the bag. Exec outputs: `success`, `failed`.
- `equip(equipID: str) -> None` — Equip a record from `Equip` General Data. Exec output: `default`.
- `unequip(slotID: str) -> None` — Unequip the current record in a class-defined slot. Exec output: `default`.
- `getEquipCount(equipID: str) -> int` — Return the owned equipment count. Output: `count`.
- `hasEquip(equipID: str) -> bool` — Check whether at least one equipment record is owned. Output: `value`.
- `getEquipInfo(slotID: str) -> str` — Return the equipped ID in a slot, or `""`. Output: `value`.

#### Movement State

- `getForbiddenMoving() -> bool` — Return whether player movement is blocked. Output: `value`.
- `setForbiddenMoving(value: bool) -> None` — Enable or disable player movement blocking. Exec output: `default`.
