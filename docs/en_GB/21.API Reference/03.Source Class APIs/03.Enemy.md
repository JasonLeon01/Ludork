# 03. Enemy

Source file: `Sample/Source/Enemy.py`

## Class `Source.Enemy.Enemy`

Scene enemy entity. Bridges `Actor` (rendering/collision/movement) and `EnemyInfo` (enemy data + event logic) via multiple inheritance.

**Class variables**

- `ID`: type `str`; default `'FILL_IT_BY_YOURSELF'` — `GeneralDataVars` combo populated from `Enemy` General Data.
- `infoComp`: type `EnemyInfoComponent`; default `EnemyInfoComponent()`
- `tickable`: type `bool`; default `True`
- `collisionEnabled`: type `bool`; default `True`
- `animatable`: type `bool`; default `True`
- `animateWithoutMoving`: type `bool`; default `True`

**Component fields**

Blueprint data may attach optional components on an enemy. These are edited as nested fields, not as node functions.

- `childActorComp` (`ChildActorComponent`) — Spawns a damage-text child actor on create. Default class is `Source.EnemyDamageText.EnemyDamageText`.
  - `className`: type `str`; default `'Source.EnemyDamageText.EnemyDamageText'`
  - `relativePosition`: type `Pair[float]`; default `(0.0, 0.0)` — x/y numeric editor (`@Meta(Vector2fVars)`).
- `lightComp` (`LightComponent`) — Inherited from `Actor`. See `05.Engine Gameplay APIs/01.Actors/01.Actor.md`.

**Blueprint API**

#### `battle()`

Original: `Source.Enemy.Enemy.battle`

Perform battle calculations against the player. On player victory, the enemy's `onDefeat` event is also fired.

**Exec Outputs:** `Win`, `Lose`, `Escape`
