# 02. Editor Basics

This page describes the everyday editor workflow. It is intentionally written as an operating guide: use the editor controls, save the project, then test the result in the Sample game.

## Opening a Project

**Open Project**

Choose `Open Project`, then select the project `Main.proj` file. The editor reads the surrounding folder as the project root.

**New Project**

Choose `New Project`, select a target folder, and enter a project name. The editor copies the default template so the original Sample project remains available for comparison.

**Recent work**

Keep experimental changes in a copied project. This makes it easier to compare maps, data, and assets with the default template when something stops working.

## Main Editor Areas

**Map list**

The map list is the normal entry point for opening and managing maps. Use it to add, rename, copy, paste, delete, or edit map properties.

**Editor panel**

The central panel edits the active map. Switch modes according to the task: tiles for terrain, actors for placed objects, and lights for light sources.

**File explorer**

The file explorer opens project files such as blueprints, animation data, localisation files, and data tables. Prefer opening files from here instead of editing them outside the project context.

**Console**

The console shows test-game output and editor messages. Use the filters when checking warnings, errors, or normal messages.

## Main Menus

**File**

Project creation, opening, saving, packing, and exit commands.

**Edit**

Undo and redo where the active editor supports them.

**Game**

`Test Game` starts the current project. `Game Settings` edits the values stored in `Main.ini`.

**Database**

System config, animation overview, tileset data, common functions, general data, and plugin-provided tools. The official locale plugin adds the locale table and export commands here.

**Help**

Documentation, language switching, and editor information.

## Keyboard Shortcuts

| Shortcut | Action |
|----------|--------|
| `F1` | Contents (help documentation) |
| `F2` | Game Settings |
| `F3` | Reload Module |
| `F4` | New Blueprint |
| `F5` | System Config |
| `F6` | Animation Overview |
| `F7` | Tilesets Data |
| `F8` | Common Functions |
| `F9` | General Data |
| `F10` | New Animation |
| `F11` | New Curve |

These shortcuts match the corresponding items in the **Game** and **Database** menus. `Ctrl+S` saves the project; `Ctrl+Z` / `Ctrl+Y` undo and redo where the active editor supports them.

## Saving

Use `Save` after changing maps, blueprints, curves, general data, settings, common functions, animation data, or localisation. A test game reads saved project files, so unsaved editor changes may not appear.

For curve editing details, see `07.Curve Assets`. For localisation changes, use the official locale plugin's editor or export command so the language files are refreshed.

## Testing

Start `Test Game`, reach the title screen, and check the area you changed. For display and language changes, closing and restarting the test game is often clearer than staying in the same session.
