# 02. Editor Basics

This chapter explains how to open, create, save, and understand Ludork projects in the editor. It describes user-facing workflows only, not the editor's implementation code.

## Opening a Project

After starting Ludork, choose `Open Project` from the start page or the menu, then open the project's `Main.proj`. `Main.proj` identifies a folder as a project that the editor can open.

To create a project from the built-in template, choose `New Project`, select a target folder, and enter a project name. The editor copies the template into the new project folder; later edits happen in that new folder.

## Saving a Project

Use `Save` after editing data. Saving writes the currently edited data and may also synchronise runtime configuration files. Save once before testing the game.

If a test run still shows old content, first check whether the related maps, blueprints, general data, configuration, and locale table have been saved or exported.

## Typical Project Folder

The template project is a runnable RPG project and a starting point for new projects. A typical structure looks like this:

```text
Project/
  Entry.py
  Main.ini
  Main.proj
  Assets/
  Data/
  Engine/
  Global/
  Source/
  Save/
  Temp/
```

Common folders:

- `Assets`: images, audio, fonts, shaders, window skins, cursor images, and other resources.
- `Data`: maps, tilesets, autotiles, blueprints, general data, common functions, and localisation data.
- `Source`: project-specific game rules, such as scenes, windows, player logic, items, enemies, and node functions.
- `Engine` and `Global`: template runtime and shared systems. Ordinary projects usually start by editing data and `Source`.
- `Save`: runtime save files.
- `Temp`: runtime temporary files.

## Template Project Contents

The current template uses 32-pixel tiles, a default logical game size of `640 x 480`, `Map_01.dat` as the start map, and `(6, 12)` as the start position.

The template includes:

- Two maps: `Map_01` and `Map_02`.
- One normal tileset: `Tileset_01`.
- Three autotiles: `grass`, `lava`, and `galaxy`.
- Blueprint categories: Actors, Consumables, Doors, Enemies, Equips, Items, and Teleportations.
- General data tables: Class, Enemy, Equip, Item, Special, and State.
- A common function: `PrintTest`.
- Two exported locales: `en_GB` and `zh_CN`.

## Test Entry

After saving the project, use `Test Game` to run the current project. The editor runs the project entry script and loads the current project's `Main.ini`, `Data`, and `Assets`.

During testing, first confirm that the game starts from the expected map and position. Then check whether maps, actors, events, audio, language, and windows reflect the currently saved data.
