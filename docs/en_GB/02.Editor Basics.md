# 02. Editor Basics

This chapter describes the basic workflow for maintaining the Ludork default template in the editor.

## Opening and Creating

After starting Ludork, use `Open Project` and choose `Main.proj` in the project folder. To create a new project from the Ludork default template, use `New Project`, choose a target folder, and enter a project name. The editor copies the template for you.

Work in the copied project when experimenting, so the Ludork default template remains available for comparison.

## Saving

After editing maps, blueprints, general data, settings, or localisation, use `Save`. Save once before testing so the game reads the latest data.

If you edited localisation, also confirm that the language files were exported. Saving through the editor localisation table normally does this.

## Common Entry Points

- `Game Settings`: language, scale, frame rate, audio switches, and volume.
- `System Config`: title, game size, fonts, icon, cursor, window skin, tile size, start map, and start position.
- `Tilesets Data`: tileset image, passability, and material settings.
- `Common Flows`: reusable visual flows.
- `General Data`: items, equipment, enemies, states, specials, and jobs.
- `Localisation Table`: English and Chinese text.

## Testing

Use `Test Game` to start the current project. On the title screen, check language, scale, volume, and start settings first, then enter a map and verify blueprints, events, audio, and saves.
