# 05. Item

The **Item** tab maps to `/Data/General/Item.dat`. It is linked to `ItemInfo` and supports `onUse` and `onGet` blueprint events.

## Fields

**name**, **desc**

Locale keys for inventory, shops, and messages.

**icon**

Path relative to `/Assets/` for UI display.

**usable**

Whether the item can be triggered from the item menu. Keys are usually `false`; utility items such as `EnemyBook` or `Teleport` set this to `true`.

**price**

Shop buy/sell reference value.

**cost**

When `true`, using the item consumes one unit from inventory. Keys and similar non-consumables often keep this `true` so use removes the key; adjust per item behaviour.

## Blueprint Events

**onUse**

Runs when the player uses the item from the menu (only when `usable` is true).

**onGet**

Runs when the item is added to inventory.

Implement item-specific behaviour here or in dedicated window code that reads the item ID.

## Linking Map Blueprints and Doors

Item blueprints under `/Data/Blueprints/Items/` set `attrs.ID` to a member key (for example `KEY_Y`).

Coloured doors reference the same IDs through `needKeyID` on door blueprints. Changing a key ID requires updating every door and blueprint that references it.

## Default Sample Members

| ID | Role |
|----|------|
| `KEY_Y`, `KEY_B`, `KEY_R` | Yellow, blue, and red keys. |
| `EnemyBook` | Opens the enemy compendium. |
| `Teleport` | Opens floor teleport UI. |
| `BreakWall`, `ClearWall`, `BreakLava`, `BreakIce` | Terrain tools. |

Add a General Data member before creating the matching item blueprint or shop entry.
