# 06. Player

The **Player** tab maps to `/Data/General/Player.dat`. It is linked to `PlayerInfo` and defines party-member identity, class assignment, and battle stats loaded at runtime through `Player.initInfo`.

Unlike **Class**, which only describes job metadata and equipment slots, **Player** records describe concrete party members (name, stats, starting class, battle animation).

## Fields

**name**, **desc**

Locale keys for HUD, menus, and save data display.

**CLASS**

Member ID from `/Data/General/Class.dat` (for example `Warrior`). At runtime the player reads that class record to resolve equipment slots and default gear.

**MAXHP**, **ATK**, **DEF**

Core battle stats stored on `PlayerInfoComponent` and used by `Battler` calculations.

**EXP**, **GOLD**

Current experience and currency totals for the party member.

**LEVEL**

Current level. When `LEVEL` changes the player receives automatic stat growth driven by the module-private multipliers in `Source/Player.py`:

- `HP += delta * _LEVEL_HP_GAIN` (default: 400)
- `ATK += delta * _LEVEL_ATK_GAIN` (default: 2)
- `DEF += delta * _LEVEL_DEF_GAIN` (default: 2)

This is implemented via the `Engine.Utils.Monitor` mechanism in `Player.__init__`. The resulting HP is automatically clamped to `MAXHP` by the built-in `Battler` HP monitor.

**ANIMATION_KEY**

Animation asset name under `/Data/Animations/` played during enemy battles (for example `attack`). Leave empty to skip a player attack animation.

## Blueprint Events

`PlayerInfo` does not declare blueprint events in the Sample template. Set the tab's linked type to `PlayerInfo` in the General Data editor if you add `@RegisterEvent` hooks on that class later.

## Linking Player Blueprints

Player blueprints under `/Data/Blueprints/Actors/` set `attrs.ID` to a member key here (for example `Bravor` in `BP_Actor_Braver.json`).

General Data supplies name, class, and base stats; the blueprint controls map presentation (`texturePath`, `speed`, `lightComp`, and so on). Blueprint `infoComp` fields can override individual stats such as `HP` without duplicating the whole record.

At runtime `Player.initInfo` loads the matching member by `ID`, applies fields onto `infoComp`, then equips default items listed in the linked class `slot` dictionary.

## Default Sample Members

| ID | Purpose |
|----|---------|
| `Bravor` | Default Sample hero; class `Warrior`, `MAXHP` 2000, battle animation `attack`. |

When adding a new party member, create the General Data record first, ensure `CLASS` points at a valid **Class** member, then author or duplicate a player blueprint with the same `ID`.
