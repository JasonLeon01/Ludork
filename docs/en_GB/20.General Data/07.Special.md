# 07. Special

The **Special** tab maps to `/Data/General/Special.dat`. It has no linked type and no per-member blueprint graphs. Records here define named battle abilities referenced from enemy `special` dictionaries.

## Fields

**name**, **desc**

Locale keys for the enemy book and documentation strings.

**kwargs**

Optional default parameters for the special. The Sample set leaves this empty for most entries; battle code reads per-enemy values from the enemy's `special` map instead.

## How Enemies Reference Specials

On the **Enemy** tab, each enemy's `special` field lists special IDs with optional value strings:

```text
special: { "MultiHit": "2" }
```

The key must match a member ID in this tab. An empty value still enables the special; numeric strings supply parameters where the battle system expects them (for example `MultiHit` hit count).

## Default Sample Members

| ID | Battle behaviour (summary) |
|----|----------------------------|
| `Poisoning` | Damage-over-time style special tied to combat resolution. |
| `Weaken` | Reduces effective attack while active. |
| `Hard` | Raises effective defence against attackers. |
| `Magic` | Attacker ignores defender defence. |
| `MultiHit` | Multiplies per-round damage by the configured hit count. |
| `Compete` | Raises attack to at least the opponent's attack. |

See `10.Actors Enemies and Items` for how `Battler` applies these during damage calculation.

## Usage Notes

- Add the **Special** record first, then reference its ID from enemy `special` entries.
- Renaming a special ID requires updating every enemy that references it.
- `kwargs` on the special record is reserved for shared defaults; per-enemy overrides use the value string on the enemy record.
