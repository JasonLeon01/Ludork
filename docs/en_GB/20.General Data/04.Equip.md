# 04. Equip

The **Equip** tab maps to `/Data/General/Equip.dat`. It is linked to `EquipInfo` and supports `onEquip` and `onUnequip` blueprint events.

## Fields

**name**, **desc**

Locale keys shown in the equipment window and tooltips.

**icon**

Path relative to `/Assets/` for the inventory and equipment UI.

**slot**

Slot name that must exist on the player's class `slot` dictionary (for example `weapon` or `shield`).

**attrPlus**

Dictionary of attribute bonuses applied while equipped. Keys are player attribute names such as `ATK` or `DEF`; values are numeric strings evaluated at runtime.

Example from `Sword_A`:

```text
attrPlus: { "ATK": "10" }
```

## Blueprint Events

**onEquip**

Runs when the player equips this item.

**onUnequip**

Runs when the player removes it.

Use these for custom feedback or scripted bonuses beyond `attrPlus`.

## Linking Map Blueprints

Equip blueprints under `/Data/Blueprints/Equips/` set `attrs.ID` to a member key (for example `Sword_A`). The blueprint defines the floor pickup appearance; stats and icons come from General Data.

## Default Sample Members

`Sword_A`, `Shield_A`.

Create the General Data entry first, then duplicate a blueprint example and align `ID`, `texturePath`, and collision as needed.
