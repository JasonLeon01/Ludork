# 17. Game Settings Reference

This page lists the main Sample game settings and how they are used.

## Main.ini

**script**

Startup script. The default is `Entry.py`.

**language**

Active language key. The project should contain a matching exported file in `Data/Locale`.

**scale**

Window scale applied to the logical game size.

**framerate**

Target frame rate.

**verticalsync**

Enables or disables vertical sync.

**musicon, soundon, voiceon**

Independent audio switches.

**musicvolume, soundvolume, voicevolume**

Volume values from `0` to `100`.

`voiceon` and `voicevolume` apply to the single active voice clip used by dialogue. Turning voice off stops the active voice; changing `voicevolume` immediately updates the active voice volume.

## System.json

**title**

Window title and game title text source.

**gameSize**

Logical content size.

**fonts and fontSize**

Default font list and base text size.

**icon, cursor, windowskinName**

System images selected from `Assets/System`.

**cellSize**

Tile size used by map logic.

**coverOpaqueAlpha**

Alpha value for dark or covered screen overlays.

**startMap and startPos**

New-game map and tile position.

## Audio.json

Common sound slots include cursor, confirm, cancel, error, shop, save, load, gate, stairs, pickup, and equipment sounds. `titleBGM` selects title music.

## Runtime Access

Use the generated system config layer for current settings:

```python
from Global.System import System

if System.getSoundOn():
    volume = System.getSoundVolume()
```

## Verification

After changing settings, test the title scene, a map scene, menu windows, save/load, language switching, and a restart of the test game.
