# 02. Doors

Doors are obstacles and puzzle elements on a Mota map. Default examples are under `/Data/Blueprints/Doors/`; mechanism doors are under `/Data/Blueprints/Doors/ConditionDoors/`.

## Doors that open on contact

`BP_Door_General` opens as soon as the player collides with it. If you need the same behaviour with a different look, derive a new blueprint from it and change `texturePath` (and `defaultRect` if required).

## Coloured key doors

Yellow, blue, and red key doors are provided as `BP_Door_Y`, `BP_Door_B`, and `BP_Door_R`. They inherit from `BP_Door_General` and use `needKeyID` to name the key required from `/Data/General/Item.dat`.

To add, for example, a green door, derive from `BP_Door_Y`, set `needKeyID` to the matching key ID, and update the texture rect to suit your tileset.

## Mechanism doors

Doors in `/Data/Blueprints/Doors/ConditionDoors/` do not consume keys. They open through the event bus. Derive from the matching example depending on whether a variable condition is required.

### Condition mechanism doors

When the door should open only after a game variable reaches a target value, derive from `BP_Door_ConditionExample` and adjust:

- `eventBusVariableName`: the linked game variable name.
- `targetConditionValue`: the value required before the door opens.

Condition doors use the shared event bus key `ConditionalDoorAutoOpen` as a notification that some condition variable may have changed. The example registers this key on `onCreate`; when the bus fires it reads `eventBusVariableName`, compares it to `targetConditionValue`, and opens the door if the condition is met. Multiple condition doors can share the same bus key as long as their variable names are distinct.

When wiring a matching enemy or other trigger, increment or set the same game variable, then call `TriggerEventBus("ConditionalDoorAutoOpen")`. Copy the example node graph when wiring a new door of this kind.

### Trigger mechanism doors

When the door should open as soon as the event bus is triggered, with no variable check, derive from `BP_Door_TriggerExample` and adjust:

- `eventBusName`: the event bus to listen on.

The example registers the `AutoOpen` blueprint event on `onCreate`; any logic that calls `TriggerEventBus` with the same key will call `openDoor()` directly. Copy its node graph when wiring a new door of this kind.
