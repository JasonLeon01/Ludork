# 04. Equips and Items

Equips and items are straightforward to author: ensure each blueprint's `ID` matches a General data record.

## Equips

Equip blueprints are under `/Data/Blueprints/Equips/`. `attrs.ID` must match a key in `/Data/General/Equip.dat`; for example, `BP_Equip_Sword_A` uses `Sword_A`. Bonuses, names, and descriptions come from General data; the blueprint mainly defines how the pickup looks and collides on the map.

## Items

Item blueprints are under `/Data/Blueprints/Items/`. `attrs.ID` must match a key in `/Data/General/Item.dat`; for example, the yellow key item `BP_Item_Key_Y` uses `KEY_Y`. Keys, guide books, wall-breaking tools, and similar pickups follow the same rule: duplicate a close example, then change `ID` and texture.

When adding a new record, create the General data entry first and reuse the same `ID` in the blueprint so the runtime can resolve it.
