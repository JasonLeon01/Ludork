# 18. Testing and Troubleshooting

Use this page as a compact checklist when the Sample project does not behave as expected.

## Standard Test Route

1. Save the project.
2. Export localisation if text changed.
3. Start `Test Game`.
4. Start a new game.
5. Move, click path, open the menu, and trigger nearby actors.
6. Test doors, stairs, enemies, items, shops, enemy book, and floor teleport.
7. Save, return to title, and continue from that save.
8. Switch language, restart, and check visible text.

## New Content Does Not Appear

**Likely causes**

The edited panel was not saved, the wrong project was opened, the asset was moved without reselecting it, or the changed blueprint is not the one placed on the map.

**Checks**

Open the map or data file from the editor, confirm the value, save again, then restart `Test Game`.

## Missing Text

**Likely causes**

The localisation key is absent, one language is blank, or exported language files are stale.

**Checks**

Confirm both `zh_CN` and `en_GB` contain the key, keep placeholders identical, export localisation, and restart the test game.

## Actor Does Nothing

**Likely causes**

The actor has no collision, the trigger page is wrong, the tag does not match, the condition cannot become true, or a blocking window has consumed input.

**Checks**

Test the actor on a small map, verify its tag, and add a temporary print or message event to confirm the page runs.

## Save Looks Wrong

**Likely causes**

The save was made before the change, the actor tag changed, terrain edits use different map or layer names, or the custom saved field has no default.

**Checks**

Test with a new save and an old save. Confirm position, inventory, equipment, states, removed actors, changed terrain, and floor points.

## Audio or Effects Fail

**Likely causes**

Audio switches or volume are off, the file is in the wrong folder, the map uses another sound setting, or effect timing is too short to notice.

**Checks**

Test the sound from a simple event, then test it in the final map flow.

## Plugin Tool Does Not Appear

**Likely causes**

The plugin folder name does not match `Plugin.json`'s `name`, `types` contains an unsupported value, `minEditorVersion` is newer than the current editor, `Plugin.py` raised during load, or a required plugin-local import was delayed after the temporary `sys.path` entry was restored.

**Checks**

Confirm the plugin is under `Plugins/{PluginName}`, check the startup warning, keep plugin-local imports inside load-time code or load them through `EDITOR.plugin_dir`, and restart the editor. For custom data types, reload the project after changing `datatype` hooks so plugin data is cleared and loaded again.

## Console Clues

Use the editor console filters to separate normal messages, warnings, and errors. For event debugging, a short print action is often faster than guessing which branch ran.
