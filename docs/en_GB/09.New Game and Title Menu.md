# 09. New Game and Title Menu

The Sample game starts at the title scene. The title scene is the first runtime check for settings, language, scale, audio, save loading, and new-game data.

## Commands

**Start**

Creates a new game instance and enters the configured start map. The default start map is `Map_01.dat` and the default start position is `(6, 12)`.

**Continue**

Opens the save-load window in load mode. Loading restores position, player values, inventory, equipment, states, changed terrain, removed actors, floor teleport records, and the current map.

**Config**

Opens the runtime config window. It can change language, scale, frame rate, vertical sync, audio switches, and volume values.

**Exit**

Closes the test game.

## New Game Player

The default player is the Braver blueprint. It starts with HP, attack, defence, level, class, inventory, and equipment data from the Sample project.

## Runtime Setting Access

The generated runtime config mirrors `Main.ini`. Game code can read the active values through the system config layer. A typical custom window or scene should prefer existing config accessors instead of reading `Main.ini` again.

```python
from Global.System import System

language = System.getLanguage()
scale = System.getScale()
```

## Test Notes

Language and scale are easiest to verify by closing and reopening the test game after changing them. Save loading should be tested with a fresh save and with a save that already contains removed actors or changed terrain.
